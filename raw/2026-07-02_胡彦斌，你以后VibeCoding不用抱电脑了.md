# 胡彦斌，你以后Vibe Coding 不用抱电脑了

**作者**: 发现明日产品的

**来源**: https://mp.weixin.qq.com/s/jeKweD2IzEOs1-rkBKW5Wg

## 摘要

过去Vibe Coding依赖时刻守在电脑前与AI对话，但随着Cursor和OpenClaw发布手机端App，用户如今可通过手机启动、远程控制和审批电脑或云端Agent，并在任务节点接收推送提醒。这打破了传统AI产品“仅限聊天窗口”的单一交互模式，让人在AI持续执行长时间任务时彻底摆脱电脑束缚，实现真正的移动端掌控。

Vibe Coding 最经典的一张照片，大概就是走到哪里都要带着自己的电脑。

胡彦斌前段时间在小红书发布了一张在路上拿着笔记本电脑的照片，配文说「Vibe Coding 的都懂这个姿势。」

![](https://mmbiz.qpic.cn/mmbiz_png/dCG7OC48IfJpRMPkRH3cO4XfEsN9CChrUM7RzFILONyYcJw7PibBLTyBgwLLlurZKGUzQeWVrJC93BFJljoHSHPdIvGb0GTBK425uIZUYb4Y/640?wx_fmt=png&from=appmsg)

我们几乎习惯了用 Agent 就是用电脑，带着安装了 Claude Code、Codex、Cursor、OpenClaw 的电脑，到咖啡店、会议室，坐在电脑前，不断和它对话、修改、等待新的结果。

![](https://mmbiz.qpic.cn/mmbiz_png/dCG7OC48IfLmI841ic9mS3KecdIPJUCNJXfN12ohuaMNhjsMDMicpqARkDzoPbtExoK2icr2zEGHibw6604vl7xIxgdaVtzNJybYic0mPc7dVRjU/640?wx_fmt=png&from=appmsg)

但从这段时间的进展来看，这些守在电脑前等 AI 回复的那些事，可以靠一部手机完成了。

引领了 Vibe Coding 潮流的编程工具 Cursor 发布了 iPhone App，以及引领了今年 Agent 潮流的 OpenClaw 也发布了 iOS 和 Android 原生 App。

很难说它们曾经确实有代表过某种产品的趋势，现在就还能继续作为下一个风向标。但是把两个 App 的发布放在一起看，我们会发现它们回答的是同一个问题：

**当 AI 开始持续替我们工作之后，人还需要一直守着电脑吗？**

Cursor 在官方博客说，新的 App 既可以允许我们在手机上启动云端 agent，也能远程控制电脑上正在运行的 agent。

它会在 agent 完成任务、需要输入、准备好 review 时，通过推送和锁屏实时活动提醒我们。

App Store 页面也把它定义成一款「AI coding agent for big idea」，功能包括启动 agent、查看截图和视频、检查 diff、用语音对话、合并 PR。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/dCG7OC48IfKvxqYElVYPKGpurQNUyT2nuKe6iaCUl7JMY4PicIHtsxXvBibragQAweibpiag5PThD8BSpTvNoVrlHcV7oGpD6hCfLIySwNR6m4as/640?wx_fmt=png&from=appmsg)

而 OpenClaw 在手机端的明确形态，也和电脑上的 OpenClaw 一样，「 **Personal AI on your device** 」，运行在你设备上的个人 AI 助理。

它把手机作为一个节点，这个节点也能像 Cursor 一样用来审批和跟进，但还能聊天、语音、在用户允许时接入相机、屏幕、位置和通知。

聊天窗口不再是 AI 的全部

过去三年，大多数 AI 产品都围绕着同一种交互展开。

打开 ChatGPT，输入一句话，等待回复，继续输入下一句；整个过程建立在一个默认假设上：AI 只有在你和它对话的时候，才开始工作。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/dCG7OC48IfKGicGQYPDOsrKkr51hCWMdkTOT5jJpZbjhNNaxlghrjicMd3hQap7ISdYQTUVgbxIhJzKOB3E67FHW7oGhPzDLatT8mMCWH3DxI/640?wx_fmt=png&from=appmsg)

因此，无论是网页版、桌面版还是手机 App，不同的平台，承担着几乎相同的角色，本质上都只是一个聊天窗口。

Agent 的流行，开始改变了这一点。

眼下无论是写代码、分析数据、整理文档，还是执行复杂工作流，一次任务往往都会持续几十分钟，甚至几个小时。我们已经不需要在输入框里，一步一步地告诉 AI 接下来该做什么。

更多的时候，都是 Agent 自己在持续推进任务。真正需要我们参与的，只剩下一些关键的节点，像是确认是否执行某个操作、审核生成的结果，甚至是批准支付和发送信息等。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/dCG7OC48IfJBqLy4iasmbAjRgHvy3RPPia32rxWzyyibg5U6zbicKdAbAdzgyibx4oabSJv3mycCVDn4SbNUStpHF78rm5KlsBtPvwglEmEhwYEM/640?wx_fmt=png&from=appmsg)

于是，所有的 Agent 都开始寻找新的交互方式。最早就是通过飞书、Telegram 这类 IM 即时通讯工具来承担审批的角色，后来发现聊天窗口容纳的信息太有限了。

Claude 先是推出了 Dispatch 和 Remote Control 等远程控制的功能，让我们在手机上就能操作电脑上的 Claude Code。

OpenAI 也很快在 ChatGPT 内增加了 Codex，我们已经可以直接在 ChatGPT 手机 App 中查看任务状态、接收通知、审批操作，即使离开电脑，也能继续管理正在运行的 Agent。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/dCG7OC48IfKgNn3JGGcf8j2y0eYxer2o16y4TB2x30b9QbnKkL3qj2WLITp3L2kqiatmcrEU6AmkyA3aKg9Kn488ich4kpwzQqQtChO8yD5RI/640?wx_fmt=png&from=appmsg)

根据外媒的爆料，OpenAI 更是扬言要重做 ChatGPT，将 Codex 的全部核心能力直接融合进 ChatGPT。

ChatGPT 和 Claude 本身就是作为通用聊天助手的手机应用，而对专门在电脑上使用的 Cursor 和 OpenClaw 来说，给手机做一个新的 App，似乎也成了某种心照不宣的选择。

Cursor，把 Coding Agent 装进手机

就像它在 App Store 里面分类还是开发者工具一样，Cursor 的路线是继续围绕工程任务展开，只是把 agent 从本地编辑器延伸到云端和手机。

我们可以从手机选择仓库，启动一个 agent，指定模型，用语音描述需求，也可以把本地计划交给云端 agent 继续跑。等它产出结果，再在手机上看截图、日志、diff，决定是让它继续改，还是直接合并。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/dCG7OC48IfJebZHOwF5zqq41j19L6X1I5PRicOoW77LI2KEegByL08GicCGybPUw6j4ic8y0siaSjRGBfdZW2jugIC23mDoXD3Z67teJ48UtvIk/640?wx_fmt=png&from=appmsg)

这听起来像一个给程序员做的移动工作台，但它真正解决的是异步工作的问题。

想象一下，你已经合上电脑，准备下楼吃饭。

半小时前，你在 Cursor 里丢给 agent 一个任务：修一下用户反馈里的页面错位，顺便跑一遍测试。然后在等电梯的时候，手机锁屏上弹出一条提醒：任务跑完了，生成了 diff、截图和日志，现在需要你确认下一步。

你不需要打开电脑，点进 Cursor 手机 App，看一下改动范围，圈出截图里还没对齐的地方，补一句「这里再往右 8px」，agent 又继续干活。吃完饭回来时，它已经准备好一个可以合并的 PR。

![](https://mmbiz.qpic.cn/mmbiz_png/dCG7OC48IfJ3cH9vZefvuMCC7iaVBuE9ibbTQvz0wBibH03aoUvszuW8hkmeCgQ1tVicj8tPuviaEAZCBI6tCavE7Fkzk05mS1Xxmmd3OSWKPOJY/640?wx_fmt=png&from=appmsg)

还有语音功能，可以直接口述提示词

以前我们让 AI 改代码，往往还要坐在旁边等；一旦看到它报错，我们就要复制错误日志给 AI 解决；它生成的内容不对，我们要回到电脑前重新描述；写完了，还要再检查。

Cursor 想把这套过程拉长，任务可以在云端继续跑，人可以离开桌面，只有当事情卡住、完成或需要判断时，手机再把我们叫回来。

目前，Cursor 仅推出了 iOS 版 App，安卓用户暂时还不能体验到，并且手机 App 也需要开通 Cursor 付费会员才能使用。

OpenClaw，把手机变成 Agent 的一部分

相比 Cursor，OpenClaw 的方向更进一步。

当电脑上的 Agent 在后台执行任务时，它可以直接调用手机提供的能力，像是摄像头、麦克风、定位、通知、Push-to-talk 等功能。

![](https://mmbiz.qpic.cn/mmbiz_png/dCG7OC48IfK1f9qNqIAfUE9A2DqI0lb3uFDCS4L5yCrhDnNA2Myddut3vibpgk3rTnYd8BaK4UsmlicWvsRl0uu0wvQ8NdUF1Pa1Py5Tj0flE/640?wx_fmt=png&from=appmsg)

OpenClaw 也有语音功能，可以直接口述提示词

例如 OpenClaw 需要确认付款时，手机弹出审批；OpenClaw 需要对现场的环境进行拍摄时，它可以直接调用手机摄像头；需要获取当前位置，也可以通过手机完成。

如果说 Cursor 是把「审批」搬到了手机，那么 OpenClaw 更像是把整个 Agent 一起带进口袋。

要承担更多的能力，也就注定了 OpenClaw 在体验上会有更多的问题。

社交媒体上的反馈也更像对一个实验性 App 的容忍：有人欢迎官方版终于出现，也有人吐槽前端粗糙、配对和设置还不稳定。

![](https://mmbiz.qpic.cn/mmbiz_png/dCG7OC48IfLWb5VTySajC0E8l6RadgFwYLibQIFibmwgEPHgSUaWiaCjagp9gpgGicZR12SibcI8tVUdLhaA65yLau7t2F9HsibNubCF1Lp77qh7Q/640?wx_fmt=png&from=appmsg)

OpenClaw 手机版目前推出了 iOS 和 Andriod 版本，在使用之前，它都必须先完成和本地 OpenClaw 的配对，手机 App 更像是它的一个频道，一个有着更多能力的频道。

从 Codex 手机版到现在的 Cursor 和 OpenClaw App，似乎都在说，一种更接近真实工作协作的交互方式：AI 持续工作，人类按需介入。

过去我们觉得「用手机写代码/做复杂工作」是个伪命题，因为屏幕太小、不好敲键盘。有些手机厂商通过用更大的屏幕，不断折叠来优化在手机上的体验。

现在，输入这个动作，越来越不受屏幕大小和键鼠的限制。 Agent 手机端 app 配合 Typeless 这类语音输入法。正在模糊掉手机和 PC 的工作场景。

而在硬件之外， **如果手机本就不是用来「生产」，而是用来「做决定和审批」的** ，手机在软件上成为一个超级控制台，似乎就是未来的趋势。

毕竟， **一个真正持续工作的 Agent，不应该要求用户一直坐在电脑前。**

我们正在招募伙伴
