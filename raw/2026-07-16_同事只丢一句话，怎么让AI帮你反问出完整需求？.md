# 同事只丢一句话，怎么让 AI 帮你反问出完整需求？

**来源**: https://waytoagi.feishu.cn/wiki/E1jjwSQiliCqBjkPS4wcO8dfnnc

---

## 摘要

面对一句模糊的需求线索，直接动手极易导致返工与“接锅”。本文介绍了一种利用AI解决该问题的方法，即不急于执行，而是先让AI将需求反问拆解成一份“需求沟通清单”，并评估出简单沟通、开会或暂缓等推进思路。同时，AI能在会前进行推演，明确必问的关键问题、预判对方回答及后续收敛方式。最终生成的清单可作为沟通时的“提词器”，帮助产品经理理清思路、避免遗漏，实现先澄清再动手的高效协作。

Hello 大家，我是阿昆。

一个 PM 最容易被坑的瞬间 ，就是别人丢来一句话，你立刻回了句：好的。

老板说：做个项目执行数据看板。

你说： 好的。这就开始危险了。

这句话里没有说清楚： 看板给谁看 、周会怎么用、老板到底想催进度还是看风险、项目负责人怎么解释数据、第一版做到什么样算可用。

听起来像需求，实际你把 线索当需求 接了。

最危险的地方在这里

**只要你开始画页面、排需求、找研发，这个需求就从“他没说清楚”变成了“你没理解对”。**

如果你是产品经理、项目经理、运营，或者任何经常要把别人一句话变成可执行事项的人， 这个方法就是写给你的 。



我最近做了一个 Skill ，专门解决 PM 最常见的接锅场景。

它不帮我写 PRD，就是先帮我把这句话反问拆解成一份 需求沟通清单 。

# **01**

**PART**



**先帮我反问**

ASK FIRST · 先澄清再动手

最容易返工的需求，一开始都是一句很正常的话。

老板说要一个项目执行数据看板。

你第一反应，行，不就是数据看板么。系统里一共就这些数， 开搓吧 。

**实际上呢？**

  
**“项目”是哪类项目？**

  
**“执行”看进度、风险、负责人动作，还是交付质量？**

  
**“数据看板”是周会用、汇报用、日常监控用，还是追责用？**

  
**“老板要看”不等于“老板会用”**

  
**“有数据”不等于“能支持决策”**

先让它来“采访”你一下：你知道什么、 价值清不清楚 、哪些人有必要拉进来。

![](https://feishu.cn/file/ETEybUCpuo64MWx7w69cqj0Bn3b)



能补充的就给一下，不能回答的让 AI 继续帮你拆 。

我这里面给需求设定了三种 推进思路 。

**1**需要约会 ： 需求价值有，但待确认的东西也多，而且不止要跟一两个人确认。

**2**简单沟通 ： 需求价值有，还有几个小问题，确认了就可以推。

**3**暂缓推进 ： 没看到需求价值，说白了，不知道为什么做。建议你再想想，跟对方再聊聊，别着急往下推。

不过即便 AI 判断的是暂缓推进，只要你说需要，它都可以 继续分析 。

![](https://feishu.cn/file/GQBHbwfsLoAUrMxdDsFcH59Tngd)



这一步，你需要重点看一下它给的 建议确认人员 的类型。

不需要的、不存在的、约不到的，直接告诉它。这样后面拆解的问题也 更精准 。

简简单单，它就给你拆好 沟通清单 了。

![](https://feishu.cn/file/QWIubwxHmo3XYAxUgNxcRldVnRh)

最终产出物是一份 Markdown 文档 和 HTML 阅读版 。

建议直接看 HTML 的，可读性好一些。

# **02**

**PART**



**真正有用的是：问谁、问什么、怎么收敛**

MEETING PREP · 会前推演

先看这次沟通到底 要解决哪几件事 。

要优先确认的，是会影响后续执行方向的 关键问题 。

![](https://feishu.cn/file/DUE4b9rfCocAX4xFqHqcHMzKntY)



再扫一下目前已经明确的， 还需要确认的点 都是什么。

![](https://feishu.cn/file/JTkybJk2HoPS6TxMyD7cQAM5n6l)

**最重要的来了。**

问题清单 。

好容易约次会，回来才发现 漏了要确认的东西 多尴尬。

AI 会告诉你：

  
**哪些问题必问，哪些可以后问**

  
**对方可能怎么答**

  
**如果对方这样答，你下一步怎么收敛**

帮你在会前做 沟通推演 ，捋清思路。

![](https://feishu.cn/file/LppPbGj2goKXA4xjiPQcviYwnjd)



最后，再帮你敲一遍重点。

![](https://feishu.cn/file/YlE5bPNFFomDpbx0llNcgmhmn5c)



这份材料的角色

**它就是你的“提词器”。**

开会的时候，把这份材料放着，开开录音，你就可以专心沟通了。

# **03**

**PART**



**要怎么用？**

HOW TO USE · 三步启动





先判断，再沟通，最后才进入方案

第一步：保留需求原话丢进去

第二步：回答它反问你的背景问题，给它补充一点信息

第三步：确认推进方式和确认人员，让它生成沟通清单

**帮你理清背景**

  
**这句话是谁提的？**

  
**他是使用者、拍板人，还是转述人？**

  
**这个需求想解决谁的问题？**

  
**你能不能说出它的用户价值或业务价值？**

  
**这件事是在什么场景下被提出来的？**





**帮你生成清单**

  
**应该找谁确认**

  
**哪些人没必要拉进来**

  
**会议前要准备什么**

  
**哪些问题必须问**

  
**哪些问题可以后问**

  
**对方可能怎么回答**

  
**你听到不同回答后该怎么收敛**

# **///**

**LAST**



**小结**

LAST · 写在最后

总的来说，就是 **先判断价值** ，再补充信息。

  
  
workflow

```Plain Text
一句话需求
→ 先补背景和价值
→ 判断推进成熟度
→ 找对确认对象
→ 生成角色化问题
→ 带着会前材料去沟通
```

这个 Skill 就是一个 需求沟通会前准备器 。

目的不是为了让你“照本宣科”，是为了 防止你漏问关键问题 。

真正要解决的事

**帮你消除“不确定”带来的执行风险。**

下次再有人只丢一句话给你， 不要先回“好的” 。

先把原话交给这个 Skill，让它帮你判断这事该不该推、找谁确认、后续沟通怎么问。
