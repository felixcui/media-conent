## 摘要

本文分享了利用AI自动化生成复习资料的实践案例。受家长制作物理信息图的启发，作者将知识点转化为信息图的完整规则封装成专属Skill。用户只需输入知识点名称，该Skill便能自动识别学科与年级、拆解核心概念、匹配专属配色，并生成精准的插图描述指令，从而一键输出高质量甚至动态的学科复习图，有效解决了家长重复编写提示词的痛点。

## **第一步：做一个 Skill，丢知识点就出图**



既然要做，就直接做成 Skill，不想每次都重新写 prompt。



我把 **怎么把一个知识点变成信息图** 的完整规则封装成了一个 Skill，叫 linyuebanzi-edu-infographic 。它做的事情很简单：



💬

你给它一个知识点名字，它自动完成：



**识别** → 声现象？初二物理第二章。勾股定理？初二数学。光合作用？初一生物。自动判断学科、年级、章节。



**拆解** → 把知识点拆成 3-5 个核心子概念。声现象 = 振动发声 + 声波传播 + 音调响度音色 + 声速。



**选色** → 同学科不同章节自动分配不同颜色，不会出现全蓝的尴尬。



**画图** → 每个子概念配具体的插图描述——不是"画一个声波"这种模糊指令，而是"音叉左右振动箭头 + 声波扩散弧线 + 旁边标注'振动产生声音'"这种能直接执行的描述。



```Plaintext
linyuebanzi-edu-infographic/
├── SKILL.md
└── references
    ├── accuracy_checklist.md
    ├── errata.md
    ├── illustration_guide.md
    └── prompt_template.md
```



Skill 搓好了，直接上手试。先从老王做过的声现象开始：

![](https://feishu.cn/file/TW4qbcPfAoPuRxx1Nj8c2UNOnsd)



![](https://feishu.cn/file/WoKBbBljmoNWlNx5pDJcEtj5nXg)

四个知识模块，声音的产生、声音的传播、声音的特性、声速。暖橙配色，音叉振动、鼓面声波、三种特性的波形对比都画得清楚。底部还有一条"学习提示"，先判断是否振动产生，再判断是否有介质传播，最后区分音调响度音色。



声现象跑通了，接着试了几个不同学科，看 Skill 的泛化能力怎么样。



### **理科**



物理丢了一个"光现象"，生物丢了一个"光合作用"，化学丢了一个"空气的组成"。三个都是理科，但配色自动做了区分，光现象偏青绿，光合作用偏草绿，空气的组成偏灰绿，不会搞混。



光现象的三棱镜分光七色光画得很到位，光合作用的叶绿体截面图、反应方程式、影响因素的曲线图全部到位，空气的组成饼图标出了氮气 78%、氧气 21%，红磷燃烧测氧气含量的实验图也画上了。







### **文科**



语文试了"静夜思"，出来一张暖棕色的古诗卡片。原文、字词注释、诗意、主旨，四段拆解，还配了月光和窗户的小插画。



英语试了"一般现在时"，蓝色系，句型结构、例句、时间标志词都整理好了，三单加 s 的规则还特意高亮。



历史试了"丝绸之路"，沙漠配色，张骞出使、路线图、交流物品、历史意义，骆驼和丝绸的图标很有味道。



六个学科，每张丢一个知识点名字就出来了，不用改 prompt、不用调配色、不用想排版。这就是封装成 Skill 的意义。



但要说一句实话： **AI 出的学科信息图，必须做一次交叉审核再给孩子用。** 生成速度很快，但不等于可以无脑信任。



我跑欧姆定律的时候就被 DeepSeek 抓出过两个硬伤，控制变量条件没加、伏安法电路图漏了滑动变阻器。发现之后把这些坑加进了 Skill 的勘误表，下次生成就不会再犯。



Skill 已经开源了，装上就能用，不用自己从头搓。用的时候发现知识点有错，直接在 GitHub 上提 Issue，我会同步到 Skill 的勘误表里，下次生成同一个知识点就自动修正。





## **第二步：从静态到动态**



信息图做完了，但老王那句话还在我脑子里，"直接动态显示"。



静态信息图有个天然的短板：它讲不了"过程"。声现象这个知识点，核心就是振动、传播、波形变化，全是时间维度上的事。一张图里波形是死的，"频率越高音调越高"只能靠文字标注，学生看到的还是结论，不是现象。



还有一层。图挂在墙上，孩子看不懂的地方还得家长在旁边讲。要是它自己会讲呢？



所以我又做了一个 Skill， linyuebanzi-teaching-animation 。丢一个知识点进去，能出无声动画也能出带配音的完整教学视频。无声的适合文章里内嵌，重点说说配音视频。



直接用上一篇文章里的将军饮马，读者看过交互版，正好对比一下视频版：



💬

7 段结构：开头用故事引入（将军从军营到河边饮马再赶到帐篷，怎么走最短），然后抽象成数学模型、作对称点、连直线找最优点、讲清楚为什么最短，还延伸了正方形和坐标系里的变形题，最后一句话总结"作对称，化折为直"。分镜、配音、字幕、动画全自动。



上一篇文章里，我儿子是自己拖动 M 和 N 点拖明白的。这次视频版不用拖，动画一步步演示"为什么要做对称、怎么化折为直"。



**一个让你动手试，一个帮你看懂原理，两种方式解决同一道题。**



怎么做到的？整个流程是这样的：

![](https://feishu.cn/file/GWsBbptj2oevOpxoiPpcZGgLnEU)

核心设计就两条： **先配音后画面** ，音频定节奏，画面跟着对齐； **AI 管创意，脚本管纪律** ，拆知识点和画图交给 AI，时间轴、配色、字幕这些机械活全是脚本自动跑的。



跑的过程中也踩了不少坑，第一版像会动的 PPT、公式符号被字体吞掉、场景编号对不上。但这些坑修完都进了 Skill 的规则表，下次生成就不会再踩。



**这也是封装成 Skill 和每次现写 prompt 最大的区别。**



## **两个 Skill，两种用法**



这次做了两个 Skill，一个出静态信息图，一个出动图和配音视频。两个用的是同一套配色规则，同一个知识点丢进去，出来的图和视频是一个调子，格式随你选。
