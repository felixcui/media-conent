## 摘要

本文介绍了GitHub上的开源AI项目Cua，这是一个专为“computer use”打造的开源工具箱，而非直接可用的AI助手。它包含操作桌面应用的Drivers、提供可控环境的Sandbox和用于测试的Bench三个核心模块，支持Linux、macOS、Windows及安卓等多平台本地或云端运行。普通用户可通过简单安装将其接入支持MCP的Agent工具中，从而赋予AI像人一样操作电脑的能力。

## Cua

第一个是 Cua，这是一个在 GitHub 斩获了 19.5K Star 的项目。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/9wvRWyQEWCg0XIxoCf6dnJFh3tAy4cttdCpILvFef4iahw4ibjKAhXBNqjBBn9uxaiaHFvRwvdibYuxf1M32W49dwqWYDgTMIn9EJwy1QZGA8FE/640?wx_fmt=png&from=appmsg)

最近大家应该经常听到一个词，叫 computer use。简单说，就是让 AI 不只是在对话框里回答你，而是真的能看屏幕、点按钮、输入文字，像人一样操作电脑。

Cua 做的也是这个方向，但它的位置不太一样。

它不是一个新的 AI 助手，也不是打开就能用的聊天工具。更准确地说，它是在做 computer use 背后的开源工具箱。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/9wvRWyQEWCjhYZcc9wriclublmnUuibkwWM7pQ7gkx2OnHNb3vic1wsqZQ7mmKyGOuWmLJ2mM3CYuKEnfN1hPzJ9x758dibt2FOnAnHKLXQj1oA/640?wx_fmt=png&from=appmsg)

它里面有三个核心模块：

Cua Drivers 让 Agent 可以操作桌面应用；

Cua Sandbox 给 Agent 一个可控的电脑环境，用来截图、点击、输入、跑命令；

Cua Bench 用来测试这类 computer-use agent 到底靠不靠谱。

![](https://mmbiz.qpic.cn/mmbiz_png/9wvRWyQEWChujYV5VF6t0EYq7XgbGZPQexiaK7KQ8eazW0GfPuZo4MKkHoNibuXLmhynxclpELyGly3xO1MIRXPJmZgQfQGr0fySqmicFpn88o/640?wx_fmt=png&from=appmsg)

比如 Codex 自带的 computer use，是一个已经包装好的成品能力。你在 Codex 里用，它就服务于 Codex。

![](https://mmbiz.qpic.cn/mmbiz_png/9wvRWyQEWCiayO4D7ibH1y8fbwPk7Rn4dnzEQKeJ3J9VQcJl4HFum18eR9Mp2DxhE2cdxX5C4Pztt1BuNmickNibtCic3hFBuNazOJZ0m18Weoz8/640?wx_fmt=png&from=appmsg)

因此 cua 对咱们普通人来说，最大的作用就是在用小龙虾、Codex、Claude Code，或者其他支持 MCP 的 Agent 工具，可以尝试把它接进去，让 Agent 多一层操作电脑的能力。

更特别的是 cua 是支持操作 Linux、macOS、Winodws 甚至安卓系统，运行环境可放到云端或本地。

![](https://mmbiz.qpic.cn/mmbiz_png/9wvRWyQEWCgiaYuNR1Iic9TUiauMdDUAspDG9Y7ht5Fcl9tfQsBF45KVaiaNolB006Jr3pqB0cJicO2f0UhFWcmLFrqv2IfS3mzphQ0MA5UyKUYA/640?wx_fmt=png&from=appmsg)

接入的方法也比较直接。

Windows 用户可以在 PowerShell 里运行官方安装命令 👇

```
irm https://raw.githubusercontent.com/trycua/cua/main/libs/cua-driver/scripts/install.ps1 | iex
```

装好之后，再把 cua-driver mcp 加到支持 MCP 的 Agent 工具里就好了。

```
claude mcp add --transport stdio cua-driver -- cua-driver mcp
```

开源地址：

https://github.com/trycua/cua

## Agent Reach

这个项目我非常推荐大家安装。

平时让 AI 搜东西、看网页、查资料的时候，大家应该都遇到过类似情况：网页读不出来，视频字幕拿不到，Twitter、Reddit、小红书这些平台一查就被拦，最后 Agent 只能告诉你「我无法访问」。

![](https://mmbiz.qpic.cn/mmbiz_png/9wvRWyQEWCiamL3xmUIn2DBsmiaGgiat27qxyeE5VJBzaZibJMEHrUwsjOVtCjv0CLwDqQFtcibWFML8pGwMNQr2yJiaqTibmCPomQtRnRPhtjgvZw/640?wx_fmt=png&from=appmsg)

Agent Reach 解决的就是这个问题。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/9wvRWyQEWChFW1rtkOUyGOR9beRbaS4pdEwmn6ePGWIm60X2XHRnJWWwWVeYTcITC2bZwtqRDezrCdNiaIQ5t5xSegnOrmZfrlZsFfLRicZP4/640?wx_fmt=png&from=appmsg)

它把网页、YouTube、RSS、GitHub、Twitter/X、B站、Reddit、小红书、LinkedIn、V2EX 等渠道，整理成一套 Agent 可以调用的工具。

你不用每个平台都单独研究怎么接、怎么登录、怎么绕过限制，它会帮你选当前更稳的接入方式。

![](https://mmbiz.qpic.cn/mmbiz_png/9wvRWyQEWCjbfsaaTOjFWV1llWhCgNjoSyqNvNGlI4InAjU4EHe2GIuOBWteVqickhd0Be547sgDxV8bhQsjQBAjgu9riclALM92jibIbuU5JY/640?wx_fmt=png&from=appmsg)

安装也很省事。你只需要把这句话发给自己的 Agent

帮我安装 Agent Reach：

https://raw.githubusercontent.com/Panniantong/agent-reach/main/docs/install.md

后面它会按文档一步步安装。装好之后，你就可以更自然地让 Agent 帮你浏览、搜索、读网页、看视频、查 GitHub 项目，或者去小红书、Twitter、Reddit 上找信息。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/9wvRWyQEWChmoWHgyd4ctS1icRRP62lgaM7HhwhAK7mL9x0epsvYzJiappLKw0txfCNngCJFyhTXbrz85Tg0rSGKUEAabaiaoKqObtfcZicJIP0/640?wx_fmt=png&from=appmsg)

![](https://mmbiz.qpic.cn/mmbiz_png/9wvRWyQEWCg61t3Dfn6qXmFuOhzySAR4TAmw1ianH64eBhaWEL4DibZ9icEBJKNxIlk4dIBgwEIRjPtNOcSm6K9ZsNc2qiaGmskfZwyNOlAYpxk/640?wx_fmt=png&from=appmsg)

当然，像 Twitter、小红书这类需要 Cookie 或登录态的平台，我建议准备一个专用小号。毕竟这类平台风控比较敏感，别直接拿主账号去折腾，万一被限制就很麻烦。

开源地址：

https://github.com/Panniantong/Agent-Reach

## agency-agents-zh

咱们一般写角色提示词，往往开头的第一句就是：“你是一个XXX专家”。

但问题是，第一句话很容易写，后面就卡住了。

这个角色应该怎么拆任务？应该用什么标准判断结果好不好？最后应该交付一份什么东西？如果是自己熟悉的领域，还能往下补几句；一旦遇到不熟悉的方向，就只能让 AI 自己猜。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/9wvRWyQEWCjE0icyGnRjLibTlz4ymbHWoWovDniaDFiaINOJib14vzdibKWe1acxoFX0ianjOkl5NIBpgNPmQcdsynH7ch1EInsQiaIXaa6HkyabRpk/640?wx_fmt=png&from=appmsg)

有时候它能顺着角色往下做，有时候只是换了个称呼，输出还是很泛。

而 agency-agents-zh ，你可以把它理解成一套已经整理好的「专家 skill 合集」。它里面有 266 个即插即用的 AI 专家角色，覆盖工程、设计、产品、营销、金融、安全、游戏等 20 个部门。

![](https://mmbiz.qpic.cn/mmbiz_gif/9wvRWyQEWCiaiaSObpUTH4tOZtFgpUg46jNZnPeNse1ybm9NwIzlFmS88eP8OicRM1pvgMnXY5mNU0aoZaiaf9f9XqO3LXH5hicwpbYd9Ek9Nrr0/640?wx_fmt=gif&from=appmsg)

这些角色不是只写一句「你是某某专家」，而是把角色身份、工作方式、交付物都提前写好了。

比如你要改 README，可以叫技术文档工程师；要做产品规划，可以叫产品经理；要检查代码，可以叫代码审查员；要做内容，可以叫公众号编辑、小红书运营、B站内容策划。

![](https://mmbiz.qpic.cn/sz_mmbiz_gif/9wvRWyQEWCgQiao8pk8AMCVMJbByM0Ek7wQj9aBWcWpEnsffaWVbBqVoccSicFn7dukeDwbRo20HSCrcusUktPGdthm5TXowd73qKPxUaD3M0/640?wx_fmt=gif&from=appmsg)

这东西最省事的地方，就是不用每次从零编角色。你缺哪块能力，就让 Agent 切到哪个角色就好。

而且相比于原版项目，这个中文版本不只是翻译英文，还加了不少更贴近国内场景的角色，比如小红书、抖音、微信公众号、B站、飞书、钉钉、跨境电商这些。

![](https://mmbiz.qpic.cn/mmbiz_png/9wvRWyQEWCjewHXTWIO8xAd8OMkzdjxicnAiaOrNiaWCsZtAvkP7c6cCdjicA3jqgQHnL5z437pB44H515Q1ricwSInpG0BMvbeFe2RxUWLCEuibw/640?wx_fmt=png&from=appmsg)

它的使用方式其实不用想复杂。

我觉得最简单的办法，是先去它的在线专家库里逛一圈，看到需要的角色，就把对应提示词复制出来用。

![](https://mmbiz.qpic.cn/mmbiz_gif/9wvRWyQEWCgwIuGrQNA3wYGPkxHqb2dlLwnPlvNAjDWXzA94haYlTaMGcIHuRsQDms0Zr79jNfZ2P5IeULN1bcvfYDtARibkPLA269lsA2vs/640?wx_fmt=gif&from=appmsg)

开源地址：

https://github.com/jnMetaCode/agency-agents-zh
