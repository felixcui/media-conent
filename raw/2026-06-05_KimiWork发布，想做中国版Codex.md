## 摘要

Kimi Work是一款面向知识工作者的通用型本地Agent，界面酷似Codex。其编程能力出色，生成网页的审美与专业度超越Codex，但缺乏计划确认模式和内置浏览器批注修改功能，运行速度也稍慢。通过内置的WebBridge插件，它能自主控制浏览器完成信息搜集与报告生成，虽然搜索策略略显笨拙，但产出报告详细具体，整体表现优秀，展现出从编程Agent向全能工作Agent转型的潜力。

Kimi work 开始公开测试了，官方说它是面向知识工作者的通用型本地 Agent。

我们划重点来理解一下，面向知识工作者包括哪些人群，比如金融、科研、法律等，他们经常要做数据分析、数据检索、报表等等。

通用型就是它已经不满足 coding 了，现在要从一个 Coding Agent 转变为 Working Agent，也就是说出了帮你写代码，现在主打一个帮你工作、干活。

它以桌面 App 的形式存在，是个本地 Agent。

它的内置是 Kimi Code。因为它是一个 Agent，所以它还提供 Agent 的基本能力，例如：安装和使用 Skill、运行定时任务等。

简单来说，这种定位让人很快就会联想到 Codex 或者是 Claude code，所以也有网友说，这是中国版的 Codex。

事实真的是这样吗？我们一起来看一下：

首先看这个界面，咱不能说很像吧，只能说跟 Codex 是一模一样。

▲ 图：界面对比

不过这个不重要，关键是它的能力到底怎么样。我这里简单举几个例子来试试看：

01编程能力

我想让它写一个简单的网站。因为我之前有一只特别爱的狗，是边牧，所以我想做一个边牧的网站。我就写这么简单的提示词，看看它会有什么效果。

提示词提交后，它会先思考然后开始干活。

▲ 图：先思考然后开始干活

这里跟 Codex 有一个明显的差异：在 Codex 里，我可以选择“计划模式”，确认完计划后再执行；但 Kimi work 目前还没有这个模式，它上来就直接开始干活了。

在干活过程中，你会发现它的界面跟 Claude Code 几乎一模一样，右边会显示任务进度。

▲ 图：右边会显示任务进度

我们在用 Claude 或者 Codex 时，正式干活前它其实会提很多问题让我们确认，但 Kimi Code 目前好像还没有这个环节，它是直接上来帮你做。

这样可能会导致一个问题：因为它没有进行需求确认，做出来的东西跟我们想要的可能会有一些差异。

我对比了两个测试的速度，相对来说 Kimi 还是比较慢的，不知道是不是因为我用的版本的问题。

做完之后我们看一下这个效果，发现它整个审美是非常好的。但有个问题，就是你看涉及到边牧的这些图，它全是用 SVG 画的，这就会产生很严重的问题。

▲ 图：涉及边牧的图全是用 SVG 画的

不过这个其实很简单，我把它替换成真实图片，它很快就帮我完成了。

最终生成的页面，说实话真的非常专业、非常美观。

▲ 图：最终生成的页面

说真的，这个审美、信息丰富度、专业度都超过了 Codex。

▲ 图：Codex 生成的页面

不过这里有个问题：目前它还没有内置浏览器，所以我们打开网页看效果时，只能用默认浏览器去看。

这就会导致另外一个问题：我们没法在内置浏览器里像使用 Codex 一样，在想要修改的地方直接做批注并修改。

目前在 Kimi Work 里面，如果你想修改，还只能在对话框里写一大堆描述，告诉它该怎么改。

相对来讲，这还是比较麻烦的一点。但我相信这些功能，他们后面应该都会加上。

02浏览器控制

第二个我想测试的功能是它的浏览器控制功能。

Kimi work 内置了 WebBridge 的浏览器插件。通过这个插件，你就可以控制整个浏览器了，就跟人自己操作一样。

我们把这个插件装好之后，就可以直接在 Kimi work 里面去使用了。

比如说，我跟他说用 WebBridge 帮我查一下 X 上面最近一周的 AI 热点，那很快他就会打开浏览器访问 X，然后搜索相关的 AI 信息。

这方面有一个很有意思的现象：它会把每一个关键词都打开一个页面做搜索，然后一下子打开十几个页面进行搜索。

▲ 图：一下子打开十几个页面进行搜索

这个过程很快，它会迅速把搜集到的信息整理成一份数据给到我。

▲ 图：迅速把搜集到的信息整理成一份数据

这种方式其实有点笨，但不管怎么样，我觉得它还是能很好地完成这个任务。而且他的报告真的是非常详细、具体，这远超了我的预期。

我之前也用过 OpenClaw 这样的工具，发现它们做得非常粗糙且运行很慢。在这一方面，Kimi Work 真的非常优秀。

整个中间过程完全是自主操作，不需要我参与。但正常来说它可以再聪明点，把多个关键词合并成一个复合搜索来做的。

不管怎么样，我觉得浏览器操作这个功能已经非常强大了。我们都知道 CodeX 可以操作浏览器，Claude Code 也可以操作浏览器。其实作为本地 Agent 来说，操作浏览器是一个非常重要的技能。（操控电脑也是，但这个版本的 Kimi work 目前还不具备）

03Skills

第三个方面就是 Skills。

我们都知道不管是 CodeX 还是 Claude Code，它们其实都有大量的预置 Skills 可以使用，当然你也可以自己写 Skills、自己安装 Skills。

那 Kimi work 目前也是一样的，只不过它内置的 Skills 相对来讲会少一些，而且它没有像 CodeX 那样的插件，目前还只有 Skills。

这里包括了常用的办公相关的工具，比如说处理文档之类的的。当然还有蛮多是跟投资、股票分析相关的，我觉得大家可以试一下。

▲ 图：Kimi Work内置的 Skills

为了演示，我这边先来看一个做 PPT 的吧。

跟它的工具里面一样，调用一个 Skills：

1先安装

2安装好了之后，在对话框里面用斜杠命令引进来

3再附上你的提示词就好了

▲ 图：调用 skills 生成PPT

我发现很快他就把这个生成好了。

▲ 图：生成好的PPT

说实话，他这个审美还真的是很不错，跟很多同类产品相比，确实让人眼前一亮。

04数据分析

最后一个是我想简单做的数据分析。

说实话，正常的数据分析可能需要大量的数据源，然后做很深度的分析；但我这边因为没有这么多数据，所以我就简单来做一个演示。

我想让他做的是英伟达（NVIDIA）近三年的财报分析。我写了这么一个题目，看它会怎么样。

他首先在没有提供数据源的情况下，会自己先到网络去搜索。搜索之后，他会做一个比较全面的分析，最终生成一份 Excel 形式的报表给我。

▲ 图：最终生成一份 Excel 形式的报表

我看了一下，基本上还是不错的，但确实这里面的报表从美观度上来讲还是差一些。至于具体的数据怎么样，我还没细看。

所以如果你做数据分析的话，我建议你重点利用这些功能。

以上就是一些简要的介绍。

实际上，Kimi work 还有一个很重要的功能，就是它支持 Agent 的集群。

这个功能可以根据任务的复杂度，自主创建包含多达 300 个子 Agent 的团队，帮你处理那些更复杂、复杂度更高的任务。

但这个功能需要开通会员，我这边就没有去测试了。

05写在最后

总体来说，我觉得称它为中国版的 CodeX 当然也不为过，只不过这个还算是 CodeX 比较早期的版本。

CodeX 里面很多的重要功能，其实它目前还不具备。但从目前来看，效果其实已经非常不错了。

而且 CodeX 发展到今天，其实也不是一天就做到的，也是花了蛮长时间。我相信假以时日，Kimi work 也能做到类似的程度，让我们拭目以待吧。

对了，目前仅支持 Mac 版，Windows 版据说也快上线了。想要体验Mac版可以到这里下载：

https://appsupport.moonshot.cn/api/app/pkg/latest/macos/download

另外，本文非官方推广，仅快速体验。所以测试可能不够完整、不够深入，具体怎么样不妨自己去试试。
