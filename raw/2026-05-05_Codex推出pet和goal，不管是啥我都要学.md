输入 `/pet` ，你的Codex终端里就会多一只小动物。这不是幻觉，而是OpenAI最近给Codex添的新功能。

用 `/hatch` 命令还能自定义宠物外观。但开发者们的反应很有趣：一个程序员说，"我们让AI写代码，它却给了我们情感支持宠物。"

另一位开发者评论："世界上最先进的代码助手现在有了虚拟宠物。这要么是有史以来最好的开发者体验决策，要么证明了连AI公司都知道程序员凌晨两点很孤独。"

但也有人直接开嘲："根本没人要这个功能。"还有人质疑："你们是把时间花在开发AI版电子宠物上了吗？"

根据OpenAI的官方文档，电子宠物功能是可选的。开发者可以通过设置启用或禁用宠物，还能自定义宠物外观。

![Codex app Appearance settings showing theme selection, color controls, and font options](https://mmbiz.qpic.cn/mmbiz_jpg/rY5icXvTTrJ9e1B5VaYDypibwNg3pLqvesaaC5mficXeCoQOH4CgS1R50kicVBicDLgDn85IKB2HibBsbSicWp0xUb2XxFpV6ru0B4z3t8croXdnUc/640?wx_fmt=jpeg&from=appmsg)

同时，Codex CLI 0.128.0版本推出了 `/goal` 命令。这个功能要实用得多，允许开发者设置长期目标，让AI助手跨对话轮次持续追踪进度。

![](https://mmbiz.qpic.cn/mmbiz_jpg/rY5icXvTTrJibBooSw9aaBTFJ4QankyKJSt3PmSibjCwkwFtbURcibT7aRPYdIBuQTljVgkbiausbv5jgVcM3I88fXT2On4uDwsDCib3iayA0uc1yI/640?wx_fmt=jpeg)

开发者Felipe Coury说，这是"对Ralph循环的重新诠释"，旨在让目标跨对话轮次保持活跃。该功能由Pyright作者Eric Traut开发，目前仅在CLI版本中可用，需要在配置文件中启用：

```toml
[features]
goals = true
```
