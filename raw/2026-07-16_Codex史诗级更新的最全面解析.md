# 1、Work 到底是什么？和 Codex 有什么区别？

此次 Codex 最大更新在于新增 Work 区域。很多人关于 Work 最大的疑问在于，其界面几乎和 Codex 一模一样，两者具体有什么区别呢？

Work 更适合研究、分析和内容交付。它默认围绕项目文件、网页等业务资料展开，常用搜索、浏览器和文件生成工具，最终产出文档、表格、PPT 等成果。

Codex 更适合软件开发和可执行任务。它默认围绕代码库、当前目录、Git 状态和项目规则展开，常用 Shell、文件编辑、构建、测试和 Diff 工具，最终交付一组经过运行验证的修改。

（具体区别见下图）

![](https://feishu.cn/file/HgmEbFUhNoIl3Axg35McKzXHnL5)

总结而言，Work 擅长把资料变成成果，Codex 擅长把代码变成经过验证的改动。研究、分析和内容交付优先使用 Work；软件开发、调试和代码修改优先使用 Codex。

# 3、内置浏览器更新：将Chrome装进Codex里

Codex 内置浏览器得到了多处重大更新，主要包括：

1️⃣ 支持直接导入谷歌账户信息，包括网页cookie和密码

2️⃣ 支持同时打开多个标签页

3️⃣ 支持直接下载网页文件

![](https://feishu.cn/file/GoPQbG97DonL84xm14Yccdcgncb)

该更新使得Codex覆盖了所有常见的浏览器工作，网页浏览、资料下载、页面调试等任务现在都可以在Codex里直接完成。

# 5、Ultra mode：多agent并行执行任务

在推理强度方面，本次更新增加了新模式ultra。

![](https://feishu.cn/file/CeGVboNbeoXm7BxmLRfcmshJnuf)

开启ultra后，Codex 会默认协调 4 个 Agent 并行工作，将复杂任务拆解为可同时展开的工作线。

注意该模式会极大程度消耗token，因此最适合复杂开发任务和深度研究等工作，普通任务建议用high即可。



# 6、将GPT网页端聊天接入Codex，适合做头脑风暴工作

点击左侧的chat，可以看到网页端的所有聊天记录。

![](https://feishu.cn/file/VOq3biwKpoKxfUxpqJ3cxJA6nig)

你还可以在 Codex 执行主任务的同时，在这里与 GPT 讨论问题、进行头脑风暴，并不打断正在执行的主任务。

如果发现讨论对主任务有价值，点击右上方 Add to task，就能把这段内容直接加入当前 Codex 上下文，非常牛逼。
