# Zed 1.0上线即爆火！Atom 原班人马用 Rust 重塑编辑器，丝滑性能 + AI 原生，堪比游戏引擎！开发者：现在一点也不怀念 VS Code

**作者**: 林芯

**来源**: https://mp.weixin.qq.com/s/xvaNMau3-TwD877j0lnKGQ

## 摘要

由Atom原班人马耗时五年打造的代码编辑器Zed 1.0正式发布并迅速爆火，该产品彻底摒弃了导致卡顿与高内存占用的传统架构，完全使用Rust语言从零重写，并自研GPU加速的GPUI框架，像游戏引擎一样渲染界面以实现极致丝滑的交互体验。Zed以突破性的高性能、AI原生能力和实时协作作为三大核心亮点，成功解决了VS Code等编辑器在处理复杂项目时的性能痛点，促使大量开发者直接转向这一下一代代码编辑器。

就在昨天，Zed 正式发布了 1.0 版本。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Zhxqo7xEfgI6WdJUPFo3tbrWzvicXv9iaZcDZwG3Dl1A63PP4mWONwJicsv3mhRMROdW3b9QhvODg9Vrn1oGyASvnTGI17rSSUH0EkQ7E6Zcp4/640?wx_fmt=png&from=appmsg)

这个历时五年开发、经历超过 1000 个预发布版本、累计百万行 Rust 代码，Atom 编辑器原班人马打造的“下一代代码编辑器”，终于走到了——官方称“大多数开发者可以安心切换的时刻”。

Reddit 有网友评论：“我几乎每天都用 Zed，快两年了。我甚至没意识到它不是 v1.0”

![](https://mmbiz.qpic.cn/mmbiz_png/Zhxqo7xEfgI9zIwBBnhcLyO5KQBpBQIYVMxQGiah7uPpqh9vRVPFiagNrsqEsZLpfJP8a0jYUXzuO3KJgian86JRczneWP1MBb4nUrIicxk6NvQ/640?wx_fmt=png&from=appmsg)

要知道，VS Code 凭借庞大的插件生态和微软背书，几乎成了行业事实标准。但近年来，随着项目越来越复杂、AI 功能深度集成，越来越多开发者开始抱怨：启动越来越慢、内存占用越来越高、输入偶尔卡顿，体验感下滑。

而 Zed 从一开始就剑走偏锋。它没有选择 Electron，而是用 Rust 完全重写核心，并自研了 GPU 加速的 GPUI 框架，目标直指“性能 + AI 原生 + 实时协作”。

那么，这个在 Github上拥有 8 万 stars 的 Zed 带来了哪些惊喜？

我们一起来看看！

Zed 1.0 ：上线即爆火

Zed 的故事，要从 Atom 说起。

2011 年左右，GitHub 推出了 Atom 编辑器，它以“可 hack”的理念和丰富的插件生态迅速走红。但随着项目越来越大，基于 Electron 的架构逐渐暴露性能瓶颈：启动慢、内存高、大项目卡顿。

这些问题最终成为 Atom 难以克服的硬伤，并且VS Code，凭借更优的性能表现与更强的生态迅速崛起，在 Microsoft 收购 GitHub 后，资源进一步向 VS Code 倾斜。

直到，2022 年 Atom 正式宣布停更。

在当时，大家的评论都是“挺了11年，终究败给VS Code”。

今天，Zed 1.0上线即爆火！

Atom 原核心团队成员彻底抛弃 Electron，用 Rust 语言从零构建一款全新的代码编辑器，还自研了 GPUI（GPU 加速 UI 框架），让整个编辑器窗口像 3D 游戏一样在 GPU 上渲染，从而实现丝滑的输入、滚动和渲染体验。

![](https://mmbiz.qpic.cn/sz_mmbiz_gif/Zhxqo7xEfgLykdhb1yswMaq8fESqoxgWGyJEp1MvVRhuagaRcspxDghqhrvvBFFsU9wMp3kiaDeAxE6XfB5agibjscfam4oZkLLpeFedicHQ2o/640?wx_fmt=gif&from=appmsg)

1.0 发布时，官方特别强调：“1.0 版本并不意味着完成，也不意味着完美。它意味着大多数开发者都能很快上手 Zed。”

那么，Zed 1.0 究竟有哪些硬核能力？

Zed 1.0 的核心功能亮点

官网的描述是：Zed 是一款极简代码编辑器，专为速度和与人类及人工智能的协作而设计。

从描述中可以看出它的亮点：性能、AI 原生、实时协作。

性能：把编辑器当“游戏引擎”来做

## Zed CEO Nathan Sobo 在Zed1.0的发布页面讲述：“Web技术为发布灵活软件提供了便捷的途径，但也带来了上限。无论我们多么努力，都无法让 Atom 比它所建立的平台更好。”

于是，Zed 放弃了 Electron 技术栈，转而使用 Rust 重写底层，并引入 GPU 加速的渲染框架。

这带来的变化也是显著的：

- 启动和打开项目速度明显更快
- 输入与滚动延迟极低
- 在大型代码库中依然保持流畅体验

![](https://mmbiz.qpic.cn/mmbiz_png/Zhxqo7xEfgIRWiaFqibyT3NxUYHXqWofwnKVmlzWXAsZzKUEicbt8KrbnKCOTDMqE1hJianILaSa36gMeYAdoqZdTRtemYsDic3H5QBFFHsUF9ek/640?wx_fmt=png&from=appmsg)

AI 原生：不是插件，而是“内置”

相比 GitHub Copilot 这类插件式方案，Zed 更强调 AI 的“原生集成”。

在实际体验中，它的 AI 能力体现在：

- 支持代码生成、修改、解释、重构等多种操作
- 提供按键级的编辑预测
- 支持并行调用多个模型 Agent

同时，Zed 允许开发者直接接入不同模型服务，例如 OpenAI、Anthropic 以及 DeepSeek 等。

![](https://mmbiz.qpic.cn/mmbiz_gif/Zhxqo7xEfgK62yUn2OHhY753YGKq1jTMI51Tg4KCiahibzO9Hc0vKZJzefvxdzibPRkWpicR8Dclo1SgBKze9rrNiaScg7B2icc0e59pPlibdTtHM8/640?wx_fmt=gif&from=appmsg)

实时多人协作

这可以说是 Zed 最具特色的一点。它基于 CRDT（无冲突复制数据类型）实现多人实时编辑，带来的体验接近在线文档工具：

- 多人同时编辑同一文件，光标位置实时可见
- 协作小组分为频道和私人对话两部分
- 支持语音沟通、屏幕共享
	![](https://mmbiz.qpic.cn/mmbiz_png/Zhxqo7xEfgIdBVthIElkxJuSpwwGPVmpV1N7CM6vB216ia9WO0LXWFF0YpvQDhUU9P8icichbrXI1crm946eHJzpuvUZ7WObSTvC8oq6hT3vzs/640?wx_fmt=png&from=appmsg)

#### 其他开箱即用的能力

此外，Zed 还内置了原生 Git 支持、书签、Markdown 预览（支持 GIF）、Vim 模式等实用功能，跨平台（macOS、Windows、Linux）体验高度一致。

绕不开的 VS Code

今天所有编辑器，几乎都绕不开 VS Code。

它的优势清晰且强大：插件生态几乎无敌，加上微软的持续投入和整合，已经是事实上的行业标准。

但正是这种“什么都做”的庞大体系，开始显露出难以回避的代价：

- **性能** ：Electron 架构带来的高内存占用、项目启动延迟、大文件卡顿，已经成为重度用户的日常痛点，且很难通过插件修补来解决。
- **AI 插件** ：Copilot 等 AI 能力以插件形式挂载，虽能用，但总有一种“割裂感”——需要切换上下文、调用方式受限，智能辅助与编辑体验并没有融为一体。
- **协作的非原生** ：Live Share 等远程协作方案强在功能兼容，弱在体验深度，延迟、断连、状态不同步等问题仍然存在，更像是“共享屏幕”，而非真正意义上的实时协同创作。

有网友将 Zed 和VS Code 进行了基准测试的比较：

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Zhxqo7xEfgJeAAt6PFcibOV6pcf71axeKu3iaTjMAVyI4icrrcgq9L1O4ib6nwWiawj9nIVzz8rKvIzcHyRnmGhxRIZnCU7NibQ5HvicEOeWhJUicAM/640?wx_fmt=png&from=appmsg)

如果你日常痛点是“卡顿”，Zed 优势明显；如果你极度依赖海量特定插件，目前 VS Code 生态仍更成熟。

开发者：非常喜欢“禁用所有 AI”的功能

Zed 1.0 发布后，Hacker News 和 Reddit 社区迅速掀起讨论。Hacker News上的开发者表示：“这是我用过的最棒的现代编辑器”

![](https://mmbiz.qpic.cn/mmbiz_png/Zhxqo7xEfgJhKibdWz7EZ0IQDuWxlrlydauGeF0rmp8tcabu7GQumVqYuqOVFvAmdW1icSwU6qEtYI4GkHrd5uQJhNnY3RoTrmicklmmkFFUe8/640?wx_fmt=png&from=appmsg) ![](https://mmbiz.qpic.cn/mmbiz_jpg/Zhxqo7xEfgJ3eiaevEsC6bibIGTMicCHicpgUuty88ATqtzcwrtWh2LsBW5wn5dfYnoGJFkaLul6sicxweOjPVzKaGzFptWuZz75LDWd8l85VEUM/640?wx_fmt=jpeg)

但是，在YouTube的官方视频下面，有网友提到一个风险点：Zed 的代码自动补全和 AI 相关特性，默认会在后台将代码片段发送给第三方 AI 提供商（例如 OpenAI、Anthropic 等），除非用户主动关闭。

1.0 之后，Zed 会往哪里走？

官方的路线图给出了答案：ACP 协议改进（让 Claude Agent、Codex 等外部 Agent 与 Zed 交互更顺畅）、Zed on the Web（网页版，支持从任意设备打开项目）、Plan Mode（规划模式，让 Agent 在修改代码前先进行完整规划）

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Zhxqo7xEfgL1Wpf3vFwMrkJcw5LAnj7s3RtnHK5Mf4nCl68lEVlIrM5RmsxxseoEBdMc3upgoZmzR4SMLQwuiaHsKkTTVJD3iaVQNDsD5C38A/640?wx_fmt=png&from=appmsg)

写在最后

“我们已经发售了超过一千个版本的 Zed，但它们都是从零开始的。”

Zed 1.0 不是“完美”的终点，而是一个里程碑式的起点。

对于被 Electron 的臃肿拖累已久的开发者来说，Zed 极可能是一个“丝滑编码的体验”。
