# 发现一个特有想象力的 Agent 开源项目。

**作者**: 阿颖

**来源**: https://mp.weixin.qq.com/s/i-qxfFUIhyVYTxJMw3Q9WQ

---

## 摘要

本文介绍了一个名为Tutti的开源项目，旨在解决当前不同AI Agent（如Codex和Claude Code）各自为政、上下文和文件孤立的问题。Tutti提供了一个实时共享空间，让多个Agent能够操作同一份文件、共享上下文和文件系统，用户可以在一个Agent中直接引用另一个Agent的会话内容，从而实现无缝协同工作，极大地提升了多Agent协作的效率。

猛的一看日历，才真正意识到 2026 年上半年已经告一段落了。

这半年经常跟朋友开玩笑，说让我梦回到了 2013 年的状态，那时候，新的 App 一个接一个。我也是那个阶段，萌生出来创业的想法。

今年前半年，模型的变化仍然是一浪接着一浪，然后新的 AI 应用也一个接一个的蹦出来。我只能想到一个文邹邹的词来描绘内心的图景：百花齐放。

我告诉自己，小伙子，珍惜这个时间吧，一辈子应该碰不到几次。

抒情完毕。今天再给大家分享一个理念超有意思的开源项目，叫 Tutti，是我刷 GitHub 的时候偶然发现的。

Tutti 在音乐术语里是齐奏的意思，所有声部同时演奏。彻底理解了这个项目后，我感觉这个名字起得真好。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/iagroyM7YaBmVvwZqG7vQE9V2UYicjotBENfSZQDicWxWabY7YEH0mHISKVV5KuKq22GA1icicHsMeKdXcXeXrPpCAa7Q5d3GWKeQYatOR9pVa1I/640?wx_fmt=png&from=appmsg)

然后我又顺着 GitHub 找到了他们产品的官网：

https://tutti.sh/

看起来是一个创业项目，还在早期。麻溜装完打开之后，我一度以为这是个虚拟机。大家看一下产品的界面。

![](https://mmbiz.qpic.cn/mmbiz_png/iagroyM7YaBl3TnFToJ8ia5kibP3QKNv3DUTG87cJGyOugx7PDy4EicbzNZGLia4oV4QUdjUuwwadhO0QiavkgPKhrBHXibibveFbibPk5us04pbBRNg/640?wx_fmt=png&from=appmsg)

紧接着发现，它可以导入 Codex 和 Claude Code 里的所有会话。哦，明白了。录了个视频。

原来他想做的事情是把不同的 Agent 打通。

现在不同的 Agent 各自为政，每个 Agent 产品的会话都只存在于自己的产品里。在 Claude Code 里聊的上下文，Codex 不知道。

在 Codex 里干的活，Claude Code 也看不到。要切换 Agent，只能手动再同步一次信息。

比如我之前的干法是让 AI 帮忙总结一次信息，然后再把这个信息复制到想切换的 Agent。

看来 Tutti 第一步要做的是把所有 Agent 放到一个实时共享的空间里。这些 Agent 可以操作同一份文件，共享上下文，共享文件系统。不用来回倒腾了。

我演示下流程。

比如我想做一个官网，在 Codex 中用 Image 2 模型生成了官网的图片。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/iagroyM7YaBn5DZ8EaePaH3GsZ7IibKwxTTb6YLPQW3YHnicFBcXHgV3UcLlhib3OFS7QXZiaBBCbOspeibsxNFhoseL7Tjf42nMhjefjxffo6sKM/640?wx_fmt=png&from=appmsg)

同时，我的 Claude Code 接入了 DeepSeek V4 Pro，因为这模型比较便宜，所以我可以让它帮我干一些难度不大的工作，比如，基于我们团队的会议录音帮我们整理产品文档。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/iagroyM7YaBnLLOs5ibC8Upb65LzALhINXlJSR3Bu2xTkwVxGOrobgloTjgPfevBDkuyaB33QLoaxeGFtia6I9NQicH0Gr02bic3pAMkpIbP887w/640?wx_fmt=png&from=appmsg)

现在等于是同一个项目，Claude Code 里有一个会话，Codex 里有一个会话。

现在我要做的是，在 Codex 中，基于我的产品设计图和产品文档，帮我把网站开发出来。

神奇的地方来了，大家看。这个时候我在 Codex 直接输入@，就可以直接引用刚才在 Claude Code 中的会话。

![](https://mmbiz.qpic.cn/sz_mmbiz_gif/iagroyM7YaBnQdkKVC3Dholkrk2tDiclNvfxyBnehNzz0vI4UUicgXCkn8k8uRRdlqfCCg8H7oVsYs87JaRlgJAicic9tJd95ovHsT3VazgPa5Ic/640?wx_fmt=gif&from=appmsg)

也就是说，Tutti 中，所有 Agent 能够共享会话、文件、任务等等上下文。

![](https://mmbiz.qpic.cn/mmbiz_png/iagroyM7YaBklCiaAE01dUMGJ28J6NCBBxF8NgybGWaSlEiakuxC0NGiadvp2ZRuTyAy3Tnshg1g4ntrIVLxzIqdP1W2x4RBwwvSuIoSPBDia3Lg/640?wx_fmt=png&from=appmsg)

这个构想太有意思了。因为过去这些不同的 Agent 之间是互相独立的软件。

这半年如果跟我一样体验过不同的桌面端 Agent 产品，就能知道我说的是什么意思。

而从更大的层面看，整个行业去年也一直在探索 Agent 之间怎么通信。现在 Codex、Claude Code 这些产品功能趋同，各干各的问题不算大，用户麻烦一点也还好。

但未来会有更多不同分工的 Agent，它们之间本身就需要协同。怎么协同？行业想到的思路还是跟之前一样，做协议。

Tutti 这个产品的设想是，让大家能够共享同一个空间，也许就不需要通信了。在这个基础上，产品就有想象空间了，这个我一会再说。

停下来我们先去解释另外一个问题。

刚才无论是 Codex 还是 Claude Code，我没有再重新登录，它直接复用的是我本机上的账号。这是怎么实现的？不会被封号吧？

研究了下代码，Tutti 不是拿到了什么特殊接口，也没有绕过登录，更不是把我的账号密码传到了 Tutti 的服务器上。至少从它开源出来的代码看，这件事的实现方式非常简单。

以 Codex 为例。Tutti 是在本机直接启动 Codex 的运行时。我的电脑上虽然没有单独装一个 Codex CLI，但 Codex 桌面 App 自己会带一个可执行文件，类似在这个位置：

```
/Applications/Codex.app/Contents/Resources/codex
```

Tutti 找到它之后，就可以像启动一个命令行工具一样启动 Codex。下面是 Tutti 中的 Codex，很明显，这是他们在 CLI 的基础上，自己做的一个图形界面。

![](https://mmbiz.qpic.cn/mmbiz_png/iagroyM7YaBm72t1Zkof8Ov1QFKL4eHCgBkk5SjD5fyyA5Q7fuVrEb9iaNOvslWxTvULFtRkkA1kMLvWsmiaZYdnqLWcKZbCwa9TpmXCkiaLVBg/640?wx_fmt=png&from=appmsg)

那为什么不用重新登录？

因为 Codex 的登录态本来就保存在本机。

比如本机的 `~/.codex` 目录里，会有认证文件、会话记录和配置。Tutti 启动 Codex 时，会为这次运行创建一个临时的 Codex 配置目录，然后把本机已有的登录文件和会话目录接进去。

可以理解成，它新开了一个临时办公室，但门禁卡用的是我电脑上原来那张。

Claude Code 也是类似逻辑。它调用本机已经安装并登录过的 Claude Code 运行时。只要本机 Claude Code 原来能正常使用，Tutti 调起来之后也能沿用这个状态。

目前 Tutti 只支持 Claude Code 和 Codex，但我看 GitHub 的文档说，他们接下来很快也会集成 OpenClaw、Gemini、Hermes 等等 Agent。

除此之外，Tutti 还有很多内置应用，比如产品原型设计、AI 幻灯片、AI 文档、视频剪辑工具。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/iagroyM7YaBkYf9K5nRbugD4micZsIfrpfUS1KK8Ik3rLzw56yUxmCHJZ4HENJosiaKIIMbdMGMNqtakQKibU1DjVXYkSlhF9qySjDqjJOIS55w/640?wx_fmt=png&from=appmsg)

打开幻灯片试了一下，发现它可以直接复用 Codex 和 Claude Code 的能力。

也就是说，这些应用是在基座 Agent 的智能能力之上，又做了一层场景化的封装。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/iagroyM7YaBm0BVFa3k3w0ibLFHZmP0VQY6602uQuo7kDE7xRDIBXg1gAJ2iaiaugJ8o2a5veYE9ShDjbxJ74J3pG3VH8DTZlaDVDMfYWnichyVk/640?wx_fmt=png&from=appmsg)

我似乎明白了 Tutti 的思路。因为它可以复用底层 Agent 的能力，所以在这个基础上做出了一批高频应用。像操作系统一样。

下面是它生成的花木兰主题幻灯片，很好看。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/iagroyM7YaBmZ3oGkdc8XJXKuSOwfibtr2ErzE8SROIXYLotsrjsMqamD1yzlGL0ckMyCHJ9NyDBgeJdLaib4XZaQt6mDZ1ynZwtdRrsxexOEQ/640?wx_fmt=png&from=appmsg)

虽然这事在 Codex 里也能做，装一个 Skill 就行。但 Tutti 把这些场景做成带图形界面的独立应用，这样对普通用户更友好。

而且像 PPT 的应用，它还内置了修改功能，生成之后可以直接在界面上调整。

市面上还有很多专门做 Slides 的付费产品。

在 Tutti 的逻辑里，它做的这些垂直场景的应用，直接可以复用像 Codex 这样的智能能力，所以不需要再额外付费。

下面这是产品原型设计的应用。逻辑跟幻灯片一样，底层复用的是 Codex 或者 Claude Code 的能力，并在这之上做了一层产品原型设计的界面。输入需求之后，它直接生成原型。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/iagroyM7YaBmwvG3wI4alMuv6abLic6zR8kXPUyVUBcgHX37UM8vBYQ2bDVNjsF8e53oRSQ18n1YwvvM5b7r04FvWsMzHt4wBc9cXsA7xPr98/640?wx_fmt=png&from=appmsg)

太有趣了。

研究了半天，看完他们官网的博客，我彻底理解了 Tutti 想做的事情。

就像我刚才说的，在 macOS 上打开，乍一看会觉得它是个虚拟机。

其实可以粗糙地理解为，它就是在操作系统之上搭建了一个虚拟环境，这个虚拟环境里跑着一套为 Agent 设计的操作系统。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/iagroyM7YaBnNUOjwcPMMKNhkIgTHghK6GqaQyq1YIRqZxN7lOSbq32ibhj6Svqnvg9qa0UBX6tudynuGeLDSSYa6gTBMb4B0byJOgN7TWSvc/640?wx_fmt=png&from=appmsg)

这个操作系统做了两件事。

第一，它复用宿主机上的 Codex、Claude Code、OpenClaw 这些 Agent 的智能能力。

第二，在这些智能能力的基础上，又提供了一批垂直应用。

而这些应用之间的会话、上下文、文件系统不需要刻意去打通，因为它们本来就在同一个运行时环境里，信息天然是共享的。

所以 Tutti 里有一个完整的、Agent 和人在一起协作的环境。

还有一个功能目前还没有做出来，但博客里提到了。这个工作环境可以直接分享给同事。

点一个链接，同事就能进入到我正在工作的环境里，看到所有的 Agent 会话、文件、产出，直接接着干。不用导出，不用同步，不用解释做到了哪一步。

这个应该是他们官网的 Tutti·VM 版本想做的事情。目前还没正式发布，但已经可以加入 waitlist 了。链接在这里：

https://tutti.sh/waitlist

Tutti·VM 版本想做的事，我理解大概是这样：Agent 仍然在本地运行，但 Agent 相关的会话、任务、文件和上下文，会实时在云端生产，所以没有延迟。

这样一来，我就可以随时切换电脑继续干，也可以把链接发给同事。同事打开之后，直接就能进入我刚才的工作环境里。

比如我和 Claude Code 聊到哪了，Codex 正在跑什么，哪些文件已经生成了，哪些任务还没完成，这些都能接上。

Tutti 团队在产品理念里写了一段话，我觉得说得很透彻。

过去的协作方式基本上是消息驱动的。人要把自己的想法通过工具表达出来，再把结果发给同事，同事理解之后再转化到自己的脑子里。

这个过程弯弯绕绕，大量有用的上下文在传递中丢掉了。

Agent 已经在改变我们的工作方式。接下来大部分知识类工作都会在 Agent 中进行，这个方向基本可以确定。但 Agent 之间还要像以前一样割裂吗？

Tutti 给出的答案是不应该。

让所有 Agent 天然待在同一个运行时环境里，信息不需要传递，因为大家本来就在同一个空间里，思维对彼此透明，谁需要什么信息，随时可以拿到最完整的、无损的上下文。

很久没有在 GitHub 上碰到一个项目，从产品理念上就让我这么兴奋了。
