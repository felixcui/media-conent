# 1. What is Claude Code? （什么是 Claude Code？）
原视频：


中文整理：
- Claude Code 不是“给你一段代码建议”的聊天机器人，而是一个能直接在开发环境里行动的自主编码工具。它能理解整个代码库，读取和编辑文件，运行终端命令，并接入你现有的开发工具链。它可在你的终端、Visual Studio Code、网页上的云桌面应用以及Jet Brain的集成开发环境（IDE）中使用
- 和普通的 Claude 聊天相比，最大的差别是它有“行动能力”：
  - 不只是告诉你“应该怎么改”。
  - 而是可以自己去读文件、定位问题、执行命令、修改代码、再验证结果。
- Claude Code的核心机制，是让大模型在一个循环里持续工作：
  - 先理解任务。
  - 再决定是否要调用工具。
  - 拿到工具结果后继续判断下一步。
  - 直到任务完成。
- 典型能力包括：
  - 理解和追踪代码库中的功能实现。
  - 跨文件排查 bug。
  - 构建脚本、运行测试、安装依赖。
  - 必要时搜索最新文档或外部资料。
- 但它并不是“全知全能”：
  - 仍然受上下文窗口限制。
  - 仍然可能误解意图、引入新 bug、或者过度设计。
  - 所以你仍然需要做审查和验收。

想要高效用好Claude Code，掌握以下核心概念至关重要：
1. **上下文窗口**
可将其理解为 Claude 的临时工作记忆。它能容纳海量信息，但无法一次性承载全部内容。这也是其自主执行能力的核心原理：无需把完整代码库全部载入记忆，就能通过合理策略在代码库中精准检索所需答案。
1. **默认请求许可**
Claude Code 在执行指令、修改代码文件前，都会主动向你发起确认请求。无论你是全程手动把控，还是交由工具自主运行，始终由你掌握全部操作主导权。
1. **存在失误**
和所有开发工具一样，Claude Code 并非绝对完美。它可能曲解你的实际需求、引入新的错误，或者过度设计解决方案。
这一节想传达的核心：
- 把 Claude Code 理解为“会自己动手的开发搭档”，而不是“更会写代码的聊天框”。
- 你给它目标，它会自己决定查什么、改什么、验证什么。

# 2. Installing Claude Code （安装 Claude Code）
原视频：


中文整理：
- Claude Code 可以在终端、IDE、桌面端和网页端配合使用，但本系列主要以终端形态为主。
- 教程里强调了多平台安装路径，核心思路是：
  - 本地终端优先。
  - IDE 或网页端如果要访问本地项目或本地终端能力，需要和本地环境连接。

## 终端安装与使用
如果你使用的是Mac OS、Linux或WSL，可使用此curl命令一次性完成安装。
```bash
# macOS / Linux / WSL
curl -fsSL https://claude.ai/install.sh | bash

```


如果你更喜欢使用 Homebrew，也可以使用 brew install 来安装它。但请注意，这并不具备自动更新功能。
```bash
# Homebrew
brew install --cask claude-code

```

如果你使用的是Windows PowerShell:
```powershell
# Windows PowerShell
irm https://claude.ai/install.ps1 | iex

```

如果你使用的是Windows  CMD:
```plaintext
# Windows CMD
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd

```

当然Windows用户也可以使用WinGet命令
```powershell
# WinGet
winget install Anthropic.ClaudeCode

```

- 安装完成后，输入以下命令来运行Claude Code
```bash
claude 
```

然后你将完成一些初始设置，比如选择颜色主题、使用云账户登录（可以是专业版、旗舰版或企业版账户），也可以使用 API 密钥。

## VS Code 端安装与使用
1. 打开VS Code，搜索 **Claude Code**，找到 Anthropic 官方出品插件，认准蓝色认证标识后完成安装。
1. 安装完成后建议重启 VS Code 生效。
1. 调出操作面板：Windows/Linux 按 `Ctrl+Shift+P`，Mac 按 `Command+Shift+P`，搜索 Claude Code 即可新建操作标签页。
1. 快捷入口：打开项目文件后，直接点击界面上的 Claude 图标快速唤起。
1. 自定义使用模式：可在设置文件中关闭可视化界面，仅保留纯终端操作模式。

## JetBrains 系列编辑器（IDEA 等）安装
1. 前往 JetBrains 插件市场，搜索安装 Claude Code 插件。
1. 安装完毕重启编辑器，重启后界面会显示 Claude 专属图标。
1. 点击图标调出专属面板，内置终端交互界面，可搭配云端桌面协同使用。

## 桌面客户端使用
1. 安装Claude桌面客户端并完成账号登录。
1. 页面顶部找到**Code**切换按钮即可启用 Claude Code 功能。
1. 界面风格和 Claude 对话界面相近，支持指定项目文件夹、调整操作权限，还可直接在云端开发环境中编写代码。

## 网页端使用
1. 访问网址：`https://claude.ai/code` 即可在线使用。
1. 功能逻辑和桌面客户端基本一致，**仅支持绑定 GitHub 代码仓库**进行开发。

## 各使用场景优缺点对比
1. **终端模式**：功能更新推送最快，追求最新功能首选。
1. **编辑器插件**：使用体验高度统一，能深度融入日常编码流程。
1. **桌面客户端**：可后台静默运行，方便一边处理其他事务一边调用工具。
1. **网页端**：适合远程协作 GitHub 项目，支持多任务会话并行办公。
## 
这一节的实用结论：
- 如果你想要最顺手、最完整的体验，优先在本地终端环境里安装并启动。
- 如果你只是想更快上手，先把 `claude` 命令跑起来，再去接 IDE 或网页端即可。

# 3. How Claude Code Works （Claude Code 是怎么工作的）
原视频：


中文整理：

我们都清楚，Claude Code 和普通聊天对话类应用截然不同，下面为你讲清它的核心运行逻辑，核心是一个递归的 **Agentic Loop。**
## 核心运行：**Agentic Loop**
1. **输入指令：**你向 Claude Code 输入需求提示词。
1. **搜集所需信息：**它结合大模型分析需求，调取完成任务必备相关内容，模型会返回文本内容或可执行的工具调用指令。
1. **执行实际操作：**依据指令做出对应行为，例如修改代码文件、运行终端命令等。
1. **结果核验判定：**自动校验执行结果，判断是否达成你最初提出的需求目标。
1. **闭环循环**
- 结果达标：任务结束，等待你的下一条指令
- 结果未达标：重新进入循环流程，反复执行操作、核验，直至任务完成且结果无误

**人工干预权限：**整个运行过程中，你随时可以补充信息、中断进程、引导模型思路，把控最终任务走向。

## 上下文窗口机制
1. 上下文窗口决定了它能留存、回看的内容上限，包含对话记录、代码文件内容、命令执行日志等所有信息。
1. 一旦达到存储上限，Claude Code 会**自动压缩整理会话内容**：剔除无用信息、精简总结冗余内容，主动释放上下文空间，保证程序正常运行。

## 核心支柱：工具调用能力
1. 普通 AI 助手仅支持纯文字输入输出，无中间实操能力；而工具调用是智能Agent的核心根基。
1. 借助各类内置工具，Claude Code 可自主判断时机调用功能推进任务，例如读取文件、联网检索等。
1. 它还支持语义检索，精准判断调用对应工具，并获取工具执行结果。

## 权限运行模式
默认状态下，修改文件、运行终端命令**必须提前征得你的手动确认**，按下 `Shift+Tab` 可快速切换运行模式：
1. **自动编辑模式**：自动同意文件修改操作，运行命令依旧需要手动授权
1. **规划模式**：仅使用只读类工具，先梳理完整执行方案，再正式开始执行任务
**重要提醒：**切勿随意关闭权限校验、放任其自由执行命令，一旦出现逻辑错误，极易引发难以察觉的代码故障与运行风险。
## 
Claude Code能够读取完整代码项目、自主执行实操动作、自我校验工作成果，这也是它和普通聊天 AI 最本质的区别。
这一节的核心认知升级：
- Claude Code 不是一次回答。
- 它更像一个“会思考、会调用工具、会回头验证”的执行循环。

# 4. Your first Claude Code prompt （你的第一条 Claude Code 提示词）
原视频：


中文整理：
你向 Claude Code 下达指令的方式，和使用普通 AI 助手完全一致。掌握以下用法，既能规避风险，又能大幅提升使用效率。
## 文件编辑权限切换
1. 你可自由选择两种操作模式：要么让工具**自动确认所有文件修改**，要么每次改动都**手动授权确认**。
1. 快捷键：**Shift+Tab** 快速切换两种模式。
1. 自动确认模式规则：自动同意文件编辑、新建文件操作，但执行终端命令依旧会向你申请权限。
1. 模式选择无优劣之分，按照自己的使用习惯选择即可。
## 规划模式（Plan Mode）
该模式同样在 Shift+Tab 调出的菜单内开启
1. **运行逻辑**：接收你的需求后，仅调用**只读类工具**，分析项目代码、调研可行实现方案，遇到模糊需求还会主动向你提问确认细节。
1. **输出结果**：整理出一份详尽完整的执行方案，清晰列明后续所有操作步骤。
1. **适用场景**：适合做复杂功能迭代、安全代码审查，尤其擅长拆解多步骤开发需求，是复杂开发任务的首选模式。
## 实战演示：一键实现深色模式
1. 进入项目根目录，多次按下 Shift+Tab 切换至规划模式。
1. 输入清晰需求指令：为整个项目全局添加深色模式，在页面顶部导航栏增设明暗模式切换按钮；结合现有浅色主题配色，搭配适配度高、观感舒适的对比色系，完成整体开发规划。
1. 核对规划方案无误后，即可按照方案正式执行开发。
1. 开发完成后，可清晰查看全部操作流程与决策思路，最终顺利完成深色模式开发。
## 高效使用核心建议
1. 提示词尽量**描述详尽、需求明确**，减少理解偏差。
1. 想要全程把控开发流程、跟进每一步操作，完全可以自主把控进度。
1. 正式动手开发前，优先启用规划模式，让工具吃透你的全部需求、敲定细节方案，再落地执行，稳妥又高效。

# 5. The CLAUDE.md file （CLAUDE.md 文件）
原视频：


中文整理：
## `.claude.md` 文件作用
它是 Claude Code 最实用的功能之一，相当于给项目搭建**长期专属记忆**。
1. 项目内没有该文件时，每次启动 Claude Code 都要从零开始：重新遍历代码库、梳理项目依赖、梳理已完成功能，还容易自行主观猜测项目规范，很难精准对齐你的开发思路。
1. `.claude.md` 是放在**项目根目录**的 Markdown 文件，每次开启会话都会被自动读取，等同于项目专属的开发指引手册。
1. 文件内的所有内容，会自动追加到你的每一条提示词中，潜移默化统一开发规范。
1. 快捷生成指令：输入 `/init`，即可让 Claude 自动根据当前项目结构生成对应的 `.claude.md` 文件。
## 文件内容示例（实战规范）
以 Next.js 15 项目为例，可写入这类统一规则：
- 技术栈：采用 App Router、Tailwind CSS、Drizzle ORM
- 运行规范：本地启动开发服务、执行测试、代码格式化
- 代码风格：缩进使用 2 个空格，优先使用具名导出
- 目录规范：所有接口路由统一放在 `app/api` 目录
- 开发原则：优先使用服务端动作，尽量少用传统接口路由
写入后，你让它新建 React 组件、编写业务代码时，会自动沿用这套规范，无需反复重复叮嘱。
## 记忆文件层级划分
### 项目级 `.claude.md`
存放于项目根目录，跟随项目版本管理同步提交，**团队全员共用**，统一团队项目开发标准。
### 用户级 `.claude.md`
存放在个人配置文件夹中，仅对本人生效，**全局所有项目通用**。可填写个人编码习惯、注释风格、固定开发偏好等私人设定。
## 实用使用技巧
1. **固定开发习惯**：若你需要强制统一写法（如优先用服务端动作），可直接让 Claude 将该规则存入记忆，后续打开项目自动生效。
1. **引用项目文档**：想让它读取项目内指定文档作为参考，直接用 `@+文件路径` 即可快速引用。
1. **精简编写原则**：新项目初期建议先不创建该文件，在使用过程中记录需要反复纠正的开发要求，最后再整理写入，保证文件简洁精炼，只留存必要规则。
## 总结
Claude Code 使用体验好坏的核心在于**上下文信息 + **`**.claude.md**`** 记忆文件**。编写时先写明项目技术栈、个人 / 团队编码偏好、常用运行命令，再逐步补充细则，就能大幅提升开发效率，告别反复纠正指令的麻烦。

# 6. Explore → Plan → Code → Commit 工作流
原视频：


中文整理：
用好 Claude Code，牢记这套标准流程：**调研探索 → 制定方案 → 编写代码 → 提交代码。**多数人直接让 AI 上手写代码，后续要反复修正调整，效率大打折扣。
## 规划方案
1. 规划模式下**仅只读文件，不做修改**，只会读取项目代码、梳理实现思路
1. 开启方式：按下`Shift+Tab`，输入框上方显示规划模式即成功
1. 使用示例需求：在图片上传流程中新增格式转换功能可让 AI 自主定位流程节点、判断是否需要安装新依赖、梳理完整实现思路
1. 流程优势：AI 会读取相关代码、查阅相关资料，输出完整执行方案；你先行审核，不满意可直接要求增补、修改内容。**此阶段是最佳修正时机，全程还未编写任何代码**

## 编写代码
1. 方案确认无误后，点击同意执行，AI 自动按清单逐项落地开发
1. 可提前设置文件修改权限：自动确认编辑 或 每次修改手动确认
1. AI 会主动排查代码问题，过程中依旧可随时调整思路、纠正偏差
1. 规划模式留存完整思考逻辑，后续开发决策更贴合项目需求

## 提升方案准确率实用技巧
1. 需求标准写明确，让 AI 清晰判定合格交付标准
1. 配齐辅助工具减少反复沟通
  - 开发网页界面时安装配套浏览器插件，支持 AI 操控页面实测 UI 效果
  - 项目内置完备测试用例，可让 AI 自测校验，也可让 AI 自主编写测试代码
  - 统一测试判定标准，避免出现无效校验结果
1. 高频问题固化方案遇到反复出现的同类问题，直接让 AI 把解决办法存入`.claude.md`记忆文件，永久复用

## 提交代码前
1. 人工自测验证功能无误
1. 调用代码审查工具，完成代码自查评审
1. 让 AI 按照你的习惯风格自动生成规范提交备注
1. 提交推送代码，开启下一项功能开发

## 工作流完整价值总结
- 探索调研：补齐项目所需全部上下文信息
- 方案规划：定下执行标准与验收依据
- 代码编写：人机协同打磨，敲定最终实现效果
- 提交入库：完成代码审核与版本推送，无缝衔接下一开发任务

# 7. Context Management in Claude Code （Claude Code 的上下文管理）
原视频：


中文整理：
## 什么是上下文
上下文就是 Claude Code 的**运行内存**。你读取的文件、执行的命令、发送的对话消息，都会占用上下文窗口空间。可以把它理解为模型的记忆存储空间，所有交互行为都会不断往里累加内容，而存储空间存在上限，做好优化十分关键。

## 自动压缩机制
当内容快要达到上下文上限时，系统会**自动执行压缩整理**：提炼留存关键信息，剔除无用的工具调用日志与冗余结果，释放大量内存空间。⚠️ 注意：自动压缩可能会丢失部分过往对话细节。

## 手动上下文管理指令
- 这节课给出的三个关键命令非常重要：
```bash
/compact
/clear
/context

```

1. `**/compact**`** 手动压缩**
压缩当前会话所有历史内容，精简冗余信息，保留核心工作记录，适合想腾出空间、又不想清空过往记录时使用。
1. `**/clear**`** 清空上下文**
彻底清除全部会话记忆，从零全新开始，完全摒弃之前所有操作记录。
1. `**/context**`** 查看上下文状态**
直观查看当前内存占用大小、各类内容占比，还有可视化数据图表，清晰掌握资源使用情况。

## 使用场景取舍
1. **开发同一功能、临近超限**：执行压缩，保留当前业务相关上下文，保证开发连贯
1. **完成旧功能、开启新需求**：直接清空上下文，避免旧内容造成思路干扰、判断偏差
1. **长期通用固定规则**：写入`.claude.md`文件，脱离临时上下文永久生效，不用重复告知

## 优化上下文实用技巧
1. **提示词务必写详尽明确**简略模糊的指令看似简短，反而会让模型主动翻阅大量代码、自主推演思考，额外消耗更多上下文资源；一两句话把需求说清楚，反而更省空间。
1. **关停无关 MCP 服务**MCP 服务器会默认把所有可用工具载入上下文，项目用不到的 MCP 服务建议直接关闭。
1. **优先使用 Skill 工具**功能和 MCP 类似，但不会将全部内容载入上下文，有效节省内存。
1. **巧用子代理分担任务**子代理和主代理相互独立，拥有专属独立上下文窗口，可并行运行。查询类简单任务（如查找认证接口地址等）交给子代理处理，仅把最终结果汇总给主代理，大幅减轻主上下文负担。
## 核心总结
高效用好 Claude Code 的关键就是管好上下文：长会话精简用`/compact`，换新需求重置用`/clear`；需求描述精准清晰，定期查看占用状态；非核心查询任务交给子代理分流，最大化利用内存空间。

## MCP 基础定义
**MCP（Model Context Protocol 模型上下文协议）**
是一套开放通用标准，作用是让 Claude Code 能够无缝对接各类外部工具与数据源。
你提出需求时，它可自动判断调用对应外部工具，精准理解并完成任务。

## MCP 核心价值
使用 Claude Code 开发时，大量业务相关数据并不在本地代码内，而是存放在数据库、办公协作软件、开源仓库等外部平台，MCP 就是打通这些外部资源的核心桥梁。
同时**工具调用**是AI Agent的核心能力，区别于普通AI仅纯文字问答，Claude Code 可借助各类工具执行实际操作，落地完整任务。

## 实际应用场景
1. 团队使用 Linear 项目管理工具：接入对应 Linear MCP 服务，可直接同步同步项目工单、任务详情
2. 查阅依赖库最新文档：接入文档类 MCP 服务，快速获取官方最新技术资料
3. 更多第三方连接器可在官网 https://claude.com/connectors查找

## MCP 服务添加与分类
1. 添加命令：使用 `claude mcp add` 快速接入MCP服务
2. 两大服务类型
- **HTTP 服务端**：对接云端远程第三方服务，通过网络实现连接
- **Stdio 本地服务端**：运行在你本机电脑的本地进程服务
3. 会话内管理：在 Claude Code 中输入 `/mcp`，可查看已连接服务、运行状态，一键停用闲置服务

## MCP 三种作用域
1. **本地作用域**：仅在当前项目生效
2. **用户全局作用域**：绑定个人账号，所有项目均可通用
3. **项目级作用域**：通过 `project.mcp.json` 文件配置并提交版本库，团队所有成员拉取项目后自动同步配置，统一开发环境

## 上下文优化避坑要点
1. 弊端：即便暂时不用，已配置的MCP服务也会把工具定义信息占用上下文空间，配置过多会挤占可用内存
2. 优化方案
- 及时停用长期闲置的MCP服务，减少资源占用
- 有对应CLI命令行工具（GitHub、AWS等）优先用CLI，比MCP更节省上下文
- 优先使用Skill能力：仅将名称与简介载入上下文，需要用时再完整加载，轻量化更省空间
3. 自动适配机制
当MCP工具占用上下文超过10%，Claude Code会自动切换为**工具检索模式**，按需查找调用工具，但该模式执行稳定性会有所下降

## 核心总结
1. MCP实现Claude Code与外部工具、业务数据源互联互通
2. 借助命令快速添加服务，通过配置文件实现团队统一同步
3. 日常维护及时清理无用服务，严控上下文占用
4. 合理搭配CLI工具与Skill能力，平衡功能实用性与运行效率

# 9. Hooks in Claude Code （Claude Code 里的 Hooks）
原视频：


中文整理：
- Prompt、CLAUDE.md、skill 都是在“尽量引导 Claude”。
- Hook 则是在“强制某件事按规则执行”。
## Hooks是什么
hooks可以在 Claude Code 整个运行生命周期的**指定节点自动执行命令**。
它和其他功能最大区别：**执行结果确定、必定触发**。

举例：
你写在 `claude.md` 里让它改完文件自动格式化，偶尔会失效；
但写成hooks规则，**每次都会强制执行，毫无例外**。

## 常用实战场景
1. 文件修改完成后自动格式化代码
2. 记录所有执行命令，满足合规审计需求
3. 拦截高危操作，禁止修改生产环境配置文件
4. 任务执行完毕自动推送消息通知

## 配置入口与基础写法
所有hooks统一在 `**.claude/settings.json**` 中配置
配置三要素：
1. 选定触发事件
2. 可选设置匹配规则，限定生效工具
3. 填写需要自动运行的终端命令

## 五大核心触发时机
1. **用户提交提示词前**：提交指令后、模型开始处理前触发
2. **工具调用前置（pre tool use）**：调用任何工具**之前**执行
3. **工具调用后置（post tool use）**：工具执行**完成后**触发
4. **消息通知触发**：Claude 推送通知时运行
5. **会话结束触发**：模型完成全部应答后执行

## 高频用法讲解
### 1. 改文件自动格式化（最常用）
选用工具调用后置hooks
匹配规则设为：单文件编辑、批量编辑。只要 Claude 修改代码文件就自动触发，可根据文件后缀执行对应格式化命令

### 2. 拦截高危操作（强约束管控）
前置hooks可直接阻止工具执行
hooks通过标准输入获取工具名称与入参JSON
通过**退出码**控制行为：
- 退出码 `0`：放行，正常执行操作
- 退出码 `2`：直接阻断操作，错误信息同步返回给 Claude
模型会明确知晓拦截原因并自行调整行为

适用管控场景：
- 禁止写入生产配置目录
- 拦截含 `rm -rf` 等高风险删除命令
- 禁止直接向主干分支提交代码
可强制落地团队硬性规范，而非仅口头提醒。

## 作用域
1. 该配置属于**项目级配置**，可提交到代码仓库
2. 团队所有成员拉取项目后，自动同步全套hooks规则
3. 搭配项目环境变量，可直接调用项目内脚本，不受当前工作目录影响

## 使用总结
1. **后置hooks**：用来做自动格式化、操作日志记录
2. **前置hooks**：用来拦截危险行为、强制校验规范
3. 统一在项目配置文件编写并入库，全员统一生效
4. 凡是**必须次次执行、绝对不能遗漏**的操作，不要写在提示词里，直接配置hooks实现强制管控

# 一页速记
- 先探索，再规划，再写代码，不要一上来就让 Claude 开改。
- 大任务先进 plan mode。
- 项目长期规则写进 `CLAUDE.md`。
- 同功能继续做，用 `/compact`；切换新任务，用 `/clear`。
