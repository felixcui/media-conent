# 为了攒一支Agent团队，我把工作流搬进了Moxt

**作者**: AI沃茨

**来源**: https://mp.weixin.qq.com/s/e_jSzwxbACRVJab3acno_A

---

## 摘要

本文介绍了作者使用Moxt平台组建多角色AI Agent团队，将选题调研、品牌设计、产品测评等日常工作流自动化的实践经验。核心亮点在于：一是在Moxt中可创建分工明确的专职AI同事并为其安装定制技能，使AI能产出具有思考深度的成果，例如设计建议基于数据反馈而非主观判断；二是设置"唱反调"角色进行交叉审核，有效弥补单一AI输出的盲区；三是通过飞书集成实现团队共享AI同事，降低协作门槛。

## 作为一个AI人，

我每天都在想着把所有工作流Agent化，要是把我每天要干的拆开来看，就是选题，调研，写稿，测产品，做设计，整理素材。

这个过程中需要我去判断和表达的时间其实不到一半，剩下的全是重复劳动，搜资料，找参考图，对比竞品参数，把散落各处的素材整理成文档。

最近在用一个Moxt的新平台养我的AI同事们，看看他们能不能学会我的工作流。

![Image](https://mmbiz.qpic.cn/mmbiz_png/VNz1x8bH8FwwkWx4vtGqQQSs2wvOMoGc5938Gv040hrclgCewicOj577jWxX8xJMfeSJ6lklrNIxad9Cx3mSX53yJUuHNeq1SkxlGyu4qXnk/640?wx_fmt=png&from=appmsg)

打个比方，我想给自己的账号做一套品牌视觉升级，就可以对Moxt里的momo说，

👍

我是自媒体卡尔的AI沃茨，帮我做个品牌视觉调研，搜一下同类AI自媒体的视觉风格，当下流行的配色趋势和目标读者的审美偏好。

在没有喂给任何我的资料的情况下，momo回答匹配度还挺高。调研了海外和国内的AI自媒体视觉风格，几大主流平台用户喜好，细节我就不一一说了，直接给你们看看结论，

我蛮赞同蓝紫配色已产生严重设计疲劳了。

于是，我决定让它正经帮我做一个新的品牌视觉。

但光靠它一个不够，我又建了四个专职AI同事：把控整体风格方向的设计总监，做调研的，出方案的视觉设计师，还有喜欢唱反调的。

也可以在市场里挑选适合自己项目的同事。目前有10个分类，涵盖了这些领域的主要工种，非常有一人公司OPC的感觉。

每个AI同事还可以另外培训，用人话说就是从moxt里学新skills。

也可以把我在GitHub看到的好用技能发进去。比放，我给设计师小C安装一个设计类的skills，VoltAgent/awesome-design-md。还要求小C每次出设计方案之前对标这个skills里的5个品牌。

然后我们来试试看，

我把我的小红书主页链接丢进去，生成了一份新的品牌视觉调研报告。然后在小C的对话框里，直接调用这份报告，让它设计一整套视觉。

然后它给了我一个MD文档，对标了fiigma、pinterest等5个品牌，确定了品牌的定位和调性，色彩系统，字体系统以及IP角色系统，还考虑了品牌图形元素，平台尺寸规范，包括一个品牌条和水印系统。

这里我比较惊喜的是，能从这套视觉系统里看到设计师的思考。比如，在色彩系统里，它建议我用琥珀橙做主色，而没有选择我的IP紫色。

配色逻辑里解释了这一点，

一是因为紫色和AI行业的蓝紫色陷阱重合，

二来它通过我给的小红书主页链接，发现我封面数据中表现最好的是琥珀橙色调。

虽然但是，作为一个全AI背景的设计师，我还是很担心这个文档里有我看不出的bug。

于是，我打开了和小X的对话框。这是我前面设置的唱反调的同事。因为在同一个团队，对话框里是可以直接@刚刚小C做的方案，问小X觉得这个方案怎么样。

小X先是肯定了方案的框架，说品牌定位也做得不错，然后kuku一点不给面子指出了8个问题。

光这第一条，我还真没看出来。

然后他还指出了，设计方案里说每张封面必须出现卡尔这条可以松一松，因为一些严肃的内容放卡通角色会削弱内容的分量感。

品牌设计做完，

我顺手把另一个高频工作也搬进了Moxt。

我每周都要测评AI产品，查官网，扒功能列表，对比竞品，整理截图。这套流程重复了无数次。

我在Moxt里建了一个Workspace，叫AI产品测评工作台，里面放了两个空文档，产品信息模板和测评记录。

我是不会来填这个表的，也太麻烦了。于是我创建了一个agent，它也叫卡尔。我通过对话一点点告诉他，我每天要做什么，工作流程是什么。

多轮对话之后，它和我确认了怎么做这件事。然后，我让它把刚刚的结果写入到测评信息模板里。

这是它写好的文档，我一个字没有编辑。

还没完，然后我又@小X，让它看看这个模板怎么样。它又提了四、五个建议，说没有体验感受，没有价格定位等等。然后我就说那你重新做一版。

|  |  |
| --- | --- |

这把只能说，所有不会写SOP的人有福了。。

在正式使用中也并不需要自己去填这个表。把测评的过程通过对话框发给AI同事，让他们帮你更新就行了。

前面几个案例都是在Moxt里面操作的。

但我日常沟通主要在飞书，迁移过去也比较麻烦。我就在想，能不能把AI同事都拉过飞书？

Moxt还支持飞书集成，而且我可以把其他小伙伴拉到群里，和我一起共享我配置好的AI同事，这比封装个skills更好用。

有一说一，集成飞书这功能属实也是有点绕。

我是把自己创建的AI同事，小卡、小X接过去，要从AI Teammates里点击你想同步的agent进入工作区，然后点击侧边栏的设置，在overview里点击连接飞书。

然后就按步骤给到飞书应用的app\_id 和app secret 就行。整个过程还是挺快的。迁移过去之后，在Moxt使用的记忆也还留着。

Moxt也是支持定时任务的。

我测试了一个搜索场景，

这是momo给我的结果，

搜索准确度我对比了下，还是挺高的，而且直接按照涨星数排好了，这个功能很适合团队每天定时做选题共享。

从单个openclaw到多Agent多群聊，

再到把Agent当同事养，

我已经习惯了早上起来先一口气排一大堆任务给AI同事们，他们自己去忙，去找合适的Skill，然后再回来交作业，我只管拍板。

手底下有人就是好啊，

还能24小时回我消息，

怎么有种我成了P10的感觉咧。
