# 从 0-1 跑通 AI 产品出海，没那么难

**作者**: 苍何

**来源**: https://mp.weixin.qq.com/s/6OPd0U0RQxD5ScDKijsv6g

---

## 摘要

本文作者分享了从0到1打造AI出海产品的实践经验。为了提升用户体验，作者为开源项目的海外网站增加了基于GPT image 2的在线生图测试功能。为解决API消耗成本问题，网站引入了会员与积分付费体系。此外，项目还打通了包含收藏夹功能的完整用户账户系统，并接入了GA4管理员数据分析面板。作者通过这一系列完善功能与商业闭环的设计，成功展示了跑通AI产品出海的完整流程，证明其并不困难。

最近在学产品思维，里面有个观点，很重要：

**「用户不会记住你做了什么功能，但会记住你帮他省了多少时间。」**

当用户进入这个网站，实际上他找到提示词后，看到好看的爆款图片，他会复制这个提示词到其他地方来测试。

那为什么我不加一个在线测试的能力？

![](https://mmbiz.qpic.cn/sz_mmbiz_png/zw8bZHsVSaBlmJbC3gP4icrCrS6JMwfkd3jTSqIzbiagLRqS0X3EQ25TnxmAq46nNqgHuvHHSyXV2X6cRuicN50ufHPHpRQXUMrO8UyHt9EkQo/640?from=appmsg)

这又涉及一个问题，在线测试需要消耗 token，虽然词元 API 给了项目一点 GPT image 2 的赞助，但这个额度是有限的，我无法完全免费提供。

于是我想着完全可以学习会员积分体系，也正好能打通完整的一个 AI 出海产品。

于是我花了一些时间，加了会员积分系统。用户拥有一次免费的测试额度，之后可以通过购买会员或者积分的形式来继续在线测试生成图片。

![](https://mmbiz.qpic.cn/mmbiz_png/zw8bZHsVSaAicibua92FUDmobDAltXTu4NZicR3I6ThNV1dEtp9MZiclOWSbKEEl8LMNF7OvibknWgTlHHSYic0hyGqEqFyWCmaMKS4zyzNyicUpia8/640?from=appmsg)

生成测试前，提示词是可以自由更改的。当点击「生成图片」，它就会调用 GPT image 2 的 API 来生图。

生成好后，能对比效果，还是很方便的。

![](https://mmbiz.qpic.cn/mmbiz_png/zw8bZHsVSaBQOZ76mvDbT9ia7ia6r5wfpII7cesAdOUIrhwwAKOxpELLD8xwjn3FPFUPcSHL7ibxs4mjsAibvouK9AKDgepJic3Vib4vlU73XIZMY/640?from=appmsg)

另外我还加了收藏功能，当你点击收藏的时候，他是能收藏进你的收藏夹的，因为整个用户体系也已经打通了。

![](https://mmbiz.qpic.cn/mmbiz_png/zw8bZHsVSaBrL3wp0MibeYX9eCDku5PkJF6icEQrVhyDdCQ5m4shV1jKerQbU1mXBCT6veM2flKGwYav5SAiaX9ONulGcTibbYiagVkZVyoPc7WE/640?from=appmsg)

为了完成生图测试能力，用户体系、会员与积分体系、支付体系等都需要全部拉通，是一个复杂的活，要放在以前，没个把星期搞不定。

出海谷歌登录、数据存储、 Stripe 支付都没那么简单。

现在用 AI 只需要个把小时就能开发好。

而且，已经有付费用户了。

![](https://mmbiz.qpic.cn/mmbiz_png/zw8bZHsVSaCsnaB9jCorSpIwIUgAxiaUTvC6icrYtia7KQ6qZZ5lTUhlryicWogQbtTicXKG9zETFqmJvugCNHEabialuSQS0eCDib2gcZ3VxQHEOw/640?from=appmsg)

这几天去杭州参加观猹的「谁能成为百万博主」的活动，朋友刘聪给我分享了他评测的最新 **「Qwen 3.7-Max」** 模型，说有点牛的，建议我测测。

刚好，阿里云的朋友也在，立马刷了下脸，要到了开白内测。

以上在 GPT-Image 2 Gallery上新增的功能全部都是通过阿里最新的 Qwen 3.7-Max 模型完成的。

我用 CC Switch 配置上 Qwen 3.7-Max 内测模型，直接用 Claude Code 开干。

![](https://mmbiz.qpic.cn/mmbiz_png/zw8bZHsVSaA4HIqnOCXAXbB1AaJU7gKaL1AWC1iaFOXric1Siaa6lFPGy1habaK4ibMS0icYxaB1Lsy9z3jvM1sZTFMYoVLWQ5tkHsKGeHfWCVfM/640?wx_fmt=png&from=appmsg)

这是 Qwen 3.7-Max 帮我生成的技术架构路线，Vercel 用来托管并保存环境变量信息，Supabase 用来用户邓丽，Postgree 数据库存储，记录积分、生成记录。

在 gpt-image 2 这个项目中，实际上是一个比较复杂的多文件工程，里面除了提示词管理、Skill抽象以及模板分离外，还有出海独立站的承载。

但我发现，Qwen 3.7-Max 驾驭的还是比较稳的，特别是 Agent 长时任务上能够自主执行。

今天，我也看到了阿里已经做了官宣，Qwen 3.7-Max 在多项专业模型 Benchmark （基准测试）中均取得了领先的表现。

我发现 Qwen 3.7 在连续多步的 agent 调用中保持了较为稳定的发挥。不会丢失上下文。

当我继续下达指令，要进行用户模块的开发：

> 现在开始帮加用户模块吧，我希望可以邮箱登录，也可以直接选择谷歌登录，然后把我的用户设置为超级管理员，用户体系是方便后面我要做积分什么的

它会自行实现用户登录的前端界面，并实现用户认证，配置 Supabase 数据库和超级管理员。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/zw8bZHsVSaC4UYPj7veumM1GUQJdKwlKWmZdSAnyicmXIG7cWcfuNmBhGlvpyRZCfEOVcE4u81auOicwDxia9hTz2XnEWursZWtXxEc9cRveTg/640?from=appmsg)

这里我选择先用谷歌邮箱登录，相对流程简单一些。

第一次做的时候感觉还是没理解好需求，一顿操作，给我还是整登录注册传统流程去了，我重新输入了提示词：

> 我需要谷歌登录，其他的注册体系都不用先，记住登录的流程和按钮你可以学习别人的出海站点是怎么做的。

再次出来的效果就会满足我的要求了，他会自行学习后给我登录 UI。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/zw8bZHsVSaBia1qtUWgudXQPFEVJmgHckNCKJhmOmqVdIE1FRB2MzmXzoSte0k8hZ0ibBYShd6nQSYISspBmpI57UPCoNGsQRTW26RBsJY3Vk/640?from=appmsg)

![](https://mmbiz.qpic.cn/sz_mmbiz_png/zw8bZHsVSaAhHQpGAQSyiaNK58hhtGqKEWFOgQDQib8cE4ic4sMYrticC0aCuDqExnrmbO6vYbibkgDwiaH2kAJAbqL8fOkp2Hd6icwcDxdpGpHYe0/640?from=appmsg)

这样，整个流程就完全都打通了，一个轻量的出海应用就搞定了，最麻烦的还是支付链路这一块。

Stripe 的接入坑确实不少，但 Qwen 3.7-Max 在这种多步骤、长链路的任务中表现很稳，不会中途丢上下文，也不会突然开始胡说八道。

从需求分析、架构设计、代码实现到调试部署，整个过程就像带了个靠谱的全栈搭子。

说实话，以前搞出海，光是谷歌登录 + Stripe 支付 + 用户体系这一套，没个一两周根本下不来。现在一个下午就能跑通，体感上效率提升了 10 倍不止。

而且我看了下官方数据，Qwen 3.7-Max 在编程智能体测评 SWE-Pro、Terminal Bench 2.0 这些榜单上已经超过了 DeepSeek-v4-pro-Max 和 Claude Opus 4.6，国产模型里排第一。

更离谱的是，官方测试中它能独立执行 35 小时的超长程任务，自主完成 1158 次工具调用，最终在一个全新芯片平台上把推理速度优化了 10 倍。

对我们普通开发者来说，最直观的感受就是：以前觉得出海门槛高，现在门槛被 AI 踩平了。

你只需要有一个想法，剩下的交给 AI 去执行就好。

当然，前提是你得知道自己要什么，产品思维比技术实现更重要。

AI 不会替代有想法的人，但会淘汰没想法还不会用工具的人。

如果你也想体验 Qwen 3.7-Max 的编程能力，可以去阿里云百炼上试试，模型 API 即将上线。
