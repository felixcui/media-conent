# 用不上GPT-5.6？可以试试Hermes刚上线的MoA混合Agent模式！

**作者**: DracoVibeCoding

**来源**: https://mp.weixin.qq.com/s/pbHlRqN_w1RLXnC_IgC8Ag

---

## 摘要

针对大众无法使用GPT-5.6的现状，Hermes上线了MoA（混合Agent）模式。该模式遵循“群策群力”的思路，用户在配置文件中设置最多8个已授权的参考模型生成方案，并由一个最强的已授权模型作为“判官”进行聚合评判，从而在不依赖顶级未开放模型的情况下提升Agent的综合能力，配置完成后可通过/moa命令快速切换使用。

不过，几小时前，Hermes上线了MoA（Mixture of Agents）混合模型模式：

有趣的是，Hermes团队的宣传话术也是挺酸葡萄的：既然最强大的模型用不上，那不如换个思路来提高Agent的能力！

![image](https://mmbiz.qpic.cn/sz_mmbiz_png/0m9F5vC1OGgncwH81iakUibfWCLG8jGicjxvAERYfk0kS4bUf4ykMqjSAWvjz4EXjnUW4w2ibvDEmHzk4hTQvOut83EVI9cp38R2hyiadBKLKS48/640?from=appmsg)

这似乎是最近的一个小趋势，既然GPT-5.6和Fable5绝大多数人都用不上，那是不是可以让多个大模型/Agent群策群力来实现“三个臭皮匠顶个诸葛亮”的效果呢？

本周有个日本团队发布的方案Sakana Fugu就是这个思路，用一个小模型来调度其他虽然不及Fable5但能力还不错的几组大模型，然后得到最佳方案作为输出，号称可以击败Fable5。

![image](https://mmbiz.qpic.cn/mmbiz_png/0m9F5vC1OGiaE1EAwG7iaicibbICV27kDYzZI62jVE3HnaZ8M8OXNVfibGWHVVa6mriaFFnltxFyGqiaUSAP0m27xkW8P69ACGjImXxYSHETkbWCxE/640?from=appmsg)

Hermes的MoA显然也是这个思路，只不过MoA的“判官”不是一个新的小模型，而是直接采用你已经在Hermes中成功授权的任意模型。

配置Hermes MoA模式很简单，在你的Hermes的config.yaml文件中按照如下格式写入（需要升级到Hermes最新版本）：

![image](https://mmbiz.qpic.cn/mmbiz_png/0m9F5vC1OGh9xxjwLx43SXMCgzdvibHA1fictAqXicvLAcibmxDH0TaZ51qOuZYhYVbribWprBHm50BWjMJlfUk7fXF9Ah9hOXwvOleJxqplLty4/640?from=appmsg)

```
moa:
default_preset: default
active_preset: ''
presets:
    default:
      reference_models:
        - provider: minimax-cn

          model: MiniMax-M3
        - provider: kimi-coding

          model: kimi-k2.7-code
      aggregator:
        provider: openai-codex
        model: gpt-5.5
      reference_temperature: 0.6
      aggregator_temperature: 0.4
      max_tokens: 4096
      enabled: true
```

> 注意：
> 
> •
> 
> 模型名称不要照抄我的，应该是你已经完成授权/登录的模型
> 
> •
> 
> 比如我的Hermes曾经通过OAuth或者输入API的方式完成了OpenAI-Codex（GPT-5.5）、KIMI K2.7 Code和MiniMax M3的授权/登录，我将最强的模型GPT-5.5作为Aggregator（判官），kimi-k2.7-code和minimax-m3作为Reference（臭皮匠）
> 
> •
> 
> Reference（臭皮匠）模型最多可以平行设置8个！

配置好之后，你可以在Hermes的各种交互界面中通过 /moa 命令来切换到MoA模式：

```
/moa
```
