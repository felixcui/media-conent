# 关于 Clawdbot，别被吹晕：3 分钟搞懂它到底能不能替你干活

> 来源: [绿蚁红泥天欲雪](https://mp.weixin.qq.com/s/5Q84HK9tL8fzSEC0iRgkCg)
> 时间: 2026-01-28

---

我们都想要一个“只要一句话就自动干活的 Jarvis”，而不是单调的对话框。Clawdbot 的出现，正好引爆了这个情绪。

Clawdbot 的思路很粗暴：不用去找 AI，让 AI 住进你每天都在刷的消息列表里。你在 WhatsApp/Telegram/Slack/Discord 里把它当联系人发一句话，它就会自动的**去跑工具、查文件、执行流程，然后把结果发回来**。

官网和仓库地址：https://clawd.bot/，https://github.com/clawdbot/clawdbot。

## Clawdbot 到底是什么

**你自己跑在本机/服务器上的个人 AI 助手**，开源，自己部署自己管控。你承担的是模型调用成本，不是“买会员”的平台税。

它的最大的特点就是接入聊天软件了：

- **用聊天软件当 UI**（你已经在用的入口）

- **用 Gateway 当中枢**（消息收发、连接各个平台、管理会话）

- **用 Agent 当大脑**（把你的话喂给 LLM，并决定要不要调用工具）

所以它看起来像“又一个 bot”，但本质更像：**一个有聊天入口的自动化控制台**。

Clawdbot 的差异不是“更会自主”，而是“把自主这件事做成了可部署、可扩展、可控权的个人运行时”。

说白了就是：Clawdbot 是围绕 AI Agent 打造了一个小型的运行系统，让我们可自己部署、管控，从而满足我们的“指挥欲”个人自动化。

他的主要工作在“非智能化”的工程实现上，即 AI Agent 的脚手架扩展上。

但是火归火，奈何它支持的通讯软件在我们这里都是“杂牌军”，微信、企微、飞书、钉钉这些这些正规军，它都不支持。所以国内真想提高生产力，还要想换场景。当前的演示全是国外高频使用的软件生态，毕竟像邮件这种国外命根子的东西，在我们这里重要性要弱的多。

## 它为什么火

这波热度并不神秘：大家追的不是“更会聊”，而是“终于能替我把脏活干掉”。

Clawdbot 把过去几件分散的事绑到一起了：

1. **多渠道统一入口**：WhatsApp/Telegram/Discord/Slack/Signal/Google Chat/iMessage/Teams…（以及一些扩展渠道）——你用哪个，它就在哪。

2. **工具是“第一等公民”**：内置 Web 搜索/抓取、Browser 自动化、Exec 跑命令等，不是只能“建议你怎么做”，而是能真的去做。

3. **安全机制是正经做的**：配对（pairing）、沙盒（sandbox）、审批（approvals）、工作流关卡（Lobster）——它默认就假设“你会被坑”，所以先上护栏。

## 它怎么干活

![Clawdbot2.png](https://mmbiz.qpic.cn/sz_mmbiz_png/fOBv47QeGVF40bAPaWahzBgvOATN3utiaMJiaKuBnd2Dic7xQeLLzGvIOgxibHsc3gFDR462vRUjKZibQRibJBFH4cnA/640?wx_fmt=png&from=appmsg&watermark=1#imgIndex=0)你可以把它想成一家小公司：

- **Gateway = 前台 + 交换机**：负责把各平台的消息接进来、再把结果发回去；同时承载配对与一些安全策略。

- **Agent = 负责人**：拿到消息后，组装系统提示词、上下文、可用工具清单，去调用你配置的模型。

- **Tools/Skills = 员工技能**：

- Tools（内置一等工具）：web_search/web_fetch、browser、exec…

- Skills（可扩展能力）：会被压缩成一份清单塞进 prompt，模型“知道自己能用什么”。

**Memory = 会议纪要 + 知识库**：把关键上下文沉淀到你控制的工作区里，供后续检索使用。

关键点：**它不是“一个更强的模型”。它是把模型、工具、渠道、记忆组织成了一个可运行的系统。**

## 持久记忆

都说可以全天候运行的个人助理了，那么这个个人怎么来？当然是记忆，也就是数据存储。

Clawdbot 的实现是：**把工作区当成“助理档案柜”，**配置示例里直接把工作区设为 `~/clawd`，你可以清晰知道数据落在哪。

它的优势不是“永不遗忘”，而是三件事：

- **可检索**：需要旧信息时，先在本地记忆里找，再喂给模型。

- **可编辑/可回滚**：你能像改笔记一样改它的“记忆”，甚至用 Git 管（这点对高阶用户很香）。

- **可控增长**：会有会话管理与抽取摘要的需求，否则日志只会无限膨胀——这不是缺点，是现实。

所以有着本地化的记忆系统来实时记录你的数据：文件，聊天，历史记录等，从而让数据可以积累，并且个人管理**。**

**一想到这，我估计大部分人和我一样害怕：要是 Claude 或 ChatGPT 的号被封了，那数据可全没了。**

AI的删库不跑路的事相信大家都见过，安全起见，大家试玩时最好做好数据备份。我也大致按风险分了三类，可以做尝鲜时的参考：

### 1）信息型（低风险）

查资料、总结、写草稿、生成清单。**这类任务几乎没副作用，适合拿来热身。

### 2）半自动（中风险）

比如：读网页/抓取信息、整理邮箱草稿、生成日程建议，但**最后一步让你确认**。
Browser 工具的官方建议也很直白：登录类操作尽量手动登录，不要把账号密码交给模型。

### 3）全自动（高风险）

跑命令、改文件、发消息、真正“动你的系统”。
Exec 能跑 shell，沙盒与审批能把风险压住，但前提是你真的去配置它。

**它能做到“像人一样操作电脑”，但你必须把它当成“会犯错的实习生 + 有权限的运维脚本”。**

## 最大风险

Clawdbot 官方安全文档的核心态度很明确：
把链接、附件、复制来的指令都当成不可信输入；高风险工具（exec/browser/web_fetch/web_search）要做 allowlist、沙盒隔离、分 agent 权限。

它提供的护栏也很硬核：

- **Pairing**：陌生人先拿配对码，没批准就不处理消息（WhatsApp/Discord 等渠道都支持类似策略）。

- **Sandbox**：把工具执行关进 Docker 沙盒，必要时只给只读目录。

- **Approvals**：命令需要你点头才真正落地。

- **Lobster 工作流**：把多步操作变成“可重复、可审计、带审批关卡”的流程，避免模型边想边做把你带沟里。

总之，Clawdbot 值得玩的前提，是你愿意把安全当成第一功能，而不是“以后再说”的无脑一把梭。**

## 谁适合上手

适合的人（装了就能爽到）：

- **消息驱动工作流重的人**：日程、协作、提醒、信息收集都在聊天里。

- **愿意自托管的人**：你能接受“跑服务 + 配 key + 维护升级”。

- **想把 AI 变成系统的人**：你不满足于“问答”，你要“流程”。

不适合的人（装了大概率弃坑）：

- 只想要一个“更聪明的 ChatGPT”

- 不想碰 CLI / 配置 / 权限管理

- 打算一上来就把它接到银行、主邮箱、生产服务器 —— **你这是在赌运气**

## 想清楚再上手

如果你真想试，别从“自动发邮件/自动下单”开始，按这个顺序走：

1. **先跑起来，只做信息型任务**（总结、清单、草稿）

2. **再接工具，但强制 approvals**（尤其是 exec/browser）

3. **最后才谈自动化**：用 Lobster 把高风险操作做成“带关卡的流程”，把不可逆动作锁死。

你会发现：
真正的“未来感”不是 AI 会不会聊，而是——**你发一句话，现实世界真的发生了变化，但你依然握着刹车。**
