大家好，我是程序员小灰。

这两年 AI 技术的发展，为广大普通人带来了前所未有的机会。

在小灰的合伙人社群中，有一位正在读大二的学生，他借助 Agent + Obsidian，真正实现了一人企业，大半年时间累计变现 6 万+。

这个收入对于工作很多年的职场人，并不算什么大钱；但对于一个年仅 20 岁的在校学生来说，这样的成绩绝对秒杀了 99.9% 的同龄人！

他是如何做到这一切的呢？今天就由这位年轻人为大家一次性讲明白：OPC 怎么赚钱？Agent + Obsidian 怎么工作？他的两套后端收入系统是怎样搭建的？

下文中的 “我” 不是小灰，而是这位年轻的学生。

一、总体介绍

最近一段时间，我听到了很多新词。

OPC、一人公司、Agent、知识库、工作台、Skills……

这些词一个接着一个冒出来，确实挺让人脑壳疼的。尤其是你刚开始接触的时候，很容易产生一种感觉：这些东西听起来都很厉害，但它们到底和我有什么关系？又怎么帮我赚钱？所以我今天写这篇文章，不准备再给你堆一大堆复杂概念。

我就想把三件事情讲清楚：

第一，OPC 一人公司的赚钱系统，到底是怎么运转的。

第二，Agent + Obsidian 到底是干什么的，它们为什么能够成为一个人的第二分身。

第三，我现在自己的 OPC 已经搭成了什么样子，又是通过哪两种方式赚钱的。

先简单介绍一下我自己：

我是安十一，小灰十年风云会成员，大二在读，AI 创业中。

我从去年 10 月开始接触 AI，到现在累计通过 AI 变现超过 6+ 万元，不算拿到大结果的人，我不是程序员，也不是计算机专业的。

我的 AI 学习得还是比较厉害的，且涉及多个方面：

AI 提示词、AI 设计、AI 编程、课程制作，以及 Agent + 知识库相关的工作。

下面我们正式开始。

二、OPC 一人公司的赚钱系统，其实没有那么复杂

OPC 的英文是 One Person Company，也就是一人公司。

很多人一听「一人公司」，脑子里面想到的可能是：

一个人注册一家公司，一个人做老板，一个人把所有工作全部干完。

但我现在理解的 OPC，并不是让一个人把自己活成十个人。

它更像是一套围绕一个人运转的小型商业系统。

这个系统最核心的结构，其实就三个环节：

前端获客 → 中端转化 → 后端交付

![](https://mmbiz.qpic.cn/mmbiz_png/9ojSnBX2WWZYSWoCsvFc0F9aRRfTWUSeZrWRsiaBrSmUqnVxuJ5yYeWSRkgHLWsYbgwZ5cX4ZEhx7d6QLmuCBfZ3PcP7hEu0miamZr48RVwQE/640?wx_fmt=png&from=appmsg)

前端获客是什么？

前端获客，简单来说，就是让别人先看到你、认识你。

比如我现在会通过公众号和知乎持续写文章。

这些文章不一定发出去马上就有人买东西，但它们会慢慢告诉别人：

我是谁，我长期在研究什么，我能够解决什么问题。

除了公众号和知乎，我也会在小红书、闲鱼等平台上架自己制作的虚拟产品。

例如 AI 学习手册、教程或者其他数字产品。

有些人可能最开始根本不知道安十一是谁。

他只是先看到了我的一个产品，觉得这个东西能解决他的问题，后面才逐渐进入我的私域，继续了解我。

所以，公众号和知乎更偏向建立认知和信任。

小红书和闲鱼则更像一个具体的产品入口。

中端转化是什么？

前端把人带过来之后，中间还需要建立信任。

这就是中端转化。

比如用户添加我的微信之后，会看到我的朋友圈、社群分享、课程案例、学员反馈，以及我每天正在做的事情。

他会慢慢知道：这个人不是只会讲概念，他确实长期在做 AI，也确实做过不少真实项目。

等用户真正有需求的时候，他可能会购买我的课程，也可能会找我做一对一咨询，或者邀请我去做企业培训。

所以，中端转化不是加上微信之后，马上上去硬推销。

它更像是通过持续输出，让用户慢慢知道你是谁、你能干什么，以及为什么值得相信。

后端交付是什么？

后端就是用户真正付钱之后，你拿什么东西交付给他。

可以是一门课程、一本学习手册、一个社群，也可以是一对一咨询、陪跑服务或者企业培训。

比如我现在的后端，主要有两条赚钱线：

第一条是做课卖课。

第二条是一对一咨询和企业培训。

后面我会详细讲，我这两条线现在是怎么运转的。

三、Agent + Obsidian，又是干什么的？

把 OPC 的基础结构搞清楚之后，我们再聊 Agent + Obsidian。

Agent 这个词听起来很复杂。

你可以先把它理解成：一个不只是陪你聊天，而是真的可以帮你干活的 AI。

比如 Codex、Claude Code、WorkBuddy，这些都可以算是 Agent 工具。

它们可以读取文件、整理资料、修改内容、生成文档，甚至帮你搭建网页、工具和工作流程。

但 Agent 有一个很大的问题：它不一定真正了解你。

你每次打开一个新的任务，可能都要重新告诉它：

我是谁？我现在在做什么？我的写作风格是什么？我的产品是什么？

这次内容应该怎么写？哪些地方可以改，哪些地方不能动？

说一次还好。但如果每次干活都要重新说一遍，那你会发现，自己好像只是把原来的工作，换成了「重复培训 AI」。

这就轮到 Obsidian 出场了，Obsidian 本质上是一个本地知识库。

你可以把它理解成一个专门给自己和 AI 准备的笔记本。

在这个笔记本里面，我会记录：

我是谁；我的成长经历；我的个人定位；我的写作风格；我的产品体系..........

Agent 负责干活，Obsidian 负责提供记忆。

两个东西连接起来之后，AI 就不再只是一个临时工具。

它开始真正知道我是谁，也知道应该按照什么方式帮我工作。

在我的 OPC 里面，Agent + Obsidian 就像一个可以干活的第二分身。

它不能代替我做所有决定。

但它可以按照我提前写好的规则，处理大量重复工作。

## 四、分享一下我的我的知识库搭建方案

很多人搭知识库，就是不断往里面放文章、课程和提示词。

资料越来越多，但真正让 AI 工作的时候，它还是不知道应该先读哪里、调用什么内容。

所以我的知识库不是简单堆放资料，而是分成五个相互配合的模块。

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/9eCKqH4xIOeGxsQFDK9HkK1mjwgQY6TZSBGp3o9egTCLt1sS4Aeu6icMvuHzAsDqzYv0MUX5T29waPyEBcgMFakfhG2suqa1icibghRekHStMo/640?wx_fmt=jpeg&from=appmsg&watermark=1#imgIndex=1)

第一部分：Agent 入口层

这一部分是专门写给 AI 看的，相当于整套知识库的说明书和导航地图。

它会告诉 AI：这个知识库是做什么的、里面有哪些模块、每个模块负责什么，以及遇到不同任务时应该去哪里寻找资料。

这样 AI 不需要每次把整个知识库全部读完，而是先了解整体结构，再根据当前任务读取对应内容，速度更快，也不容易抓错重点。

第二部分：个人 IP 中控台

个人 IP 中控台主要沉淀“我是谁”。

这里会记录我的个人定位、成长经历、写作风格、产品体系、真实案例、用户反馈，以及我长期使用的方法和判断标准。

AI 读取这些内容以后，才知道应该按照谁的方式工作。它写文章时会更像我，制作课程时也能结合我的学员、产品和真实经历，而不是输出一份谁都能用的标准答案。

第三部分：知识库生长系统

知识库不是搭完以后就不再变化，它应该随着我的成长和业务一起更新。

这一部分主要规定：新资料应该放在哪里、哪些内容可以直接写入、哪些内容需要我审核、新旧信息冲突时怎么处理，以及每次更新如何留下记录。

它负责不断更新个人 IP 中控台，避免知识库里的定位、产品、案例和工作流程长期停留在旧版本。

第四部分：工作流与 Skills 层

这一部分放的是各种具体 Skills，你可以把 Skill 理解成一份给 AI 的固定工作说明书。

例如公众号写作、课程大纲制作、学习手册制作、图片规划、朋友圈创作等，都可以分别做成一个 Skill。

每个 Skill 只负责解决一类任务，写清楚要读取什么资料、按什么步骤执行、最后输出什么结果，让 AI 遇到同类任务时不用重新培训。

第五部分：工作流指挥系统

单个 Skill 只能完成一项任务，但真实工作通常需要多个 Skills 配合。

比如制作一门课程，可能需要依次调用资料分析、课程大纲、写作风格、学习手册和配图规划等多个模块。

工作流指挥系统就像总指挥，负责判断任务类型、选择需要调用的 Skills、安排执行顺序，并决定哪些环节需要我确认，最后把多个 Skills 串成一套完整的工作流程。

五个模块是怎么配合的？

当我把一个任务交给 Agent 后，它会先读取入口说明，再调用个人 IP 资料，根据任务选择对应 Skills，并由工作流指挥系统统一安排执行。

任务完成以后，新产生的内容还会按照知识库生长规则，继续沉淀回知识库。

所以，我的知识库不是一个资料仓库，而是一套能够被 AI 理解、调用、执行并持续更新的个人工作系统。

五、我的第二分身，现在能帮我做什么？

我知识库搭建好了，那么他可以给我干什么事情呢

第一，帮我写文章

我平时很多文章，不是一开始就一字一句打出来的。

我可能先用语音输入法，把脑子里面的大概想法说出来。

这一版内容通常会比较乱。

有重复、有口语、有些地方逻辑也没完全整理清楚。

然后我会把这些草稿交给 Agent。

Agent 会读取我的写作风格规范、个人经历、内容创作流程，以及这篇文章的任务要求，再帮我整理成一篇完整文章。

这不是简单让 AI「帮我写一篇文章」，而是让它按照我的风格、我的经历和我的判断去整理。

最后出来的内容，才不会像一篇谁都能写的标准答案。

第二，帮我制作朋友圈内容

我会把自己的定位、产品、课程、案例和近期工作，持续记录到知识库里面。

当我需要写朋友圈的时候，Agent 可以直接读取这些信息，它知道我最近在做什么，也知道我有哪些产品和案例。

这样写出来的朋友圈，就不会每次都从零开始编，它可以围绕我真实做过的事情，帮我生成不同角度的内容。

第三，帮我制作课程

这是我现在使用最多的一套工作流。

当我要做一门课的时候，我会先把课程资料、大纲和相关案例整理出来。

Agent 会读取我的课程大纲制作规则、写作风格说明书、图文手册规则和交付标准。

然后按照顺序帮我处理：

课程大纲、课程讲稿、学习手册、配图方案、PPT 内容和交付资料。

以前是我一个人，每做一门课都要重新想一次。

现在是我先搭好一套系统，再让 AI 按照这套系统帮我执行。

这两个状态看起来只差了一点，但实际工作效率完全不一样。

你现在应该能够看懂，OPC 和 Agent + Obsidian 之间是什么关系了。

OPC 是我的赚钱系统。

Agent + Obsidian 则是帮助这套赚钱系统运转的工作底盘。

## 六、我为什么不想再追一个又一个项目？

前面我已经介绍了，我现在是大二在读，也一直处在创业和折腾的状态。

其实从刚进入大学开始，我就不太老实。那时候我开始接触互联网，也尝试过各种项目。做过闲鱼虚拟，也做过小红书虚拟。

有些项目赚到过钱，有些项目交了不少学费。说实话，刚开始做互联网项目的时候，我最关心的问题就是：最近什么项目赚钱？哪一个项目来钱更快？哪里有新的流量机会？

这种想法不能说完全错误。因为刚开始的时候，你确实需要先跑通一次赚钱闭环。

但我后来慢慢发现，短期项目有一个非常大的问题：它不稳定。

平台规则一变，原来的玩法可能就失效了。某个产品的热度一过，销量可能马上掉下来。

甚至有一次，我的学习手册已经做到了 85%，结果平台突然调整规则。

那一下确实把我打蒙了。因为你投入了很多时间，产品都快做完了，最后却发现原来的路径可能走不通了。

后来我才真正想明白：做项目可以让我赚到一次钱，但只有建立自己的系统，才能让我持续赚钱。

所以我不再只问：「下一个赚钱项目是什么？」

我开始问自己：客户为什么认识我？流量最后沉淀在哪里？

我到底有什么可以长期卖的产品？就算某一个平台发生变化，我还能不能继续运转？

这也是我为什么开始真正搭建自己的 OPC。

## 七、我现在的 OPC，到底长什么样？

我的 OPC 没有什么特别神秘的地方，没有十几个部门，也没有特别庞大的系统。

它目前就是围绕我个人形成的三个环节，前端是内容和平台，中端是微信私域、朋友圈和社群。

后端是两套赚钱系统：课程产品 + 一对一咨询

前端：通过内容和产品让用户认识我

公众号和知乎，是我现在比较重要的内容阵地。

我会持续写自己的学习过程、AI 实操、工具使用、项目复盘和个人判断。

这些内容不会每一篇都爆。甚至很长一段时间，数据可能都很一般。

但它们会慢慢形成一项资产，用户看完一篇文章，可能不会马上找我。

但当他连续看了几篇之后，就会慢慢知道：安十一长期在做 AI 提示词、AI 设计、AI 编程和知识库。

小红书和闲鱼则更加偏向具体产品，用户可能先购买一本低客单的学习手册，再进入我的微信和社群。

所以我的前端，不只是靠一篇文章爆一下，它更像是不同平台一起工作，把合适的人慢慢带到我这里。

中端：把平台流量变成自己的基本盘

我刚开始做互联网的时候，其实没有特别重视私域。

那时候更关心每天出了多少单、赚了多少钱。

后来平台规则发生变化，我才意识到：

平台上的流量，并不真正属于我。

账号限流了、规则调整了，原来的流量可能说没就没。

但用户进入我的微信之后，我还能通过朋友圈、社群和内容继续和他建立联系。

所以现在我越来越重视自己的私域，不是为了加很多人之后天天发广告。

而是为了持续让用户知道，我最近做了什么、解决了什么问题、又跑通了哪些新东西。

私域真正重要的地方，不是微信里面有多少人，而是这些人知不知道你是谁，又愿不愿意在有需求的时候找你。

你说得对，是我理解反了。

## 八、第一套后端：从市场需求出发，快速学习并制作课程

我的第一套后端，是把有市场需求的 AI 内容快速学会、完成实操，再制作成可以销售和交付的图文课程。

我不会看到一门课程就直接去学，更不会学完以后才考虑有没有人需要。

我的第一步通常是先做市场调研，了解最近哪些 AI 方向比较受关注、用户正在为什么问题付费、同类产品的销售情况怎么样，以及这个需求属于长期需求还是短期热点。

调研完成以后，我才会判断这个方向值不值得进入。

主要看三点：市场有没有真实需求、我能不能快速掌握，以及最后能不能制作成适合销售和交付的课程产品。

因为我本身已经积累了比较强的 AI 基础和实操能力，所以很多新的 AI 工具和工作流，我通常用一两天就可以快速学完。

但这里的“学完”不是把视频看完，而是自己真正操作一遍，确认每个步骤都能独立完成，同时找出课程里哪些地方不够清楚、哪些步骤容易踩坑，以及怎样讲会更适合小白。

真正掌握以后，我才会进入课程制作阶段。课程大纲由我和 Agent 协同完成。我负责确定目标用户、课程方向、核心知识和学习顺序，Agent 负责辅助整理章节结构和每一节标题。

但 Agent 生成的大纲不能直接使用，我还需要重新检查内容有没有重复或缺失、难度是否合理、标题是否准确，以及哪些地方需要补充案例和实操。确认大纲以后，再通过 Agent + Obsidian 调用我的课程制作系统，完成正文、配图和图文排版，最终交付为飞书、网页或 PDF 形式的课程手册。

我刚开始学习 AI 绘画时，就是先学会、跑通，再把自己的实操方法整理成手册。大概卖了 253 单。

后来 OpenClaw 爆火，我也快速学习并制作了相关产品，最高的一天卖了接近 500 单，不过热点过去以后，单量也很快降了下来。

所以我现在做课的逻辑很明确：

**先判断市场需要什么，再快速学习、完成实操，最后把真正跑通的方法做成可以销售、交付和持续复用的课程产品。**

![](https://mmbiz.qpic.cn/sz_mmbiz_png/9eCKqH4xIOciaqso2zB8b2iaP0BiaELvMCSia2seVegjpwvtoXjU7wnH9fwu5jfxoiaGia6MOx8n4TTqS945d6UicdIe8C9xgia3ESJ86IfShPttOPk/640?wx_fmt=png&from=appmsg&watermark=1#imgIndex=4)

所以我现在做课程，不是凭空找一个热门话题，然后硬凑十节课。

我更希望课程从真实实践中长出来。

我先亲自跑通，再把过程中真正有效的方法整理出来。

我现在越来越觉得：

课程真正值钱的，不是里面有多少节视频，而是它能不能把一条已经跑通的路，清楚地交给另一个人。

## 九、第二套后端：为老板定制 AI 学习路径

我现在的一对一服务，主要面向中小企业老板，管理层，按小时收费。

这项服务不是拿一套固定课程反复讲，而是先了解老板目前在做什么、业务卡在哪里，以及他对 AI 已经掌握到什么程度，再决定后面的课程应该怎么安排。

我的流程其实很简单：

先了解业务，再判断 AI 基础，最后规划学习路径。

比如有些老板最需要解决的是内容和个人 IP，有些老板更需要提升视觉设计、客户沟通，或者把自己的经验沉淀成知识库。方向不同，课程内容自然也不能完全一样。

我最近接触的两个客户，一个是经营香港美容、皮肤管理店老板课程会重点围绕 AI 提示词、AI 设计、Skills 和知识库展开。

另一个是经营 16 家美甲店、管理近 200 人，她的课程则更偏向内容制作、视觉设计、客户沟通和门店知识管理。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/9eCKqH4xIOfO7NPj1WQewFDB0Z3n1am1Bo17KvXStOS3cTsfTjdbibTSQLrhjlZZ0EQick6H5ywQaPLbDPYQ3sickoXRzmtibQI4j2WGTssWHP0/640?wx_fmt=png&from=appmsg&watermark=1#imgIndex=5)

我现在越来越觉得，老板学 AI 最怕的不是不会用工具，而是学了一堆工具，却不知道怎么进入自己的业务。

所以我的一对一服务，真正解决的不是“教几个工具”，而是帮他找到一条适合自己业务的 AI 学习路径。

所以我现在做咨询和培训，真正提供的不是某个工具。

而是我的判断、经验、方法和把 AI 转化成实际结果的能力。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/9eCKqH4xIOcoIPXy8YR8zzH518ncx8ugCAqOUGYyg1Q7dxT5eSD573CYAnZH149wcCB1S9jXX2U2uDwgopG62TlZU5mToP5miciaf93bfj87g/640?wx_fmt=png&from=appmsg&watermark=1#imgIndex=6)

做课和做咨询，其实在互相喂养

表面上看，卖课和做咨询是两种不同的业务。

但在我的 OPC 里面，它们其实是一套循环。

课程解决的是规模化。

我可以把已经跑通的方法，整理成课程、手册和模板，交付给更多人。

用户也可以通过一个低门槛产品，先了解我的方法和能力。

咨询解决的是深度。

真实客户的问题，通常不会完全按照课程大纲出现。

你会遇到各种行业、各种业务和各种奇怪的卡点。

这些问题会倒逼我继续思考：

原来的方法哪里还不够？

哪些地方需要重新设计？

有没有新的场景值得做成产品？

当同一个问题在咨询里面反复出现，我就可以把它继续整理成：

一节新课程、一套模板、一个 Skill，或者一份新的解决方案。

而课程学员遇到更加个性化的问题时，也可能进一步进入一对一咨询和企业服务。

所以这两条线并不是分开的。

课程让我把已经跑通的方法放大，咨询则让我不断遇到新的真实问题。一个负责复制，一个负责生长。

## 十、Agent + Obsidian，不是赚钱系统，而是这套系统的底盘

讲到这里，你应该已经发现了一个问题。

Agent + Obsidian 本身并不会直接给我带来客户。

也不会因为我搭了一个知识库，别人就主动给我付钱。

真正产生收入的，仍然是：

内容、产品、咨询、案例和用户信任。

Agent + Obsidian 的作用，是让这些事情更加稳定地运转。

在前端，它帮我提高内容生产效率

我可以把自己的想法、资料和案例放进去。

Agent 根据我的写作风格和内容流程，帮助我整理公众号、知乎和朋友圈内容。

一份内容也不必只使用一次。

它可以进一步整理成短内容、课程素材、社群分享或者选题库。

在中端，它帮我沉淀用户问题和案例

用户经常问什么？课程里面哪些地方容易听不懂？

咨询时反复出现了哪些问题？哪些案例适合写进文章？

这些信息都可以不断进入知识库。它们以后又会变成新的朋友圈、新的课程内容和新的销售素材。

在后端，它帮我制作和交付产品

课程大纲、学习手册、配图方案、PPT、咨询资料，都可以在同一套知识库和工作流里面生产。

我现在经常会同时打开多个 Agent。一个处理课程手册，一个整理文章，一个继续优化知识库。

它们共用同一套关于我的资料和规则。

以前是我一个人在干活。现在更像是我负责分配任务、制定标准和检查结果，不同的 AI 负责具体执行。

![](https://mmbiz.qpic.cn/mmbiz_png/9eCKqH4xIOfB0dPe095bywmNWaPFlm0guIFkLUV8SJBLt4pHxqqiaDYrZNCBEhlGxkA8iaqRV4SXmuJGbWe3CaLgPqar4awJa9crgPuibw5AiaM/640?wx_fmt=png&from=appmsg&watermark=1#imgIndex=7)

这才是我理解的一人公司。

它不是一个人什么都自己做。

而是一个人负责方向和决策，再调动多个 AI 围绕同一个业务目标协同工作。

人负责：方向；规则；判断；决策；最终审核。

AI 负责：整理；执行；复用；提高效率；放大交付能力。

所以，OPC 的核心不是我什么都会做。

而是我能不能把自己的经验、规则和业务，逐渐交给一套系统去协同执行。

这套系统已经跑起来了，但还远远没有完成

我现在的 OPC 已经有了一个基础框架。

前端有公众号、知乎、小红书和闲鱼。

中端有微信私域、朋友圈和社群。

后端有课程产品、一对一咨询和企业培训。

Agent + Obsidian 也已经进入了我的内容、课程和交付流程。

但说实话，这套系统还远远谈不上成熟。

我的公域流量还不够稳定。

很多服务仍然比较依赖我本人。

咨询和培训中积累下来的经验，也没有全部完成产品化。

有些工作流虽然已经能跑，但仍然需要我不断调整和优化。

不过我现在已经不太焦虑了。

因为我知道自己接下来应该做什么。

第一，继续把已经跑通的课程和交付流程标准化。

第二，进入更多真实的老板和企业场景，积累真正能够产生结果的案例。

第三，把咨询和培训中反复出现的问题，逐渐做成课程、模板、工作流和解决方案。

我现在真正想做的，已经不是再学十个新工具。

而是把已经跑通的业务继续做深，把能够重复的工作继续交给系统。

## 写在最后：我现在理解的 OPC

回头看，我以前一直在寻找一个能够赚钱的项目。

今天做闲鱼，明天做小红书，后天又追一个新的机会。

这些经历并没有白费。

它们让我理解了流量、产品、私域和交付。

但它们也让我意识到：

只依赖某一个项目，你永远不知道它什么时候会停。

所以我现在更希望自己拥有的，不是下一个短期风口。

而是一套真正属于自己的系统。

这里面有我的内容、私域、产品、客户、知识库和工作流程。

我现在理解的 OPC，也不是注册一家只有一个人的公司。

更不是让自己每天从早忙到晚，把十个人的工作全部扛在身上。

它应该是：

你先拥有一套自己的业务，再让 AI 帮你把这套业务更加稳定地运转下去。

你也不需要一上来就搭一个特别复杂的 Agent，更不需要先建几十个知识库文件夹。

你可以先把三件事情写清楚：

你的客户从哪里来？

你准备向他卖什么？

你的工作里面，哪一件事情最重复、最适合先交给 AI？

先让一个真实客户，愿意为你提供的一个真实结果付钱。

然后，再把这件事情一点点做成流程、做成系统、做成产品。

这可能才是普通人开始搭建 OPC，最现实的一条路。
