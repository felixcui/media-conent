## Codex从入门到精通：2026 最全实战指南（5000字长文）

### 写在前面：2026年，为什么你必须了解 Codex

如果你今年只打算学一个AI工具，别只盯着ChatGPT，一定要看看Codex。

过去两年，AI编程工具经历了一场惨烈的淘汰赛。Cursor风光过，Claude Code封过神，GitHub Copilot坐拥千万用户——但到了2026年，所有人都发现一个趋势：真正的AI Agent时代来了。

Codex，就是这场战争中跑得最快的那匹黑马。

从2025年发布至今，Codex的周活跃用户已经突破400万。Sam Altman亲自发推说"Codex is having a ChatGPT moment"。OpenAI内部85%的员工每周都在用它。最近一个月里的更新密集程度——GPT-5.5模型上线、OSS模式、插件市场、桌面宠物、/goal目标模式、Computer Use电脑操控、从Claude Code一键迁移。

### 第一章：Codex 到底是什么？

**它不是ChatGPT的桌面版。** ChatGPT是你请的兼职顾问，每次上传文件，做完手动下载。Codex是住你电脑里的全职同事，你的文件、文件夹、浏览器、命令行、甚至桌面上的其他软件都是它的。

> ChatGPT解决的是"怎么说"，Codex想的是"怎么把事直接做完"。

**Codex vs Claude Code：** 入门新人无脑选Codex。重度用户可以两个都用——日常开发用Codex，写作或特定场景用Claude Code。

### 第二章：安装与上手

四种使用方式：桌面App（最推荐新手）、CLI命令行、VS Code插件、Web网页版。

安装步骤：装好Git和Node.js → 访问chatgpt.com/codex下载 → 登录（ChatGPT账号或API Key）→ 绑定工作文件夹。

国内用户可用CC Switch接入国内大模型（DeepSeek、GLM等）。

界面核心区分：**Chat vs Project**——Chat模式文件不自动保存，Project模式会自动创建文件夹并保存，一定要学会用！

### 第三章：八大核心能力

1. **本地文件操作**：直接读取和操作本地文件，不限数量
2. **命令行工具**：在授权情况下使用终端执行命令
3. **持久记忆（agents.md）**：全局agents.md + 项目级agents.md + 自动记忆（实验性）
4. **图片生成**：内置OpenAI image2模型，免费使用
5. **插件（Plugins）**：可安装的打包单元，把Skills、应用集成、MCP配置打包分发，App/CLI/IDE通用
6. **Skills技能沉淀**：可重复使用的方法、流程和工具组合，一句话调用
7. **MCP配置**：图形化界面配置外部工具连接，不用写JSON
8. **自动化定时任务**：组合所有能力，让Agent定时定点自动执行

### 第四章：安全管控与权限管理

三级权限体系：完全访问、需确认、禁止。建议新手用完全访问，但不要给到核心文件夹。

**计划模式（Plan Mode）**：复杂任务先让Codex出计划，确认后再执行。

**自定义指令**：一次设置永久生效，如"未经二次确认不删除文件"、"写代码优先用现成库"等。

### 第五章：2026 最新功能

- **桌面宠物 /pet**：内置8只像素风宠物，可自定义生成
- **/goal 目标模式**：只给目标，Codex自己拆解、执行，直到做完
- **Computer Use**：GPT-5.4起独占能力，像真人一样操控电脑屏幕

### 第六章：从 Claude Code 迁移

操作路径：设置 → General → Import other agent setup → Import。迁移覆盖CLAUDE.md→AGENTS.md、settings.json→config.toml、Skills→Codex Skills、MCP→Codex MCP等。

### 结语：从"问AI"到"管理AI"

我们已经从"问AI"的阶段正式迈入了"管理AI"的阶段。好的领导会做两件事：把好的方法和流程沉淀成可复用的Skills，把反复要做的事设成自动化执行的规矩。

> Codex会取代程序员吗？不会。但会用Codex的程序员，会取代不会用的。
