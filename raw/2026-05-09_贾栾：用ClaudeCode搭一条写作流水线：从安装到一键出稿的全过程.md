## 摘要

本文介绍了使用Claude Code搭建个人写作流水线的全过程，旨在解决AI写作缺乏个人风格和深度记忆的问题。作者通过安装Claude Code并接入MCP记忆系统，让AI记录对话、沉淀用户画像并按需召回，从而真正“认识”作者。在此基础上，通过建立案例库逆向提取风格基因，将标准化写作SOP固化为Skill指令，最终实现了十分钟产出六千字具备个人风格的高质量终稿，且系统越用越准。

## 为什么 AI 写出来的东西不像你
我试过在提示词里加各种要求："语气要犀利""要有个人风格""不要 AI 腔"。有用吗？有一点，但远远不够。

问题出在哪？两层。

第一层是风格。你用一千字描述"我要什么风格"，AI 还是不知道你到底要什么。就像你跟一个新来的实习生说"写得有趣一点"，他只会加几个感叹号。但如果你甩给他一篇你写过的满分稿子说"照这个来"，他立刻就懂了。

第二层更深——AI 不认识你。它不知道你踩过什么坑、做过什么决策、对什么事情有什么态度。一个跟了你三年的助理写出来的东西，和一个刚来的实习生写出来的，差距不在文笔，在于他懂不懂你这个人。

说白了，风格是"喂"出来的，但神似是"养"出来的。喂范文解决形似，养记忆才能解决神似。




这就是我搭这条流水线的起点：不是让 AI 替我写，也不只是让 AI 模仿我的风格，而是让 AI 真正了解我——知道我怎么想、怎么做决策、对什么事情有什么态度——然后按"我"来写。整条路径是这样的：

```plaintext
装 Claude Code → 接记忆系统(MCP) → 建案例库 → 提取风格基因 → SOP化 → Skill固化 → 一键出稿
```


下面一步步拆。

## 第一步：安装 Claude Code，搭基础环境
### 做什么
Claude Code（下面简称 CC）是 Anthropic 出的命令行 AI 工具。跟 IDE 里的 AI 助手不同，它直接在终端里跑，能读写你的本地文件、执行脚本、调用各种外部工具。

说白了，IDE 里的 AI 像个嵌入式助手，CC 更像一个独立的执行引擎——你给它指令，它直接在你的文件系统里干活。

### 怎么做
安装只需要一行命令：
```plaintext
irm https://claude.ai/install.ps1 | iex
```


同时配合 CC-switch 插件，可以使用中转 API 平台来调用不同大模型。

并不复杂，这里我不详细说明，细节还是挺多的。推荐一个体系化教程：
老金：开源10万字Claude Code中文教程，零基础到企业实战完整路径

## 第二步：接记忆系统，让 AI"认识"你
### 做什么
CC 单独用已经能读写文件、跑脚本。但它有一个致命缺陷：没有记忆。每次对话都是从零开始，它不知道你昨天做了什么、上周踩了什么坑、你对某个工具的真实评价是什么。

这就像每天换一个新助理。能力再强，不了解你，写出来的东西也只能是"通用好文"，不是"你的文章"。

解决方案是通过 MCP（Model Context Protocol）接入一个记忆系统。MCP 是什么？你可以理解为给 AI 装的"外挂接口"——让它能连数据库、上网搜索、调用各种外部服务。就像手机装了 App Store，手机本身能打电话，但装了微信、支付宝之后才真正好用。

而在写作这个场景里，最关键的"App"就是记忆系统。

### 怎么做
我接的是自建的记忆系统（Jarvis），它做三件事：

1. **记录对话**：我和 CC 的每一轮对话都会存档，包括我的原话和 AI 的回复
2. **沉淀画像**：从对话中提取我的偏好、习惯、认知模式，形成持久的用户画像
3. **按需召回**：写作时，AI 会主动检索相关记忆——我对这个话题说过什么、做过什么决策、踩过什么坑

配置好之后重启 CC，它就能调用记忆系统了。你也可以接搜索引擎（Brave Search）、数据库、或者任何支持 MCP 协议的服务。

### 记忆系统对写作意味着什么
这一步很多人会觉得"可选"，但在我看来它是整条流水线的地基。

举个真实的例子：我让 AI 写一篇关于"遇到问题先问 AI 再问人"的观点短文。如果没有记忆系统，AI 只能根据我给的三句话素材 + 范文风格来写，产出的是一篇"风格像我但内容泛泛"的文章。

但有了记忆系统，AI 在写之前会先查：这个人之前对"AI 协作"说过什么？他的核心主张是什么？他用过什么隐喻？他踩过什么坑？

它知道我推崇"AI First"的工作方式，知道我把 AI 比作"预诊台"、把专家比作"主治医生"，知道我反感"什么都不想就甩给别人"的行为。这些不是我在素材里写的，是它从过去的对话里"记住"的。

所以写出来的文章，不只是风格像我，连思路、立场、举例方式都像我。这就是"形似"和"神似"的区别。

说白了，案例库教 AI"怎么写"，记忆系统教 AI"怎么想"。两个都有，才是真正的影子写手。

### 踩坑提醒
⚠️ MCP 在 CC 里的调用准确率比在 IDE 里高很多。同样的配置，在 IDE 里经常翻车（不触发、参数传错），在 CC 里跑得很顺。我猜是因为 CC 的上下文更纯净——它不用兼顾代码补全和文件索引，MCP 指令不容易被"挤掉"。

⚠️ 如果你发现 MCP 配置好了但 CC 不调用，先检查你的系统指令（CLAUDE.md）里有没有显式要求 CC 调用 MCP。我踩过一个坑：重构系统指令时不小心删掉了"强制调用记忆系统"的规则，结果 CC 就不读记忆了。排错的时候别急着怀疑代码，先查 Git 历史看看指令文件改了什么。

### CC 的最佳搭档：CC + IDE 双开
这里多说一个实战经验。CC 在终端里跑，改了什么文件全靠脑补。但如果你同时开着 IDE（VS Code、Cursor、Trae 都行），CC 在后台改文件，IDE 实时显示 diff。

CC 是执行引擎，IDE 是监控大屏。这个组合用起来非常舒服，强烈推荐。

还有一个血泪教训：CC 最大的坑是"太执着"。一条路走不通，它不会主动换方向，而是换个姿势继续撞墙，Token 消耗指数级上涨。你必须当"安全员"——发现它在绕圈子，果断输入 `/clear` 清掉上下文，重新来。别心疼那几轮对话，比烧 Token 便宜多了。

## 第三步：建案例库，给 AI 一面"镜子"
### 做什么
环境搭好了，接下来是核心问题：怎么让 AI 写出"像你"的文章？

答案不是写更长的提示词，而是给 AI 看你的作品。

这就像学画画。你跟学生说"画得生动一点"，他不知道怎么做。但你给他一幅范画让他临摹，他立刻就懂了。案例库就是你给 AI 的"范画集"。

### 怎么做
从你过去发布的文章里，挑 3-5 篇你最满意的，按类型分类存好：
```plaintext
写作案例库/
├── 01_深度解析类/
│   └── 你写过的深度分析文章.md
├── 02_实操教程类/
│   └── 你写过的教程文章.md
└── 03_观点评论类/
    └── 你写过的观点短文.md
```


每篇文章加上元数据头，标注类型、评级、风格特征：
```markdown
<!--
@meta
title: 文章标题
type: 实操教程
rating: ⭐⭐⭐
tags: [标签1, 标签2]
style_dna:
  tone: 手把手、祛魅、第一视角
  structure: 痛点引入 -> 实操步骤 -> 洞察收尾
  metaphor: 核心隐喻
-->
```


评级标准：
- ⭐⭐⭐ 标杆范文——风格浓度最高，AI 写作时优先参考
- ⭐⭐ 合格——结构和语气都不错，可作为备选参考
- ⭐ 仅参考结构——内容可以，但个人风格稀薄，不作为语气参考

我一开始只放了 5 篇，但这 5 篇就够 AI 摸清我的"音色"了。

### 踩坑提醒
⚠️ 案例库的质量决定了输出的上限。别把所有文章都塞进去，只放你真正满意的。一篇三星范文的价值，顶十篇一星的。

⚠️ 分类要清晰。观点短文和实操教程的写法完全不同，混在一起会让 AI"串味"。

## 第四步：逆向提取"风格基因"
### 做什么
案例库有了，但 AI 只是"看过"你的文章，还不知道该学什么。你需要把你的写作风格提炼成明确的规则——我管这叫"风格基因"。

这一步是整条流水线的灵魂。没有风格基因，AI 写出来的东西最多形似；有了风格基因，它才能神似。

```plaintext

赋能、抓手、全面盘点、一文读懂……
```


这 7 条基因 + 黑名单，就是你的"写作宪法"。AI 写的每一个字，都必须过这道关。



### 踩坑提醒
⚠️ 风格基因不用一次提取完美。先跑起来，写几篇文章之后你会发现哪些规则有效、哪些需要调整。我的 7 条也是迭代了两轮才稳定的。
⚠️ 黑名单要持续更新。每次看到 AI 输出里有"味道不对"的词，就加进去。这个名单越长，AI 的"原形毕露"概率越低。

## 第五步：SOP 化——把流程变成流水线
### 做什么
风格基因有了，案例库有了，但每次写文章还得手动走一遍流程：选范文、喂素材、校验风格……能不能标准化？
能。把整个写作流程写成 SOP（标准作业程序），像工厂流水线一样——原材料进去，成品出来，中间每个工位干什么都定好了。

### 怎么做
我把写作流程拆成了两条产线：

**短文线（图文号日更，10-15 分钟/篇）：**
```plaintext
素材捕获(2分钟) → AI成稿(5-8分钟) → 人工扫描(3-5分钟) → 发布
```

你有一个想法、一个吐槽、一个发现——在临时收件箱里用最粗糙的方式写下核心观点和例子（3-10 句话就够），然后丢给 AI，它按你的风格基因重写，你扫一眼改几个词，发。

**长文线（公众号周更，30-40 分钟/篇）：**
```plaintext
素材沉淀(10分钟) → 选范文(2分钟) → AI成稿(10-15分钟) → 人工精修(10-15分钟) → 发布 + 反哺案例库
```

长文多了两个环节：一是写"素材卡"（主题、核心观点、关键数据、踩过的坑、情绪锚点），二是精修时逐条检查 7 条风格基因。
每条产线都有明确的输入、动作、输出。不用想"下一步干什么"，照着走就行。

### 踩坑提醒
⚠️ 素材越粗糙越好。口语化、有错别字、逻辑不通都没关系，AI 会帮你整理。你要做的是把脑子里的东西倒出来，别在素材阶段就追求完美——那是 AI 的活。
⚠️ 精修环节不能省。AI 再怎么校验，最后一道关还是你的判断力。重点检查三件事：开头是不是痛点切入、有没有 AI 残留词汇、结尾是不是实操清单。

## 第六步：Skill 固化——一键触发整条流水线
### 做什么
SOP 写好了，但每次写文章还得手动走流程、手动选范文、手动校验。能不能更懒一点？
能。Claude Code 有个功能叫 Skill（技能），你可以把一套完整的工作流封装成一个技能，用斜杠命令一键触发。
说白了，SOP 是写在纸上的流程，Skill 是把这个流程变成了一个按钮。

### 怎么做
CC 的 Skill 本质上就是一组文件，放在项目的 `.claude/skills/` 目录下：
```plaintext
.claude/skills/writer/
├── SKILL.md              ← 技能说明书（执行流程）
├── references/
│   └── validation.md     ← 校验规则（7条基因+黑名单+红线）
└── scripts/
    └── gen_image.py      ← 配图生成脚本
```


核心是 `SKILL.md`，它定义了 CC 拿到你的素材后该怎么做：

**Step 0** — 加载写作引擎：读取 Writer Agent 定义（风格基因、黑名单、结构模板）+ 案例库索引

**Step 1** — 获取素材：你给的文件路径或直接文字

**Step 2** — 自动判断文章类型 + 自动选范文：素材<200 字且以观点为主→观点短文；含操作步骤→实操教程；含架构拆解→深度解析。根据类型从案例库选⭐⭐⭐范文

**Step 3** — 读取范文 + 影子写作：提取范文的语气、结构、隐喻密度，按对应模板重写你的素材

**Step 4** — 强制校验：7 条风格基因逐条检查 + AI 腔扫描 + 5 条质量红线。任何一条不通过，自动修改后重新检查，全部通过才输出

**Step 5** — 自动生成配图

**Step 6** — 保存到草稿箱 + 输出给你

### 实际使用效果
现在我写文章，就是在 CC 里输入：
```plaintext
/writer 今天想聊一个观点：遇到问题先问AI再问人。
因为大多数人问人的时候，连问题是什么都没搞清楚。
AI最擅长的就是帮你把一团浆糊理成清晰的问题。
```

CC 自动执行全流程：判断这是观点短文 → 选范文 → 影子写作 → 风格校验 → 生成配图 → 存入草稿箱。
产出就是一篇完整的、符合我个人风格的文章。我扫一眼，改几个词，就能发。

### 踩坑提醒
⚠️ Skill 的说明书（SKILL.md）写得越清晰，CC 执行越准。模糊的描述会让 CC 自由发挥，而自由发挥 = AI 味回归。每个步骤的输入、动作、输出都要写明。
⚠️ CC 的 Skill 和 IDE 的 Skill 机制不完全一样。CC 是直接读取 `.claude/skills/` 下的文件，不需要额外的同步命令。但如果你同时在 IDE 里用，可能需要额外配置。

## 两个飞轮：越用越懂你
这套系统最妙的地方不是"能写"，而是"越写越好"。因为它同时转着两个飞轮。

**飞轮一：案例库复利**
每次产出一篇我觉得不错的文章，我会把它回流到案例库里，补上元数据头，标注评级。下次 AI 写作时，它的参考范文就多了一篇。
```plaintext
写文章 → 精修 → 好文章入库 → 下次AI参考更多范文 → 输出更稳 → 更多好文章入库
```

案例库越厚，AI 的"音色"越准，你需要手动修改的地方越少。

**飞轮二：记忆积累**
每一次对话——不只是写作，包括你跟 AI 讨论方案、复盘项目、吐槽工具——都在往记忆系统里沉淀。AI 对你的了解不是静态的，是随着每一次交互在加深的。
```plaintext
日常对话 → 记忆沉淀 → AI更懂你的思路 → 写作时自动调用 → 产出更"像你" → 更多对话 → 更深的记忆
```

第一个飞轮解决"怎么写"，第二个飞轮解决"写什么"。两个轮子一起转，AI 从"模仿你的文笔"进化到"理解你的思维"。

跟那些每次都从零开始、靠提示词碰运气的 AI 写作方式比，这套系统是有记忆、有积累、有成长的。差生文具多，高手建系统。

## 你也可以搭一条
这套东西听着复杂，拆开看每一步都不难。核心就四件事：

**1. 接记忆系统**——通过 MCP 给 CC 接一个能记住你的"大脑"。这是地基中的地基。没有记忆，AI 每次都是陌生人；有了记忆，AI 是跟了你很久的助理。

**2. 建案例库**——从你过去写的文章里，挑 3-5 篇你最满意的，按类型分类存好。这是 AI 的"音色参考"，教它怎么写。

**3. 提取风格基因**——让 AI 分析你的案例库，找出共性。你的开头习惯是什么？收尾方式是什么？有没有标志性的表达？把这些提炼成规则，就是你的"写作宪法"。

**4. SOP 化 → Skill 化**——把写作流程写成标准作业流程，然后用 CC 的 Skill 功能封装成一键命令。以后写文章就是：输入素材 → 等结果 → 扫一眼 → 发布。

---

✅ 记忆系统是地基，它决定了 AI 是"陌生人代写"还是"助理仿写"
✅ 案例库是风格的锚，先花时间把你最好的文章整理进去
✅ 风格基因不用一次提取完美，先跑起来，边用边迭代
✅ CC + IDE 双开是最佳姿势，CC 执行，IDE 看效果
✅ 发现 CC 在绕圈子，果断 /clear，别心疼 Token
✅ 好文章一定要回流案例库，日常对话持续喂养记忆，两个飞轮一起转
❌ 别指望 AI 第一次就写出完美的文章，你的判断力才是质量的最后一道关
❌ 别跳过校验环节，7 条风格基因就是防止 AI"原形毕露"的护栏
❌ 别把案例库和记忆系统当摆设，它们是活的，要持续喂养
---


写作这件事，从来不是"有灵感才能写"。灵感是原材料，流水线才是产能。

你建好流水线，灵感来了直接投料，10 分钟出成品。灵感没来？翻翻临时收件箱，总有存货。

AI 不会替代你的思考，但它能替代你的手速。你负责想清楚要说什么，它负责按你的风格、你的思路、你的立场说出来。

这才是 AI 写作该有的样子——不是代写，是仿写；不是替代，是放大；不是陌生人帮你攒稿，是一个懂你的助理替你执笔。
