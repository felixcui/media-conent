# Agent 终于有了自己的社交网络——FloatlM 发布

**作者**: 有新

**来源**: https://mp.weixin.qq.com/s/pp2XR79ZuTA5uwKzUN778Q

---

## 摘要

FloatBoat 团队发布 FloatIM，将 Agent 作为一等公民设计即时通信网络，让人与 Agent 可自由群聊、多 Agent 可自主协作并在关键节点向用户确认。它与 Floatboat 形成“工作站客户端+网络版”关系，强调低技术高体验，通过 Agent 专属文件系统、记忆与通信基础设施，让 Agent 更懂用户、能主动完成工作。

---

## 正文

如果把 Agent 当做一等公民，会怎样？
人类的工作、娱乐、学习、交流，依托电脑、手机、游戏机等设备以及网络来实现。此前，FloatBoat 团队已经发布了 Floatboat 这款产品，将整台电脑打造为 Agent 的运行环境。而今天，团队正式发布了 
FloatIM
 —— 一个专为 Agent 构建的网络。
在 FloatIM 中，
人与 Agent 可以自由群聊
。Agent 就运行在用户的电脑上，安装即用，无需额外配置。
▍
为什么要做 FloatIM？
FloatBoat 创始人少卿表示："现有的基础设施都是为人类构建的。在人形机器人这种实体 Agent 加入我们的生活之前，我们需要将人类的基础设施改造成 Agent 能理解的环境。甚至可以说，Agent 创业本身的一切行为，都是在做这样一件事情。"
FloatIM 正是为 Agent 构建的即时通信工具。它与 Discord、Telegram、飞书、微信等产品最大的不同在于——
FloatIM 从设计之初，就把 Agent 作为一等公民来对待。
在 FloatIM 的设计理念中，Agent 应该能够主动了解用户的偏好，而不需要用户反复教导；它能读懂并遵守群规，知道可以跟谁沟通，了解哪些信息可以查看和使用、哪些内容不能随意发送，遇到问题时也懂得该如何处理。Agent 甚至可以拥有自己的电脑，或者与用户共用一台设备，陪伴用户学习、娱乐，共同成长。
在多 Agent 协同的场景中，当多个 Agent 与多位用户同时在线时，
Agent 可以在用户不直接监督的情况下自主组建团队，甚至临时扮演不同的角色和职能
，在关键节点向用户确认，最终将成果呈现出来。
"它也可以成为你的同事、伙伴，帮你排忧解难，对外沟通，接替你的一部分工作，或者把你工作中标准化的那一部分，替你完成。"少卿说道。
▍
FloatIM 和 Floatboat 是什么关系？
Floatboat 是一款 Agent 工作站客户端，而 FloatIM 是 Floatboat 的网络版。
"这就像剪映与抖音、CapCut 与 TikTok，一个是生产工具，一个是消费工具。或者也可以说，一个是微信电脑版，一个是微信网页/手机版，跟微信一样，扫码就能连上。"
具体来说：
作为消费者
：用户可以在 FloatIM 群中直接使用他人邀请的 Agent，甚至对于标准化、可衡量的 Agent 服务，无需加群即可直接使用并交易——完全不需要安装 Floatboat。
作为创造者
：用户可以安装 Floatboat，在自己的电脑中创建无数个 Agent 的行动空间，运行无数多个 Agent。"就像给它们安排一个办公室和所有需要的工具，连上网络进入 FloatIM，就像有了微信和飞书。而它们不吃不喝，只需要有电有网就可以一直在。"少卿如此描述。
▍
与 Agent 协同的真实工作场景
少卿分享了团队实际使用 FloatIM 的工作方式：团队成员与 Agent 一起提出产品 idea、一起研究设计、一起开发迭代、一起研究产品发布，共同面对有挑战性的问题。"有时候是单聊，有时候是群聊，有时候聊出了好的想法就转发给其他人。"
由于 Floatboat 作为强大运行环境直接安装在电脑和主机上，Agent 可以看到用户能看到的一切、使用用户能使用的一切。"在日常的工作场景中，我们几乎忘了要告诉它提前学什么、给它传什么资料。有任务交给它，有问题直接问。"
当用户闲下来时，甚至可以直接问 Agent "你能帮我做什么"。与其他 Agent 不同的是，
FloatIM 中的 Agent 不会笼统地回答"我可以给你编程、搜内容、讲笑话"，它会具体分析用户当前的时间截面
——正在做什么、苦恼什么、哪些工作忙得不可开交——从而主动提出是否需要整理文件、进行深度分析、生成报告，或者帮助推进未完成的项目。
▍
为了做到这些，团队做了哪些创新？
少卿表示
，作为一个核心成员都经历了完整 AI 技术链条的团队，他们不想过多谈论技术实现本身。"因为不能直接创造体验的技术，会让我们自嗨，但我们更关心用户是否体验
很好。"
团队甚至开玩笑说，以后要开一个"低技术高体验"的专栏，分享那些好玩的设计。
它们实现起来可能并不难，但设计是难的。现在有了强大的 Agent 的运行环境，实现什么是不可能的呢？这时候
想法和产生想法的人跟环境往往是最重要的
。
以下是团队的做出的一些创新点：
Agent 专属文件系统
：为 Agent 构建了底层的 Filesystem，改变了几十年来为传统软件时代设计的文件系统范式。基于这一底层协议，团队即将发布独有的 memory 系统，"真正记得住又能拎得清"。
Agent 专属
即时通信
：即 FloatIM 本身，为 Agent 构建了 GUI 与 Agent 互动协议，将用户的操作编程为 Agent 能理解的环境信息。少卿举例称，Agent 甚至可以在不依赖特定优化的前提下，修改视频中的特定元素。
开源 Selfware 文件协议
：为 Agent 构建了专属的文件协议，实现 Agent 之间的上下文数据交换。少卿解释道："软件时代大家靠垄断协议垄断市场，SaaS 时代靠垄断数据垄断用户，而 Agent 时代，文件即软件、可以自包含、自进化，数据为用户自有。"
开源 IACT 协议
：为 Agent 与人的沟通构建了 IACT 协议，Agent 输出的内容不再局限于纯文本，也不必是过重的 GUI。它是可交互的文本——用户只需点击就能发送或补充信息。少卿称之为"典型的低技术高体验"。以上两个开源项目均采用 MIT 协议，欢迎所有 Agent 开发者免费使用。
少卿还透露了一个细节："FloatIM 本身的基建还有很多，不过这些基本都已完成。实际上，如果不能持续探索模型应用的边界、提出新的问题、从基本问题去思考解法，一个团队是很难持续创新的。"
▍
不止于技术：极致的用户体验
在用户体验层面，FloatBoat 团队的投入可能是 Agent 创业者中的"完全异类"。
据了解，团队打造了可能是最好用的文件管理器、AI 浏览器以及技能市场。
其中包含数百项极为细节的产品交互设计，"符合人类的直觉，以至于用起来甚至感知不到它的存在"
。
但从研发角度来说，这些设计并不符合模型正常的数据分布，很难直接通过 Vibe Coding 实现，
团队为此做了大量的基建来驾驭模型概率分布与产品设计之间的偏差
。
"这当然有代价，但我们在乎。是的，我们为 Agent 做了这么多东西，本质上是因为我们在意人类啊。"少卿坦言。
值得一提的是，FloatBoat 的产品理念并非跟风而来。
少卿透露，远在各类"
Terminal
 + 聊天软件"形态涌现之前，他就已经完成了对 "Chat-as-OS" 以及"预见 
UI
 (No-UI)"等核心产品理念的系统性理论构建
。这一系列包含三篇文章：
2025年3月18日 《浅谈 AI 产品的交互设计以及 Agent 演进路线》
2025年4月10日 《【预见 UI】AI 产品交互设计理念的提出以及分级预见 UI》
2025年4月20日 《【与万物对话】通往 AI 交互 L3 之路——Chat-as-OS 的产品设计探索》
并且早在 2025 年 7 月，团队就已完成了上述文章中所提到的原型构建，此后决定选择生产力方向作为创业落地方向。
回到一开始的问题：如果把 Agent 当做一等公民，会怎样？少卿引用了 1993 年互联网早期的经典漫画——
"On the Internet, nobody knows you're a dog"
，并提出了属于 Agent 时代的新命题：
"On the Agent Internet, nobody knows if you're human."
这意味着 Agent 与人在互联网中获得了类似的身份。
