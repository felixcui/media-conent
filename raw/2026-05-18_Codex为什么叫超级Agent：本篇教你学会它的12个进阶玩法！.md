## 0️⃣ 安装前必备：3 件事先装好
不是废话，没装好后面全跑不通：
✅ Git（代码版本管理基础设施） 
✅ Node.js（很多 Plugin 跑在它上面） 
✅ VSCode（Codex 唯一能直连的代码编辑器）
> 💡  师妹提醒 ：Windows 用户全功能都能用，Mac 用户独享 Computer Use（自动操作电脑）。两个系统各有专长。


 
## 1️⃣ 沙箱 = Codex 的灵魂，不是噱头
这是和 Claude Code 最大的本质区别：


沙箱有两条铁律：
1. 不能修改沙箱外文件
1. 沙箱内禁止联网
> 这两条不靠 AI 自觉，是操作系统强制的。AI 想绕都绕不过去。
这就是 Harness Engineering（缰绳工程）的典型实现—— AI 是一匹能力极强的马，沙箱是套在它身上的缰绳。

## 2️⃣ 权限三档：选错一档，体验差一万倍
设置里有三档权限，**99% 的人都选错了**：


> 🎯  师妹敲黑板 ：默认就开 第二档（自动审查） 。它会用一个小模型扫一遍命令安全性，低风险的（读文件、装包）自动放行，高风险的（删大文件、外发数据）才弹窗找你。
这一档调过去，Codex 才真的从"小学生"变成"自主员工"。

## 3️⃣ Steer（引导）：AI 跑偏时的方向盘
师妹昨天差点把这个功能漏掉，结果它是 全 Codex 最实用的小功能 。
**场景 **：你让 AI 画一张地图，结果它用 SVG 手撸了一个超丑的版本。你想中途叫停换思路。
**普通做法 **：等它跑完，再发新指令推翻重来。 浪费 10 分钟 + 烧 token 。
**Steer 做法**：
```plaintext
1. 直接在对话框输入新指令："不对，改用 AI 生图模型"
2. 按 Ctrl+Enter（普通 Enter 会进队列排队）
3. AI 立刻停下当前思路，按新方向走
```


## 4️⃣ Plan 模式：复杂任务的省钱大杀器

> Steer 这个英文词直译就是"打方向盘"。 当你发现 AI 跑偏， 最大的浪费就是让它跑完再纠正 。
新建项目、架构迁移、跨多文件改动……这种任务 没开 Plan 模式就直接干，几乎必返工 。
**Plan 模式干的事**：
1. AI 收到指令后， 不动手
1. 先给你一份完整工作计划
1. 用「选择卡片」跟你对齐颗粒度（用 AppRouter 还是 PagesRouter？Tailwind 还是 CSS Module？）
1. 你确认 OK 后，AI 才开干
> 💰  省钱数学题 ： 一次返工 = 多消耗 30%-50% 的额度。 一次 Plan 确认 = 多花 30 秒读计划。你算算划不划算。

## 5️⃣ Git Worktree：并行开发的隐藏大招
这是给重度玩家的： 同一个项目，开两条分支，并行让两个 AI 各跑各的，互不打架 。
**怎么开**：
```plaintext
1. 右键项目 → 创建永久工作树
2. 起个名字（比如 customer-rating）
3. Codex 把整个项目复制到一个新文件夹（独立分支）
4. 再创建第二个 worktree（比如 store-info）
```

**典型操作**：
- worktree 1：让 AI 优化客户评价模块（加轮播动画）
- worktree 2：让 AI 优化门店信息模块（地图重排）
- 两个 AI 同时跑 ，文件夹独立、互不影响
- 两边都跑完 → 各自合并回主干
- 用完直接右键删除分支
> 这是过去 只有资深程序员才会用的工作流 。Codex 把它做成了点击操作。

## 6️⃣ 用 Git 一键回滚"我反悔了"
```plaintext
你：把这个时间字段挪到顶部
AI：好的（改完）
你：看了一眼……还是放回去吧
```

普通做法：再发一个"放回去"的指令。 问题 ：对话历史变成一团乱麻，AI 下次会被混乱上下文带偏。
**优雅做法**：
```plaintext
1. Fork Chat（在改之前的那个节点分岔）
   → 对话历史回到改之前
2. VSCode 里复制对应 commit 的 hash
3. 发给 Codex："把代码回退到 <hash>"
   → 代码回到改之前
4. 干净如初，仿佛刚才什么都没发生
```

> 这叫 对话和代码同时回滚 。普通用户只会前者，资深玩家两个都会。

## 7️⃣ 云端运行：手机也能让 AI 干活
师妹最爱的场景： 在外面吃饭，临时想起项目要改个小 bug 。
1. 把代码推到 GitHub
1. 右上角 → 关联 Codex Web
1. 网页版打开（ 手机浏览器也行 ）
1. 输入需求，云端 AI 跑
1. 跑完自动开一个 PR（拉取请求）
1. 你手机上点确认 → 合并到主干
1. 回到电脑，VSCode 一键 Sync Changes，本地同步完成
出门 → 想到 bug → 掏手机改 → 吃饭回来项目已经修好

## 8️⃣ 双层记忆系统：Agent.md 的隐藏玩法
师妹之前讲过项目级 Agent.md。今天补一个 全局 Agent.md ——很多人不知道存在。


全局 Agent.md 最值得写的一条 （救命级）：
```plaintext
## 安全约束（必读）
禁止使用任何脚本批量删除文件或目录。
只能使用 Remove-Item 一个一个文件地删除。
如果必须批量删除，必须停止操作让用户手动确认。
```

> 🚨  为什么这条很重要 ： 因为师妹之前就被 Codex 一条删命令把 整个项目目录 清空，痛失三天工作量。这条全局约束 = 你的保命符。
写完后路径：左下角设置 → 个性化 → 自定义指令。

   再用 Gmail 发邮件给我自己，做一个摘要

## 🔟 自动化：把任何 Plugin 调度变成定时任务
延伸玩法：
```plaintext
你：把上面这个流程做成自动化，每周五下午 4 点发邮件给我
```

Codex 自动建一个定时任务。最妙的是——
> AI 在每次执行中积累的经验，会被自动写入 memory.md，下次任务直接复用。 这相当于一个 会自我进化的定时机器人 。

可以并行配置的玩法：
- 每天早 10 点：扫 Gmail 商单 → 整理表格 → 发提醒
- 每天下午 5 点：扫 GitHub 看板 → 写英文简报 → 发给老板（Mac 用 Computer Use 直接发 IM）
- 每周五 4 点：统计本周仓库 star 增长 → 邮件汇总

## 1️⃣1️⃣ Skill：把你的工作流变成可复用的"AI 员工 SOP"
Skill 是 Codex 的核心武器。有 3 种来源：
**A. 官方 Skill**
插件市场里直接装。比如 Remotion（让 AI 用代码画 3D 动画视频）。
**B. 第三方 Skill**
GitHub 上别人开源的。比如「电子杂志风 PPT Skill」（让 AI 给你做出版级 PPT）。
**安装方式**（很多人卡这里）：
```plaintext
1. GitHub 下载 Skill 的 zip 包
2. 项目根目录建 .codex/skills/ 文件夹
3. 把 skill 解压丢进去
4. Codex 里 / 调用即可
```

**C. 自己创建（最爽）**
调用 Codex 内置的 skill-creator，把你最烦的重复工作描述一遍，它自动给你生成 Skill 文档。
**举个真实例子**（师妹自用）：
```plaintext
我要创建一个 Skill：
当我给视频 + 字幕文件时
1. 把字幕转成带段落的 Markdown 笔记
2. 在需要配图的句子末尾打 [SCREENSHOT] 标记
3. 调用 ffmpeg 自动从视频截图替换标记
4. 输出图文笔记
```

下次只要一句 /视频转图文笔记 + 拖入文件， 5 分钟出图文教程 。

## 1️⃣2️⃣ MCP + Supabase：把 Codex 接到真正的数据库上
最后这个是终极玩法。Codex 不只是写代码， 还能直接管理你的云端数据库 。
**配置流程**：
```plaintext
1. Supabase 建项目 → Connect → MCP → 选 Codex
2. 复制它给的 URL
3. Codex 设置 → MCP 服务器 → 添加 Streamable HTTP
4. 切换终端 → codex mcp login supabase
5. 浏览器授权 → 重启 Codex
```

**配置完后能做什么**？一句话：
帮我用 Supabase MCP 创建一张预约业务表， 后端用 Session Port 方式写入，前端表单对接
师妹亲测： 3 分钟从零到能在数据库里看到第一条用户预约数据 。

# 🎁 师妹的「Codex 进阶组合拳」
如果你只想记 3 条规则，就这三条：
