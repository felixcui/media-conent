# 哪些活，该交给Claude Code的 /workflows?

**作者**: 刘小排

**来源**: https://mp.weixin.qq.com/s/EOxvWU2zm8PCrTu_niSfUw

---

## 摘要

Claude Code的/workflows功能通过自动规划并调度大量Agent并行工作，突破了AI“智力超配但体力不足”的瓶颈。该功能特别适合可大规模解耦的“体力问题”而非需要连贯思考的“智力问题”。在实际应用中，它不仅能高效扫描代码库，精准揪出低风险且易修的陈年老bug，还能结合/deep-research功能，在兼顾本地代码上下文与外部深度调研的基础上，完美解决开发中遇到的复杂难题。

哈喽，大家好，我是刘小排。

上周，Claude Code新版本，伴随着Opus 4.8发布的，还有一个好玩的新功能： /workflows

前几天我发公众号文章征询大家对 /workflows 使用场景的建议 [Opus 4.8编程能力不如GPT-5.5](https://mp.weixin.qq.com/s?__biz=MzI1MTUxNzgxMA==&mid=2247502252&idx=1&sn=2f6c8884e9fe930f0e3af5ac2d796b1e&scene=21#wechat_redirect) ，刘润老师就微信私聊我了一些截图。

我发现，润总用 Claude Code /deep-research (背后其实也是 /workflows）做调研报告，效果真的很不错，同时开启了一百多个Agent，分工明确，各司其职，井然有序。

这给了我一些启发 ： **是否存在这样一些场景，以前AI Agent完成得比较勉强，现在有了 /workflows 理论上可以提升完成质量呢？**

我想，如果存在这样的场景，它们应该符合这些条件：

1. 1M 上下文不够，用 /workflows 能自动规划「到底需要多少个 Agent」，同时解决「装不下」的问题和「我懒得编排多Agent分工」的问题。
2. 任务可以大规模解耦，让串行变成并行。

做了两天的实验，两天就消耗完了 Claude Code 20x Max 的周限额（是的， /workflows 烧 Token 的确挺狠，还好今天凌晨 Claude Code也重置周限额了）。

是时候和大家分享了。

好，那这两天我到底试了什么?

---

先说个让我最爽的。

我让它去扫 Raphael 的代码库，找那种“低风险、能放心修”的陈年老 bug。

你知道那种 bug 吧—— **不致命，不紧急，但你心里清楚它在那。**

每次路过那段代码都心虚,但永远不会专门花一下午去修。因为你也怕：万一改了一个，崩了三个呢?

于是它就那么"陈"着。一年，两年。

我以前也这么放着。但这次,一个 /workflows 扔进去,它自己把代码库切成 12 个区，31 个 agent 同时扫，一批找、一批挑刺，置信度不够的直接扔。

两个小时后，甩给我一份清单。

我看了一遍——嘿，是我要的！真是那种“明显该修但一直没人修”的陈年老bug集群。

![图片](https://mmbiz.qpic.cn/sz_mmbiz_png/q6aOmBZKAbOOuLcz71LtibyRjib5iajHUTPhMfd1Q2tvQrYUc7ty9uYbjYic4lrgibs746cRLF2ib07VgQld2ZbGJHUqrA9riaUibq67IfgiaOnFpCqg/640?wx_fmt=png&from=appmsg)

那一刻的感觉是“握草，这些破玩意儿，终于有人替我翻出来了。"

然后我又试了个完全不一样的活：「我这个难题，别人一般如何解决」。

这个就是得到润总的启发。

如果只用/workflows： 可能信息茧房了。

如果只有/deep-research（以前Claude的网页版已经有/deep-research)： 无法阅读我当前代码库

因此，我的方案是： **遇到难题，同时用 /workflows 和 /deep-research**

如下图所示，一句话打开了105个Agent。

**最终效果是非常非常棒的！强烈推荐你也试试看！**

---

试玩之后，我继续思考， /workflows 到底带来了什么？

想来想去，就一句话——

**很多时候，AI的智力水平早就超配了，瓶颈是体力不足。**

一个再聪明的人，也读不完整个代码库——不是脑子不行，是装不下。

一个再能干的人，也没法同时改五十个文件——不是手不快,是只有一双。

/workflows 不让 Claude 变得更聪明，它让 Claude 变得更"多"。

所以，判断一件活适不适合/workflows，其实就问自己一句： **这是个“智力问题”,还是个“体力问题”?**

需要灵光一闪、需要连贯品味的——那是智力问题。

但凡“一个聪明人嫌烦，五十个普通人分头干就能干完”的——那是体力问题，这种情况下，放 /workflows出去，甚至叠加 /deep-research，双重buff，又快又好。

这可能就是AI Agent工具一个新的变化趋势。

它不是单纯让已经很天才的AI变得更加天才，而是AI更适合做苦活累活。
