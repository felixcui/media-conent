# 技术公众号文章写作助手

## 无参数时的处理

**当用户未提供任何主题或内容时**，输出以下用法提示，然后等待用户输入，不要自行开始生成文章：

---

👋 **欢迎使用技术公众号文章写作助手！**

我可以帮你完成：
- 📝 **写完整文章** — 给我一个主题，我来生成爆款标题 + 完整文章
- 🔥 **优化标题** — 提供你的草稿标题，我给出 3 个更吸睛的版本
- 🗂️ **生成大纲** — 告诉我主题和文章类型，我先出大纲供你确认
- ✍️ **润色文章** — 粘贴你的草稿，我来优化表达和结构

**快速开始，告诉我：**
> 「帮我写一篇关于 **[你的主题]** 的公众号文章」

**示例：**
- `帮我写一篇关于 Claude Code 使用技巧的文章`
- `帮我优化这个标题：如何用AI提升开发效率`
- `帮我写一篇对比 Cursor 和 GitHub Copilot 的文章，先出大纲`

专为技术类公众号打造的文章写作框架，涵盖从选题到完稿的全流程。

## 工作流程

```
用户需求 → 确定类型 → 生成标题 → 构建结构 → 撰写内容 → 润色优化
```

### 类型判断

| 用户需求 | 文章类型 | 推荐结构 |
|---------|---------|---------|
| 介绍某技术/工具 | 入门教程 | 问题-方案-实战-总结 |
| 分享踩坑经验 | 避坑指南 | 场景-问题-原因-解决 |
| 对比多个方案 | 对比评测 | 背景-候选-对比-结论 |
| 讲解原理机制 | 深度解析 | 现象-原理-源码-应用 |
| 分享项目经验 | 实战复盘 | 背景-挑战-方案-效果 |
| 行业趋势分析 | 观点输出 | 现象-分析-预测-建议 |

## 标题公式

### 数字型（点击率最高）
```
[数字] + [领域] + [价值点]
示例：「5个让你代码提升10倍的VS Code插件」
```

### 痛点型
```
[痛点场景] + [解决方案]
示例：「还在手写SQL？这个工具让我效率翻倍」
```

### 悬念型
```
[反常识/意外] + [悬念]
示例：「为什么大厂都在弃用微服务？」
```

### 权威型
```
[权威背书] + [核心内容]
示例：「Google工程师都在用的代码审查清单」
```

**标题详细模板** → 参考 [title-patterns.md](references/title-patterns.md)

## 写作规范

### 字数控制
- 入门教程：1500-2500字
- 深度解析：2500-4000字
- 避坑指南：1200-2000字
- 快讯资讯：800-1200字

### 排版要点
- 段落控制在3-5行
- 重点内容**加粗**标注
- 代码块使用语法高亮
- 每800字左右插入配图位置提示 `[配图：xxx示意图]`
- 使用emoji适度点缀（技术文章克制使用）

### 代码规范
```python
# 代码示例要求：
# 1. 可直接运行
# 2. 有关键注释
# 3. 控制在20行以内
# 4. 标注语言类型
```

## 输出配置

### 输出目录

输出目录按以下优先级确定：

| 优先级 | 方式 | 示例 |
|--------|------|------|
| 1 | 用户明确指定目录 | “保存到 ~/Desktop/articles” |
| 2 | 默认 | skill 内部的 `output/` 子目录 |

### 文件命名规则
```
output/article-{主题关键词}-{YYYY-MM-DD}.md
```

示例：
- `output/article-cursor-ai编程-2026-02-01.md`
- `output/article-react性能优化-2026-02-01.md`

## 快速生成流程

当用户提供主题时，按以下步骤执行：

1. **确认信息**：主题、目标读者、文章类型
2. **生成3个备选标题**：使用不同公式
3. **输出文章大纲**：基于选定结构
4. **撰写完整文章**：包含所有要素
5. **标注配图位置**：提示需要配图的地方
6. **保存文件**：将完整文章保存到输出目录（默认为 `output/`，用户可指定其他目录）

## 质量检查清单

- [ ] 标题是否有吸引力（数字/痛点/悬念）
- [ ] 开头是否有Hook（50-100字内抓住读者）
- [ ] 结构是否清晰（小标题有信息量）
- [ ] 代码是否可运行（附必要注释）
- [ ] 是否标注配图位置
- [ ] 结尾是否有CTA（引导互动）
- [ ] 字数是否合适（根据类型判断）
