# 我用 Hermes 搭了一个信息收藏助手

平时刷公众号、飞书文档、网页文章时，经常会遇到值得收藏的内容。以前我会先转发给自己，或者随手丢进某个收藏夹。问题是：收藏很容易，真正再找到、再利用，却很难。

所以我给自己搭了一个信息采集助手。目标很简单：**看到好内容，随手发出去；剩下的采集、整理、入库，交给系统自动完成。**

## 它是怎么工作的
![](../imgs/我搭了一个信息采集助手，把好内容自动存进知识库-20260626165654.png)

我把 Hermes 部署在家里的 Mac mini 上，平时通过飞书和它对话。

当我看到一篇想收藏的文章，只需要把 URL 发到飞书里。Hermes 收到后，会执行 `news_collect` 技能，自动完成几件事：

1. **抓取正文内容**  
   公众号和普通网页文章，会通过 Defuddle 提取正文；飞书文章则通过 `lark-cli` 获取内容。

2. **保存为 Markdown**  
   文章会被保存到本地 GitHub 仓库，变成可长期保存、可搜索、可迁移的 Markdown 文件。

3. **写入飞书多维表格**  
   系统会把 URL、标题、摘要等信息写入多维表格。这里更像一个内容数据库，也可以作为以后生成公众号周报的数据源。

4. **同步到 ima 知识库**  
   方便后续搜索、问答、总结和内容再加工。

5. **导入 NotebookLM**  
   用来做更深入的知识总结，也方便生成信息图、提纲和灵感素材。

最后，Mac mini 上的 Git 仓库会定期同步到远端；我的 MacBook 再定期拉取到本地，成为 Obsidian 里的参考资料库。

## 使用起来很简单

真正使用时，我只做一件事：把想收集的链接发给飞书。

![](../imgs/我的知识库-20260626153850.png)

![](../imgs/我搭了一个信息采集助手，把好内容自动存进知识库-20260626170143.png)
后面的流程都由 Hermes 自动处理，不需要我再手动复制、粘贴、改格式。

## 飞书多维表格：内容数据库

飞书多维表格主要承担“索引”和“数据源”的角色。

每条内容都会记录标题、链接、摘要等信息。这样做的好处是，后续无论是筛选主题、整理周报，还是回看某一类内容，都有结构化数据可以用。

![](../imgs/我的知识库-20260626154218.png)

## ima：用来搜索和总结

Markdown 文件适合保存原文，多维表格适合管理索引，而 ima 更适合做知识检索和总结。

当我想围绕某个主题找资料、提炼观点，或者生成一段内容初稿时，ima 就能派上用场。

![](../imgs/我的知识库-20260626154854.png)

## NotebookLM：用来深度消化

有些内容不只是收藏，还需要进一步理解、比较和转化。

这类内容进入 NotebookLM 后，可以用来做主题总结、资料归纳，也可以继续生成信息图、文章框架和选题线索。

![](../imgs/我的知识库-20260626155040.png)

## Obsidian：最终沉淀到本地

最后，所有内容都会回到我的本地 Obsidian 知识库。

Mac mini 定期把采集仓库同步到远端 Git 仓库，MacBook 再定期拉取到本地。这样我在日常写作、研究和整理资料时，就可以直接在 Obsidian 里引用这些内容。

![](../imgs/我的知识库-20260626160649.png)

## 小结

这套系统并不复杂，但解决了一个很实际的问题：**让信息从“随手收藏”变成“可检索、可整理、可复用”。**

我不再需要纠结一篇文章应该存在哪里，也不用担心收藏后再也找不到。看到有价值的内容，发给飞书就够了。

真正重要的不是多装几个工具，而是让工具之间连起来，形成一个稳定的信息流。这样，信息才会从碎片慢慢变成自己的知识资产。
