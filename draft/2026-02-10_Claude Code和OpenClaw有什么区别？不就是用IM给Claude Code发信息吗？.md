# Claude Code和OpenClaw有什么区别？不就是用IM给Claude Code发信息吗？

> 来源: [刘小排r](https://mp.weixin.qq.com/s/cJBh80KbpakKN0f5qkSX6Q)
> 时间: 2026-02-10

---

哈喽，大家好，我是刘小排。

Manus刚出的时候，就有人问：Manus和Claude Code有啥区别？不就是在Claude Code里内置一些工具吗？

Cowork刚出的时候，又有人问：Cowork和Claude Code有啥区别？不就是隐藏了写代码的界面吗？

OpenClaw出来的时候，还有人问：OpenClaw和Claude Code有啥区别？不就是通过IM发消息给Claude Code吗？

我认为，这种动不动就说“不就是”的人，不是一个好的学习者。也许他什么都没做错，他只是老了，无法接受新事物了。我们如果找一个八十岁、吃了一辈子肉夹馍的人去吃汉堡，他也可能觉得汉堡和肉夹馍没区别。

所有产品的演化，都是踩在巨人的肩膀上，进化论是地球上唯一靠谱的成功学。所以，新演化出来的好产品，一定是踩在了以前的好产品的肩膀上的，这不足为奇。

Claude Code 原本是一个编程工具。但是，我们发现，理论上，把电脑权限开到最大，Claude Code可以实现“通过自然语言交流，控制电脑的一切”。

这一点，早在2025年3月，就有人看到了。我之所以能在Claude Code上消耗了那么多Token，也是因为我很早就看到了这一点，我根本没主要用它写代码。 [揭秘 Claude Code 榜一大哥：一个 AI 创业者如何把工具用到极致｜对话刘小排](https://mp.weixin.qq.com/s?__biz=MzI1MTUxNzgxMA==&mid=2247498933&idx=1&sn=fa72632c3ed7141c0dfd6069f53e2cf0&scene=21#wechat_redirect)

Manus当然也看到了这一点。根据Manus创始团队叙述，最初让他们感到兴奋的，是一个叫做Browser-Use的开源项目，发现大模型可以自由控制浏览器、完成几乎一切浏览器上能完成的工作。但是**Manus有一个关键的洞见： Agent可以操作电脑，很好，就像实习生一样；但是，我不想让实习生和我抢电脑用啊！实习生应该有他自己的电脑！**

因此，Manus选择了“开网吧”的方式，让Agent在云端电脑（沙箱）上运行。

普通用户在使用Manus的时候，约等于在“网吧的共用电脑”上使用Agent。只是Manus的用户体验极好，没有技术背景的用户，可能都意识不到，任务是在一台单独的临时电脑上执行的。

Cowork继承了上面这两点，再叠加了另外两个洞见：

1.普通用户不需要看到写代码的过程

2.**Agent干活的时候，不是什么活都可以用“网吧电脑”实现的吧？我私人、我的企业，总有一些比较私密的资料和工具，还是得用我自己的电脑才行**。

因此，Cowork的产品形态，是一个看不到写代码过程的、在你自己电脑上运行的Agent。

OpenClaw在上面的基础上，又叠加了新的洞见：

1. 的确，很多场景下，Agent不能只靠网吧电脑，需要用我自己的电脑；但是，**没人规定，我只能有一台电脑吧？我能不能专门给实习生配置一台电脑，让它随便折腾？ 2. 我通知我的实习生干活，我是不需要坐在电脑前的吧？ 3. 实习生干完活应该自己会主动通知我才对，不需要我守着吧？**

从而，OpenClaw演化成了现在的形态： 通过IM发布任务、Agent随便操作一台自己的电脑搞坏了也不心疼。

基于这些原理，我才研究出了最好的使用OpenClaw的方式——真把它当人来用 [用了两周OpenClaw后，决定今年不招人类员工了](https://mp.weixin.qq.com/s?__biz=MzI1MTUxNzgxMA==&mid=2247501061&idx=1&sn=313eb32186afd04d4c12cedabe1b6321&scene=21#wechat_redirect) 

如果你能充分里面上面4个产品、4个阶段的所有原理，你也能用好OpenClaw。

短短一年的时间，多么美妙的过程啊！从Claude Code、到Manus、到Cowork、到OpenClaw，我们离“钢铁侠里的Jarvis”越来越近了。而这一切是真实发生的，不是科幻小说，真的让人感到难以置信。

不知道你是否能感受到其中进化论影子？ 每一次进化，都是在前人的基础上，叠加了新的技能。

洞见改变世界。
