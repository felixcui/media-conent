# 2026-04-23 摘要归档

## AI coding / 高风险后端 / Skill / RAG
- Harness Engineering：AI 能在真正“出事会炸”的后端系统里写代码吗？
  - 腾讯CDN LEGO 这类高风险、超大规模后端系统，核心不是“AI会不会写”，而是如何通过工程化手段把风险关进笼子。
  - 重点方法：Rust 版 Nonstop 零人工实现验证 AI 编码边界，再在 C++ LEGO 中落地 Harness Engineering 五层架构和多模型对抗式评审，建立从生成到上线的全链路质量屏障。
- 你写的 Skill，及格了吗？
  - 用 8 维度量化 Skill 质量，按生命周期分为：能否被发现、用起来是否顺畅、值不值得存在。
  - 关键在元数据质量、执行引导、工作流完整性、输入输出清晰度、领域知识密度、写作质量与范围聚焦。
- 【万字】抛开 RAG 谈蒸馏.skill，大概率是形式主义
  - “蒸馏.skill”容易沦为把流程和 SOP 模板化；若缺少高质量数据和知识支撑，很容易形式主义。
  - AI 原生团队应由一号位先用 AI；要让 AI 在真实业务中可信可用，核心仍是 RAG 接入私有、实时、可追溯知识。

  - GPT-Image-2（Images 2.0）分 Instant / Thinking 两种模式，后者可联网、自检并生成最多 8 张连贯图像。
  - 中文与非拉丁文字渲染、复杂构图、风格还原显著提升，知识截止到 2025 年 12 月，但生成内容逼真度也更高。
- 从技巧到 API，Image2 最完整解读
  - GPT-Image-2 已在 ChatGPT、Codex 和 API 全量上线，核心升级是“思考”能力。
  - 代价是价格上涨、2K 以上仍是 Beta、编辑请求 token 更高；适合在效果、成本与稳定性之间权衡。
- 可以优化生图提示词的提示词 / 2 / 3
  - 核心是 Visual Prompt Engine v3.0：把模糊想法转为结构化视觉指令。
  - 按角色、场景、叙事、产品、概念五类需求自动生成主体、环境、光影、色彩、镜头、风格等提示词，提高成片率并减少重试。
