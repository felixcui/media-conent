# Expanded Prompt — 如何用 Codex 画框架图

## Style block
Use the project design system: paper background `#F4F1E8`, ink `#121826`, muted ink `#4B5565`, panel `#FFFDF6`, hairline `#C9BEA8`, accent orange `#D45500`, blue `#2F4A7F`, teal `#0F766E`, yellow `#E4B649`. Display type uses generic serif; body uses generic sans-serif; labels use JetBrains Mono.

## Rhythm declaration
Pattern: hook-build-HOLD-build-PEAK-resolve. Six scenes across 34 seconds. Primary transition is blur-through crossfade with subtle paper push. Accent transition uses zoom-through into the process diagram scene.

## Global rules
Each scene has paper texture, ghost typography, structural rules, registration labels, and one focal content layer. Use large readable Chinese copy. All scenes enter with distinct GSAP choreography. Images are wrapped in device-like paper cards with perspective/slow internal zoom. No jump cuts.

## Scene beats
1. Hook: technical article authorship pain turns into a simple Codex workflow. Huge title, side note, one screenshot card, cursor dot and paper grid.
2. Old vs new: contrast drag-align-copy editing versus Codex generating an editable first draft. Two asymmetric cards and a drawn arrow.
3. Excalidraw: personal sketch workflow. Show article screenshot, three benefits: light, free, editable.
4. Process: from material to structure. Animated node graph: 자료/提炼/生成/调整. This is the peak.
5. Feishu Board: team collaboration and document sedimentation. Use Feishu screenshot with collaboration checklist.
6. Prompt + summary: show stable prompt template and final decision rule: Excalidraw for personal sketch, Feishu for team docs.

## Negative prompt
Avoid generic SaaS blue-purple gradients, tiny UI labels, identical card grids, unanimated decor, forced line breaks, and text overflow.
