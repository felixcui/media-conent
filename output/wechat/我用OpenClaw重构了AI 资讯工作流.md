# 我用 OpenClaw 重构了 AI 资讯工作流

![](../../../imgs/我用OpenClaw重构了AI资讯工作流-20260508230429.png)

> 做AI这行，每天要看的东西太多了。我之前的资讯流程有点乱：Coze抓公众号文章、飞书记收藏、飞书定时提醒...一堆工具凑合着用。每次想改点什么都很麻烦，直到把整套东西搬到OpenClaw上，才算省心了。

## 之前的状况

说下之前的情况吧：

- Coze工作流抓取公众号内容
- 飞书多维表格存收藏的文章
- 飞书例行任务做定时提醒
- 还有各种散落在书签、聊天记录里的"先马后看"

工具挺多，但用起来很折腾：

1. **平台太散**——好几个地方要维护，改个小功能要折腾半天
2. **触发不统一**——有些是定时的，有些要手动操作，经常忘事
3. **查东西麻烦**——想找一周前的某条资讯？翻飞书表格能翻到你怀疑人生
4. **不好扩展**——想加个新的信息源？得重新配一整套流程

## 用OpenClaw重新搭了一套

OpenClaw是个本地部署的AI工具，可以自定义技能、设定时任务、调外部API。我主要做了三件事：**采集、收藏、查询**。

### 每天早上自动推送资讯

现在每天7点，OpenClaw会自动抓取公众号的AI资讯，推到飞书群和公众号后台。

配置很简单：
- OpenClaw设个定时任务，每天7点触发
- 调用自己写的Skill抓公众号/RSS
- 格式化成飞书卡片消息
- 推到指定的飞书群和公众号

早上起来打开飞书，行业动态已经整整齐齐在群里了，不用再一个个去翻。

### 一键收藏文章

看到好文章，把URL发给OpenClaw就行。它会自动抓内容、写摘要、存到飞书多维表格。

以前收藏文章：复制链接 → 打开飞书 → 找多维表格 → 粘贴链接 → 写备注。至少5步。

现在：发个URL，一步搞定。

而且它写的摘要比我手写的强多了——能自动抓重点、打标签、分类主题，以后想找东西方便很多。



![收藏效果对比](../../../imgs/Pasted-image-20260507095344.png)

**多维表格中的存储效果：**

每条记录包含：文章标题、来源 URL、自动摘要、关键词标签、收藏时间。整洁、结构化、可检索。

![多维表格存储效果](../../../imgs/Pasted-image-20260507095408.png)
### 想查东西直接问

随时可以问OpenClaw，它会从收藏的资讯里找相关内容。

比如问"关于hermes的资讯？"，它就能从多维表格里检索出来。

和普通搜索不一样的是，它能理解语义。你记不清关键词，就说个大概意思，它也能找到。



![智能查询效果](../../../imgs/Pasted-image-20260507095422.png)
## 整体架构

画了个简单的流程图，大概是这样：

```
信息源（公众号/RSS/网页）
        │
        ▼
┌─────────────────┐
│   OpenClaw       │
│                  │
│  ┌────────────┐  │    ┌──────────────┐
│  │ 定时抓取    │──┼───▶ │  飞书推送     │
│  └────────────┘  │    └──────────────┘
│                  │
│  ┌────────────┐  │    ┌──────────────┐
│  │ URL 收藏   │──┼───▶│ 飞书多维表格  │
│  └────────────┘  │    └──────────────┘
│                  │         │
│  ┌────────────┐  │         │
│  │ 智能查询   │◀─┼─────────┘
│  └────────────┘  │
└─────────────────┘
```

核心就三个点：
- 所有操作都通过OpenClaw，不用切来切去
- 飞书多维表格作为唯一的数据存储
- 能自动的都自动，少做重复劳动

## 最后说两句
