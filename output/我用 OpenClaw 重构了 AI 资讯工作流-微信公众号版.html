作为一个 AI 从业者，每天要消化大量行业资讯。之前我的工作流散落在 Coze、飞书多维表格、例行任务等多个平台，维护成本高得离谱。直到我把这一切统一迁移到 OpenClaw——才真正体会到"一个工具搞定一切"的爽感。

做 AI 行业的人都知道，信息获取的速度决定了你的竞争力。但现实是，大多数人的资讯工作流是这样的：

  • • Coze 负责抓取公众号内容
  • • 飞书多维表格 负责存储收藏的资讯
  • • 飞书例行任务 负责定时提醒
  • • 各种浏览器书签、聊天记录里的"先马后看"

看起来分工明确，实际上维护成本极高：

  1. 1. 平台之间割裂——Coze 抓到的内容无法自动同步到飞书，每次都要手动搬运
  2. 2. 触发机制不统一——有的靠定时器，有的靠手动，经常遗漏
  3. 3. 查询能力薄弱——想回顾一周前的某条资讯？翻飞书表格翻到眼花
  4. 4. 扩展性差——想加一个新的信息源？得重新配置一套流程

这种碎片化的体验，就像用五把不同的钥匙开五扇门——每扇门都能打开，但你永远在找钥匙。

一、之前的痛点：资讯工作流的"碎片化之痛"

OpenClaw 是一个本地化部署的 AI 工具，支持自定义技能（Skills）、定时任务、外部 API 调用等能力。我围绕"采集 → 收藏 → 查询"三个核心环节，搭建了完整的工作流。

环节一：每日资讯自动抓取

做什么： 每天早上 7 点，自动获取微信公众号的最新 AI 资讯内容，并发送到飞书群。

怎么做到的：

  • • 利用 OpenClaw 的定时任务功能，设置每天 7:00 自动触发
  • • 通过自定义 Skill 调用 RSS/公众号内容源
  • • 抓取后自动格式化为飞书卡片消息
  • • 推送到指定的飞书群聊

效果： 早上睁眼打开飞书，AI 行业动态已经整齐排列在群里，像一份为你量身定制的"AI 日报"。

环节二：一键资讯收藏

做什么： 任何时间看到一篇好文章，只需要把 URL 发给 OpenClaw，它就会自动获取内容、生成摘要，并存入飞书多维表格。

怎么做到的：

  • • 向 OpenClaw 发送一条消息：文章的 URL
  • • OpenClaw 调用网页抓取 Skill，提取正文内容
  • • 利用 LLM 自动生成结构化摘要（标题、要点、标签）
  • • 通过飞书 API 写入多维表格的对应字段

为什么这个很重要：

以前收藏一篇文章的流程是：复制链接 → 打开飞书 → 找到多维表格 → 手动粘贴 → 写备注。至少 5 步操作。

现在：发一个 URL，1 步搞定。

而且 LLM 生成的摘要质量远超手动备注——它不仅提取核心观点，还能自动打标签、分类主题，后续检索效率提升了不止一个量级。

多维表格中的存储效果：

每条记录包含：文章标题、来源 URL、自动摘要、关键词标签、收藏时间。整洁、结构化、可检索。

环节三：智能资讯查询

做什么： 随时向 OpenClaw 提问，从已收藏的资讯库中检索相关内容。

怎么做到的：

  • • OpenClaw 连接飞书多维表格作为数据源
  • • 用户用自然语言提问，如"上周有哪些关于大模型安全的资讯？"
  • • LLM 理解意图，在多维表格中进行语义检索
  • • 返回最相关的资讯摘要和链接

和传统搜索的区别：

传统搜索是"关键词匹配"——你必须记得文章里出现过什么词。而 OpenClaw 的查询是"语义理解"——你说"AI 安全事故"，它也能找到标题是"数据泄露"的相关文章。

这种体验就像有一个读过你所有收藏文章的助手，你只需要问他问题。

四、迁移前后对比

1. 定时任务的容错

公众号抓取偶尔会因为网络波动或源站更新失败。建议在定时任务中加入重试机制和失败通知，这样即使某次抓取失败，你也能第一时间知道。

2. 摘要的 Prompt 很关键

LLM 生成摘要的质量取决于你的 Prompt 设计。建议在 Prompt 中明确：

  • • 摘要的长度限制（如 100-200 字）
  • • 必须包含的要素（核心观点、关键数据、涉及公司/产品）
  • • 标签的取值范围（预定义一套主题标签）

3. 飞书 API 的权限管理

多维表格的读写需要配置飞书应用的 API 权限。建议使用最小权限原则，只开放必要的表格读写权限，避免安全风险。

五、几点踩坑心得

这次工作流重构给我最大的感触是：好的工具不是让你做更多事，而是让你少做事。

OpenClaw 的价值不在于它本身有多强大，而在于它能把分散的工作流"粘合"成一个整体。以前我需要记住 5 个平台的操作方式，现在只需要和 OpenClaw 对话就够了。

如果你也在为资讯管理的碎片化头疼，不妨试试类似的思路——找到一个能串联所有环节的核心工具，把重复劳动交给自动化，把时间留给真正需要思考的事情。
